## 1. Введение в Flask

Flask — это легкий и гибкий микрофреймворк для создания веб-приложений на Python. Он предоставляет минимально необходимые инструменты для разработки, позволяя разработчику выбирать нужные расширения.

### Основные преимущества Flask:

* Простота в изучении
* Минималистичный подход
* Легко масштабируется
* Отличная документация
* Активное сообщество разработчиков
* Подходит для проектов различного размера

### Когда использовать Flask:

* Создание REST API
* Микросервисы
* Прототипирование веб-приложений
* Приложения обработки текста (актуально для компьютерной лингвистики)
* Веб-интерфейсы для лингвистических инструментов

## 3. Создание первого приложения

### Минимальное приложение Flask

Создайте файл app.py:

```python
from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello_world():
    return 'Hello, World!'

if __name__ == '__main__':
    app.run(debug=True)
```

### Запуск приложения

```bash
python app.py
```

Откройте браузер и перейдите на http://127.0.0.1:5000/

### Изучение кода

* **from flask import Flask** — импортирует класс Flask
* **app = Flask(__name__)** — создает экземпляр приложения
* **@app.route('/')** — декоратор, который связывает URL с функцией
* **app.run(debug=True)** — запускает приложение в режиме отладки

## 4. Маршруты и представления

### Маршруты (Routes)

Маршруты определяют, какие URL-адреса будут обработаны приложением.

```python
from flask import Flask

app = Flask(__name__)

# Простой маршрут
@app.route('/')
def home():
    return 'Welcome to Home Page'

# Маршрут с переменной пути
@app.route('/user/<name>')
def greet_user(name):
    return f'Hello, {name}!'

# Маршрут с преобразованием типа
@app.route('/post/<int:post_id>')
def show_post(post_id):
    return f'Post ID: {post_id}'

# Маршрут с несколькими переменными
@app.route('/search/<query>/<int:limit>')
def search(query, limit):
    return f'Search: {query}, Limit: {limit}'

if __name__ == '__main__':
    app.run(debug=True)
```

### HTTP методы

Маршруты могут обрабатывать различные HTTP методы:

```python
from flask import Flask, request

app = Flask(__name__)

# GET запрос
@app.route('/articles', methods=['GET'])
def get_articles():
    return 'Getting articles list'

# POST запрос
@app.route('/articles', methods=['POST'])
def create_article():
    return 'Article created'

# Оба метода одновременно
@app.route('/data', methods=['GET', 'POST'])
def handle_data():
    if request.method == 'POST':
        return 'Data received'
    else:
        return 'Retrieving data'

if __name__ == '__main__':
    app.run(debug=True)
```

### Перенаправления и ошибки

```python
from flask import Flask, redirect, url_for

app = Flask(__name__)

# Перенаправление
@app.route('/')
def index():
    return redirect(url_for('home'))

@app.route('/home')
def home():
    return 'Home Page'

# Обработка ошибок
@app.errorhandler(404)
def page_not_found(e):
    return 'Page not found', 404

@app.errorhandler(500)
def server_error(e):
    return 'Server error', 500

if __name__ == '__main__':
    app.run(debug=True)
```

## 5. Работа с шаблонами

Flask использует Jinja2 для создания динамических HTML шаблонов.

### Структура проекта

```
my_app/
├── app.py
├── templates/
│   ├── base.html
│   ├── index.html
│   └── article.html
└── static/
    ├── css/
    │   └── style.css
    └── js/
        └── script.js
```

### Базовый шаблон (templates/base.html)

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>{% block title %}My Flask App{% endblock %}</title>
    <link rel="stylesheet" href="{{ url_for('static', filename='css/style.css') }}">
</head>
<body>
    <header>
        <h1>My Application</h1>
        <nav>
            <a href="{{ url_for('home') }}">Home</a>
            <a href="{{ url_for('about') }}">About</a>
        </nav>
    </header>
    
    <main>
        {% block content %}{% endblock %}
    </main>
    
    <footer>
        <p>&copy; 2024 My Flask App</p>
    </footer>
</body>
</html>
```

### Дочерний шаблон (templates/index.html)

```html
{% extends "base.html" %}

{% block title %}Home - My Flask App{% endblock %}

{% block content %}
<h2>Welcome!</h2>
<p>{{ message }}</p>

<h3>Users List:</h3>
<ul>
    {% for user in users %}
        <li>{{ user['name'] }} - {{ user['email'] }}</li>
    {% endfor %}
</ul>
{% endblock %}
```

### Использование шаблонов в приложении

```python
from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')

    return render_template('about.html')
```

## 6. Введение в SQLite

SQLite — это встроенная система управления базами данных, которая идеально подходит для простых приложений и прототипирования.

### Основные концепции

* **Таблица** — структура для хранения данных с определенными колонками
* **Поле (Column)** — отдельный атрибут в таблице
* **Запись (Record)** — одна строка данных в таблице
* **Первичный ключ (Primary Key)** — уникальный идентификатор для каждой записи

### Типы данных в SQLite

* **INTEGER** — целые числа
* **REAL** — вещественные числа
* **TEXT** — текстовые строки
* **BLOB** — двоичные данные
* **NULL** — отсутствие значения

### Создание базы данных вручную

```python
import sqlite3

# Подключение к БД (создается, если не существует)
connection = sqlite3.connect('my_database.db')
cursor = connection.cursor()

# Создание таблицы
cursor.execute('''
    CREATE TABLE IF NOT EXISTS users (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        name TEXT NOT NULL,
        email TEXT UNIQUE NOT NULL,
        age INTEGER
    )
''')

# Добавление данных
cursor.execute(
    'INSERT INTO users (name, email, age) VALUES (?, ?, ?)',
    ('John Doe', 'john@example.com', 25)
)

connection.commit()
connection.close()
```

## 7. Работа с базами данных в Flask

### Настройка Flask-SQLAlchemy

Создайте файл app.py:

```python
from flask import Flask, render_template, request, redirect, url_for
from flask_sqlalchemy import SQLAlchemy

app = Flask(__name__)
app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///language_app.db'
app.config['SQLALCHEMY_TRACK_MODIFICATIONS'] = False

db = SQLAlchemy(app)

# Определение модели
class Word(db.Model):
    __tablename__ = 'words'
    
    id = db.Column(db.Integer, primary_key=True)
    word = db.Column(db.String(100), unique=True, nullable=False)
    definition = db.Column(db.Text, nullable=False)
    part_of_speech = db.Column(db.String(50), nullable=False)
    language = db.Column(db.String(50), default='English')
    created_at = db.Column(db.DateTime, default=db.func.now())
    
    def __repr__(self):
        return f'<Word {self.word}>'

# Создание таблиц
with app.app_context():
    db.create_all()

@app.route('/')
def index():
    words = Word.query.all()
    return render_template('index.html', words=words)

@app.route('/add', methods=['GET', 'POST'])
def add_word():
    if request.method == 'POST':
        word = request.form.get('word')
        definition = request.form.get('definition')
        part_of_speech = request.form.get('part_of_speech')
        
        new_word = Word(
            word=word,
            definition=definition,
            part_of_speech=part_of_speech
        )
        
        db.session.add(new_word)
        db.session.commit()
        
        return redirect(url_for('index'))
    
    return render_template('add_word.html')

@app.route('/word/<int:word_id>')
def view_word(word_id):
    word = Word.query.get_or_404(word_id)
    return render_template('word_detail.html', word=word)

@app.route('/edit/<int:word_id>', methods=['GET', 'POST'])
def edit_word(word_id):
    word = Word.query.get_or_404(word_id)
    
    if request.method == 'POST':
        word.word = request.form.get('word')
        word.definition = request.form.get('definition')
        word.part_of_speech = request.form.get('part_of_speech')
        
        db.session.commit()
        return redirect(url_for('view_word', word_id=word.id))
    
    return render_template('edit_word.html', word=word)

@app.route('/delete/<int:word_id>')
def delete_word(word_id):
    word = Word.query.get_or_404(word_id)
    db.session.delete(word)
    db.session.commit()
    
    return redirect(url_for('index'))

if __name__ == '__main__':
    app.run(debug=True)
```

### Операции с БД

#### Добавление данных

```python
# Способ 1

```

#### Чтение данных

```python
# Получить все записи
all_users = User.query.all()

# Получить первую запись
first_user = User.query.first()

# Получить запись по ID
user = User.query.get(1)

# Получить запись по условию
user = User.query.filter_by(name='John').first()

# Фильтрация
active_users = User.query.filter(User.age > 18).all()

# Сортировка
users = User.query.order_by(User.name).all()

# Ограничение количества

```

#### Обновление данных

```python

user.name = 'New Name'

```

#### Удаление данных


### Работа с отношениями между таблицами

```python
class Author(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(100), nullable=False)
    books = db.relationship('Book', backref='author', lazy=True, cascade='all, delete-orphan')

class Book(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    title = db.Column(db.String(200), nullable=False)
    author_id = db.Column(db.Integer, db.ForeignKey('author.id'), nullable=False)

# Использование
author = Author.query.get(1)

book = Book.query.get(1)
```

### Полезные советы при разработке

1. Всегда используйте виртуальное окружение для проекта
2. Разделяйте логику БД от представлений используя модели
3. Используйте параметризованные запросы для безопасности
4. Регулярно совершайте commit в систему контроля версий
5. Тестируйте приложение с различными входными данными
6. Документируйте код с использованием docstring
7. Используйте логирование для отладки приложения

### Типичные ошибки новичков

1. Забыли создать папку templates/ в корне проекта
2. Неправильный путь к шаблонам в render_template()
3. Забыли вызвать db.session.commit() после изменений
4. Путаница между query.get() и query.filter()
5. Забыли декоратор @app.route при создании новой функции
6. Неправильная конфигурация пути к базе данных
7. Использование изменяемых объектов как значений по умолчанию

## Заключение

Flask — это мощный и гибкий инструмент для создания веб-приложений на Python. Совокупность с SQLite и SQLAlchemy, он предоставляет полный набор инструментов для разработки современных веб-приложений, включая приложения для компьютерной лингвистики, обработки текста и создания интерфейсов к лингвистическим инструментам.

Практика является ключом к освоению Flask. Начните с простых упражнений и постепенно переходите к более сложным проектам. Не бойтесь экспериментировать и искать решения в документации и сообществе.
