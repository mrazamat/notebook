# Техники автоматического исправления текста: Обнаружение и исправление ошибок в естественном языке

## Введение

Автоматическое исправление текста (Automatic Text Correction, ATC) — это область компьютерной лингвистики, которая занимается выявлением и корректировкой различных типов ошибок в письменном тексте. Это могут быть опечатки, орфографические, грамматические и синтаксические ошибки.

Задача коррекции текста актуальна для множества приложений:
- Проверка орфографии в текстовых редакторах
- Распознавание речи и OCR-системы
- Улучшение качества пользовательского ввода
- Обработка социальных сетей и текстов с неформальным стилем

## 1. Классификация ошибок

### 1.1 Типы ошибок

Ошибки в естественном языке можно классифицировать по различным признакам:

**По природе происхождения:**

| Тип | Описание | Пример |
|-----|---------|--------|
| Опечатка (Typo) | Случайное нажатие неправильной клавиши | "программа" вместо "программа" |
| Орфографическая ошибка | Неправильное написание слова | "привет" вместо "привет" |
| Грамматическая ошибка | Ошибка в грамматической структуре | "Он идет в школа" (должно "в школу") |
| Пунктуационная ошибка | Неправильное использование знаков препинания | "Привет, мир" вместо "Привет мир" |

**По количеству измененных символов:**

- **Non-word error**: опечатка создает несуществующее слово ("teh" вместо "the")
- **Real-word error**: опечатка создает реальное, но неправильное слово ("their" вместо "there")

### 1.2 Сложность задачи

Наиболее сложными являются:
- Real-word ошибки (требуют контекстного анализа)
- Множественные ошибки в одном предложении
- Ошибки в редких или специализированных словах
- Ошибки в текстах на неформальном языке

## 2. Основные подходы к исправлению

### 2.1 Подход на основе расстояния Левенштейна

**Расстояние Левенштейна** (Edit Distance) — это минимальное количество операций редактирования, необходимых для преобразования одной строки в другую.

**Допустимые операции:**
- Вставка символа (insertion)
- Удаление символа (deletion)
- Замена символа (substitution)

**Формула:**

Для строк s₁ и s₂, расстояние Левенштейна d(s₁, s₂) вычисляется по формуле динамического программирования:

```
d[i][j] = min(
    d[i-1][j] + 1,           // удаление
    d[i][j-1] + 1,           // вставка
    d[i-1][j-1] + cost       // замена (cost = 0 если s₁[i] == s₂[j], иначе 1)
)
```

**Пример работы:**

```
Исходное слово: "kitten"
Целевое слово: "sitting"

Операции:
1. kitten → sitten (замена k на s)
2. sitten → sittin (замена e на i)
3. sittin → sitting (вставка g)

Расстояние = 3
```

**Применение для коррекции:**

Для исправления слова:
1. Вычислить расстояние от неправильного слова до всех слов в словаре
2. Выбрать словарные слова с наименьшим расстоянием
3. Вернуть топ-N кандидатов

### 2.2 N-граммные модели

**N-граммы** — это последовательности из N объектов (символов или слов).

**Виды:**
- Юниграммы (1-граммы): отдельные символы или слова
- Биграммы (2-граммы): пары соседних элементов
- Триграммы (3-граммы): тройки соседних элементов

**Использование для обнаружения ошибок:**

N-граммные модели помогают выявить маловероятные последовательности, которые могут указывать на ошибку:

```
Вероятность биграммы "teh the" при условии "the":
P(teh | context) << P(the | context)
```

**Вероятность N-граммы:**

$$P(w_n | w_1, w_2, \ldots, w_{n-1}) = \frac{count(w_1, w_2, \ldots, w_n)}{count(w_1, w_2, \ldots, w_{n-1})}$$

где count() — количество появлений последовательности в корпусе.

### 2.3 Вероятностные модели

**Модель канала шума** (Noisy Channel Model)

Модель предполагает, что исходный текст был "искажен" некоторыми ошибками:

```
Чистый текст → [Канал шума] → Наблюдаемый текст
```

Задача: найти исходный текст, который с наибольшей вероятностью мог бы привести к наблюдаемому тексту.

**Формула Байеса:**

$$P(correct | observed) = \frac{P(observed | correct) \times P(correct)}{P(observed)}$$

где:
- **P(correct)** — априорная вероятность правильного слова в языке (языковая модель)
- **P(observed | correct)** — вероятность опечатки (модель ошибки)
- **P(observed)** — нормализирующий множитель

**Упрощенная версия:**

$$correct = \arg\max_{c} P(c) \times P(observed | c)$$

## 3. Практические алгоритмы

### 3.1 Простой алгоритм исправления опечаток

**Шаги:**

1. **Токенизация**: разбить текст на слова
2. **Проверка**: для каждого слова проверить, есть ли оно в словаре
3. **Кандидаты**: если слово отсутствует, найти k ближайших слов по расстоянию Левенштейна
4. **Выбор**: выбрать кандидата с наибольшей вероятностью в языке

**Псевдокод:**

```
function correct_typo(word, dictionary, max_distance=2):
    if word in dictionary:
        return word
    
    candidates = []
    for dict_word in dictionary:
        dist = levenshtein_distance(word, dict_word)
        if dist <= max_distance:
            candidates.append((dict_word, dist))
    
    if len(candidates) == 0:
        return word  // исправление невозможно
    
    // сортировать по расстоянию, затем по частоте
    candidates.sort(by: distance, then by: frequency)
    return candidates[0]
```

## 4. Языковые модели

### 4.1 N-граммные языковые модели

**Построение модели:**

1. Подготовить корпус текстов на целевом языке
2. Извлечь все N-граммы
3. Подсчитать частоты появления
4. Нормализовать (вычислить вероятности)

**Пример для биграмм:**

```
Предложение: "Кот сидит на коврике"
Биграммы: ["кот сидит", "сидит на", "на коврике"]

Если "кот сидит" встречается 150 раз из 1000 биграмм,
то P("сидит" | "кот") = 150 / 1000 = 0.15
```

### 4.2 Проблема разреженности данных

Проблема: многие N-граммы встречаются редко или вообще не встречаются в корпусе.

**Решение: Сглаживание (Smoothing)**

**Сглаживание Лапласа (Add-One Smoothing):**

$$P(w_n | w_1, \ldots, w_{n-1}) = \frac{count(w_1, \ldots, w_n) + 1}{count(w_1, \ldots, w_{n-1}) + V}$$

где V — размер словаря.

**Интерполяция:**

$$P(w_n | w_{n-2}, w_{n-1}) = \lambda_3 P_3(w_n | w_{n-2}, w_{n-1}) + \lambda_2 P_2(w_n | w_{n-1}) + \lambda_1 P_1(w_n)$$

где λ₁ + λ₂ + λ₃ = 1

## 5. Метрики оценки

### 5.1 Основные метрики

**Точность (Accuracy):**
$$Accuracy = \frac{correctly\_corrected}{total\_errors}$$

**Полнота (Recall):**
$$Recall = \frac{correctly\_corrected}{total\_errors}$$

**Precision (в контексте выбора кандидата):**
$$Precision = \frac{correct\_suggestions}{total\_suggestions}$$

**F1-мера:**
$$F1 = 2 \times \frac{Precision \times Recall}{Precision + Recall}$$

### 5.2 Специфические метрики для коррекции

- **Correction Rate**: процент ошибок, которые были исправлены
- **False Positive Rate**: процент правильных слов, которые были ошибочно "исправлены"
- **Average Edit Distance**: среднее расстояние между предложением с ошибками и исправленным

## 6. Практические примеры на Python

### 6.1 Реализация расстояния Левенштейна

```python
def levenshtein_distance(s1, s2):
    """
    Вычисление расстояния Левенштейна между двумя строками.
    
    Args:
        s1: первая строка
        s2: вторая строка
    
    Returns:
        минимальное количество операций редактирования
    """
    m, n = len(s1), len(s2)
    
    # инициализация таблицы DP
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    
    # заполнение первой строки и столбца
    for i in range(m + 1):
        dp[i][0] = i
    for j in range(n + 1):
        dp[0][j] = j
    
    # заполнение таблицы
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]:
                dp[i][j] = dp[i-1][j-1]
            else:
                dp[i][j] = 1 + min(
                    dp[i-1][j],      # удаление
                    dp[i][j-1],      # вставка
                    dp[i-1][j-1]     # замена
                )
    
    return dp[m][n]


# Примеры использования
print(levenshtein_distance("kitten", "sitting"))  # 3
print(levenshtein_distance("привет", "привет"))  # 0
print(levenshtein_distance("teh", "the"))        # 1
```

### 6.2 Простой исправитель опечаток

```python
def simple_spell_checker(word, dictionary, max_distance=2):
    """
    Простой исправитель опечаток на основе расстояния Левенштейна.
    
    Args:
        word: слово для проверки
        dictionary: словарь с частотами {слово: частота}
        max_distance: максимальное расстояние Левенштейна
    
    Returns:
        исправленное слово или исходное слово
    """
    # если слово в словаре, оно правильное
    if word in dictionary:
        return word
    
    candidates = []
    
    # найти все кандидаты в пределах max_distance
    for dict_word in dictionary.keys():
        dist = levenshtein_distance(word.lower(), dict_word.lower())
        if dist <= max_distance:
            frequency = dictionary[dict_word]
            candidates.append((dict_word, dist, frequency))
    
    # если кандидатов нет
    if not candidates:
        return word
    
    # сортировать по расстоянию, потом по частоте (убывание)
    candidates.sort(key=lambda x: (x[1], -x[2]))
    
    return candidates[0][0]


# Пример использования
dictionary = {
    "привет": 1000,
    "мир": 800,
    "программа": 500,
    "ошибка": 450
}

test_words = ["привет", "pривет", "priwet", "program"]

for word in test_words:
    corrected = simple_spell_checker(word, dictionary)
    print(f"{word} → {corrected}")
```

### 6.3 N-граммная модель

```python
from collections import defaultdict, Counter
import math

class NGramLanguageModel:
    """
    Простая N-граммная языковая модель.
    """
    def __init__(self, n=2):
        self.n = n
        self.ngrams = defaultdict(Counter)
        self.context_counts = Counter()
    
    def train(self, tokens):
        """
        Обучение модели на последовательности токенов.
        
        Args:
            tokens: список токенов (слов)
        """
        # добавить специальные токены начала и конца
        tokens = ["<START>"] * (self.n - 1) + tokens + ["<END>"]
        
        # извлечь N-граммы
        for i in range(len(tokens) - self.n + 1):
            ngram = tuple(tokens[i:i+self.n])
            context = ngram[:-1]
            word = ngram[-1]
            
            self.ngrams[context][word] += 1
            self.context_counts[context] += 1
    
    def get_probability(self, word, context):
        """
        Получить вероятность слова в контексте с сглаживанием Лапласа.
        
        Args:
            word: слово
            context: кортеж контекста
        
        Returns:
            вероятность слова
        """
        context = tuple(context[-(self.n-1):])
        
        count = self.ngrams[context].get(word, 0)
        total = self.context_counts[context]
        vocab_size = len(set(w for words in self.ngrams.values() 
                            for w in words.keys()))
        
        # сглаживание Лапласа
        probability = (count + 1) / (total + vocab_size)
        
        return probability
    
    def score_sentence(self, tokens):
        """
        Вычислить логарифм вероятности предложения.
        
        Args:
            tokens: список токенов
        
        Returns:
            логарифм вероятности
        """
        tokens = ["<START>"] * (self.n - 1) + tokens + ["<END>"]
        score = 0.0
        
        for i in range(len(tokens) - self.n + 1):
            ngram = tokens[i:i+self.n]
            context = ngram[:-1]
            word = ngram[-1]
            
            prob = self.get_probability(word, context)
            score += math.log(prob)
        
        return score


# Пример использования
training_sentences = [
    ["кот", "сидит", "на", "коврике"],
    ["кот", "спит", "на", "диване"],
    ["собака", "сидит", "на", "полу"],
    ["птица", "летит", "в", "небе"]
]

# объединить все токены
all_tokens = [token for sent in training_sentences for token in sent]

# обучить модель
model = NGramLanguageModel(n=2)
model.train(all_tokens)

# примеры
test_sent1 = ["кот", "сидит"]
test_sent2 = ["кот", "летит"]

print(f"Score for '{' '.join(test_sent1)}': {model.score_sentence(test_sent1)}")
print(f"Score for '{' '.join(test_sent2)}': {model.score_sentence(test_sent2)}")
```

### 6.4 Контекстное исправление

```python
def contextual_correction(sentence, checker_dict, language_model, tokenizer):
    """
    Исправление опечаток с учетом контекста.
    
    Args:
        sentence: исходное предложение
        checker_dict: словарь для проверки {слово: частота}
        language_model: языковая модель для оценки контекста
        tokenizer: функция токенизации
    
    Returns:
        исправленное предложение
    """
    tokens = tokenizer(sentence)
    corrected_tokens = []
    
    for i, token in enumerate(tokens):
        # если слово правильное, оставить как есть
        if token in checker_dict:
            corrected_tokens.append(token)
            continue
        
        # найти кандидатов для исправления
        candidates = []
        for dict_word in checker_dict.keys():
            dist = levenshtein_distance(token.lower(), dict_word.lower())
            if dist <= 2:
                candidates.append(dict_word)
        
        if not candidates:
            corrected_tokens.append(token)
            continue
        
        # выбрать кандидата с наибольшей вероятностью в контексте
        best_candidate = token
        best_score = float('-inf')
        
        for candidate in candidates:
            # создать предложение с кандидатом
            test_tokens = corrected_tokens + [candidate] + tokens[i+1:]
            
            # оценить вероятность
            score = language_model.score_sentence(test_tokens)
            
            if score > best_score:
                best_score = score
                best_candidate = candidate
        
        corrected_tokens.append(best_candidate)
    
    return ' '.join(corrected_tokens)


# Пример использования
sentence = "кот сидит нa кофире"
corrected = contextual_correction(
    sentence, 
    dictionary,
    model,
    lambda s: s.split()
)
print(f"Исходное: {sentence}")
print(f"Исправленное: {corrected}")
```

## 8. Дополнительные техники

### 8.1 Алгоритм Шумского (Soundex)

Soundex — это фонетический алгоритм для поиска похожих по звучанию слов.

**Основная идея:**
- Преобразовать слово в четырехзначный код
- Использовать коды для поиска похожих слов

**Алгоритм:**

```python
def soundex(word):
    """
    Вычисление Soundex кода слова.
    """
    word = word.upper()
    
    # словарь кодов для согласных
    codes = {
        'B': '1', 'F': '1', 'P': '1', 'V': '1',
        'C': '2', 'G': '2', 'J': '2', 'K': '2', 'Q': '2', 'S': '2', 'X': '2', 'Z': '2',
        'D': '3', 'T': '3',
        'L': '4',
        'M': '5', 'N': '5',
        'R': '6'
    }
    
    # первый символ
    soundex_code = word[0]
    prev_code = codes.get(word[0], '0')
    
    # обработать остальные символы
    for char in word[1:]:
        code = codes.get(char, '0')
        
        if code != '0' and code != prev_code:
            soundex_code += code
        
        if len(soundex_code) == 4:
            break
        
        prev_code = code
    
    # дополнить нулями до 4 символов
    soundex_code = (soundex_code + '000')[:4]
    
    return soundex_code
```

### 8.2 Преобразование клавиатуры

Еще один источник ошибок — соседние клавиши на клавиатуре.

**Пример для русской ЙЦУКЕН клавиатуры:**

```python
keyboard_layout = {
    'й': ['ц', 'ы'], 'ц': ['й', 'у'], 'у': ['ц', 'к'],
    'к': ['у', 'е'], 'е': ['к', 'н'],
    # ... и так далее для всей клавиатуры
}

def get_keyboard_neighbors(char):
    """Получить соседние символы на клавиатуре."""
    return keyboard_layout.get(char, [])
```

```python
def morphology_aware_correction(word, morphology_dict):

    Исправление с учетом морфологии.
    
    Args:
        word: слово для исправления
        morphology_dict: словарь с морфологической информацией
    
    Returns:
        исправленное слово
    """
    # получить морфологические леммы и формы
    lemmas = morphology_dict.get_lemmas(word)
    
    # генерировать все возможные формы лемм
    all_forms = []
    for lemma in lemmas:
        forms = morphology_dict.get_forms(lemma)
        all_forms.extend(forms)
    
    # выбрать форму с наименьшим расстоянием
    best_match = min(all_forms, key=lambda f: levenshtein_distance(word, f))
    
    return best_match
```

## 9. Современные подходы

### 9.1 Нейросетевые модели

Современные системы исправления текста используют глубокие нейронные сети:

- **RNN/LSTM**: последовательные модели для контекстного анализа
- **Transformer**: модели на основе внимания (например, BERT)
- **Encoder-Decoder**: архитектура seq2seq для исправления

**Преимущества нейросетевых подходов:**
- Автоматическое изучение закономерностей из данных
- Высокая точность на large-scale данных
- Способность обрабатывать сложные контекстные зависимости

### 9.2 Гибридные подходы

### 10.1 Выбор метода

|-------|-----------|-------|--------|
| Расстояние Левенштейна | Опечатки | Простой, быстрый | Не учитывает контекст |
| N-граммы | Non-word ошибки | Эффективный | Зависит от корпуса |

### 10.2 Практические советы

2. **Словарь**: используйте актуальный словарь, адаптированный к вашему домену

3. **Контекст**: всегда учитывайте контекст для real-word ошибок

4. **Балансировка**: правильно балансируйте между агрессивностью исправления и сохранением исходного текста

5. **Тестирование**: используйте размеченные корпусы для оценки качества

## Заключение

Автоматическое исправление текста — это важная и сложная задача в обработке естественного языка. Эффективное решение требует комбинирования различных подходов: от простых алгоритмов на основе расстояния редактирования до сложных нейросетевых моделей.

Выбор метода зависит от:
- Типа ошибок, которые нужно исправлять
- Объема доступных данных
- Требований к точности и скорости
- Характеристик целевого языка

Понимание основных концепций и методов позволяет создавать более эффективные системы исправления текста, адаптированные к конкретным задачам.

## Глоссарий терминов

**Opечатка (Typo)**: случайное нажатие неправильной клавиши при вводе текста

**Edit Distance (Расстояние редактирования)**: минимальное количество операций редактирования для преобразования одной строки в другую

**N-грамма**: последовательность из N объектов (символов или слов) в тексте

**Языковая модель (Language Model)**: вероятностная модель, определяющая вероятность последовательности слов в языке

**Канал шума (Noisy Channel)**: модель, предполагающая, что правильный текст был искажен ошибками

**Сглаживание (Smoothing)**: техника для обработки редких или неизвестных N-грамм

**Real-word ошибка**: опечатка, которая создает реальное, но неправильное слово

**Non-word ошибка**: опечатка, которая создает несуществующее слово

**Морфологический анализ**: определение морфологических признаков слова (часть речи, род, число и т.д.)

**Контекстное исправление**: исправление ошибок с учетом окружающих слов

**Фонетический алгоритм**: алгоритм для поиска слов, похожих по звучанию
