## Введение в машинный перевод {#введение}

### Определение

**Машинный перевод (Machine Translation, MT)** — это автоматическое преобразование текста с одного естественного языка (исходный язык, source language) на другой (целевой язык, target language) с использованием компьютерных алгоритмов.

### Краткая история

- **1950-е годы:** Первые попытки автоматического перевода (система "Georgetown-IBM", 1954)
- **1960-е–1980-е:** Развитие правило-ориентированных систем (RBMT)
- **1980-е–2010-е:** Доминирование статистических методов (SMT)
- **2014–настоящее:** Революция нейронных сетей (NMT, Transformer)

### Основная схема перевода

```
Исходный текст → Анализ → Внутреннее представление → Синтез → Целевой текст
```

Различие между SMT и NMT заключается в том, **как** система проходит через эту схему.

## Статистический машинный перевод (SMT) {#smt}

### Основные принципы

SMT основан на **вероятностном моделировании**. Система выбирает перевод, который имеет наибольшую вероятность быть правильным на основе обучающих данных.

**Ключевая идея:** Дан исходный текст *e*, найти целевой текст *f*, который максимизирует:

```
P(f | e) = P(e | f) × P(f) / P(e)
```

Где:
- **P(f | e)** — вероятность целевого текста при заданном исходном (то, что мы ищем)
- **P(e | f)** — модель перевода (translation model)
- **P(f)** — языковая модель (language model)

### Ключевые компоненты

#### 1. Модель перевода (Translation Model)

Отвечает за P(e | f). Основана на **парных фразах** (phrase pairs):

**Пример:**
```
Исходный язык: "Доброе утро"
Целевой язык: "Good morning"

Вероятность этого сопоставления: 0.87
```

Модель формируется через:
- **Выравнивание слов (word alignment):** Определение соответствия между словами в параллельных предложениях
- **Извлечение фраз (phrase extraction):** Выделение фрагментов, часто встречающихся вместе

**Алгоритм выравнивания (модель IBM):**

Используются итеративные методы (EM-алгоритм) для нахождения соответствий:

```
Итерация 1: "Доброе" → Good, "утро" → morning
Итерация 2 (уточнение): вероятность увеличивается на основе контекста
```

#### 2. Языковая модель (Language Model)

Отвечает за P(f). Гарантирует, что целевой текст звучит естественно.

**N-граммная языковая модель:**

```
P("Good morning") = P("Good") × P("morning" | "Good")
```

Вероятность берется из частоты в корпусе целевого языка:

```
P("morning" | "Good") = Count("Good morning") / Count("Good")
```

**Пример с троп-граммами:**

```
Текст: "Доброе утро, солнце!"
P("Доброе утро солнце") = P("Доброе") × P("утро" | "Доброе") 
                          × P("солнце" | "Доброе", "утро")
                          = 0.0003 × 0.42 × 0.15 = 0.0000189
```

### Достоинства SMT

✅ **Прозрачность:** Можно отследить, какие фразы использовались при переводе
✅ **Эффективность обучения:** Требует меньше параллельных данных, чем NMT
✅ **Стабильность:** Предсказуемое поведение на знакомых данных
✅ **Модульность:** Легко обновить отдельно модель перевода или языковую модель

### Недостатки SMT

❌ **Хрупкость:** Ошибки выравнивания слов передаются далее по цепочке
❌ **Независимость фраз:** Система не учитывает широкий контекст (влияние дальних слов)
❌ **Дискретность перевода:** Система не может "придумать" перевод фразы, которой не было в обучающих данных
❌ **Неестественность:** Часто производит технически верные, но неловкие переводы

**Пример проблемы:**

```
Исходный текст: "Мне понравилась книга, потому что она была интересной."

SMT может дать:
"I liked the book, because it was interesting." ✓ Верно, но скучно

А более естественный перевод:
"I enjoyed the book for its compelling narrative." (теряется в моделе SMT)
```

## Нейронный машинный перевод (NMT) {#nmt}

### Основные принципы

NMT использует **нейронные сети** для прямого моделирования отображения "исходный язык → целевой язык" без явного разделения на компоненты.

**Ключевая идея:** 

```
Исходное предложение → Кодировщик (Encoder) → Скрытое представление 
                                                         ↓
Целевое предложение ← Декодировщик (Decoder) ← Внимание (Attention)
```

### Архитектуры NMT

#### 1. Архитектура Encoder-Decoder с RNN

**Кодировщик (Encoder):** LSTM или GRU обрабатывают исходный текст и создают вектор контекста:

```
Слово "Доброе"  →┐
Слово "утро"    →├→ LSTM → Вектор контекста (контекстный вектор)
Слово "<EOS>"   →┘
```

**Декодировщик (Decoder):** Другой LSTM генерирует целевой текст слово за словом:

```
Вектор контекста
       ↓
      LSTM → "Good" (вероятность 0.94)
       ↓
      LSTM → "morning" (вероятность 0.89)
       ↓
      LSTM → "<EOS>" (конец предложения)
```

**Проблема:** Весь исходный текст "сжимается" в один вектор, теряется информация о дальних словах.

#### 2. Механизм внимания (Attention Mechanism)

**Решение:** Декодировщик может "смотреть" на все входные слова, но сосредоточиться на наиболее релевантных.

**Как это работает:**

```
Декодировщик генерирует "morning":
  ↓
  Вычисляет релевантность к каждому исходному слову:
  - "Доброе": 0.72 (высокая — очень релевантно!)
  - "утро": 0.95 (очень высокая — это то, что переводим!)
  - "<EOS>": 0.03 (низкая)
  ↓
  Создает взвешенный вектор из исходных представлений
  ↓
  Генерирует "morning" с более высокой точностью
```

**Математически:**

```
Внимание (query, ключи, значения) = softmax(query · ключи) · значения
```

#### 3. Трансформер (Transformer Architecture)

Архитектура, полностью основанная на механизме внимания (внимание = всё).

**Достоинства:**
- Полностью параллельная обработка (быстрее RNN)
- Может ловить дальние зависимости лучше
- Масштабируется на большие модели (GPT, BERT)

**Схема:**

```
Исходный текст
     ↓
[Встраивание + позиционное кодирование]
     ↓
[Многоголовое внимание] × N слоев
     ↓
[Прямая нейросеть] × N слоев
     ↓
[Энкодер готов]
     ↓
[Декодер с само-вниманием и кросс-вниманием] × N слоев
     ↓
Целевой текст (генерируется слово за словом с внниманием к энкодеру)
```

### Преимущества NMT перед SMT

✅ **Плавность перевода:** Система учится естественному стилю целевого языка
✅ **Контекстность:** Механизм внимания позволяет учитывать взаимодействие удаленных слов

**Пример:**
```
Исходный: "Мне понравилась книга, потому что она была очень интересной и захватывающей."

NMT (с вниманием):
"I enjoyed the book because it was very interesting and engaging."
Система видит всю фразу и выбирает "enjoyed" вместо "liked", видя "very... engaging"

SMT:
"I liked the book because she was very interesting and capturing."
Ошибка: "she" вместо "it" (система потеряла гендер в выравнивании)
```

✅ **Открытость:** Может генерировать новые словосочетания, которых не было в обучении
✅ **Единая модель:** Все компоненты обучаются совместно (end-to-end)
✅ **Масштабируемость:** Системы улучшаются с большим объемом данных

### Недостатки NMT

❌ **Требует больших данных:** Нужны миллионы параллельных предложений для хорошего качества
❌ **Вычислительные затраты:** Требует GPU/TPU для практического применения
❌ **Черный ящик:** Сложнее объяснить, почему система выбрала конкретный перевод
❌ **Редкие слова:** Может "путаться" с редко встречаемыми словами (OOV — Out Of Vocabulary)
❌ **Галлюцинации:** Иногда генерирует слова, которые не соответствуют исходному тексту

**Пример проблемы:**

```
Исходный текст: "Компания инвестирует в КРПП технологии."

NMT может дать:
"The company invests in KRPP technologies." ✓ (сохранил аббревиатуру)

Или (реже):
"The company invests in creative processing technologies." ✗ (галлюцинация: "КРПП" 
переведено неправильно, система "угадала")

SMT:
"The company invests in KRPP technologies." ✓ (копирует, как в обучении)
```

## Сравнение SMT и NMT {#сравнение}

### Таблица различий

| Характеристика | SMT | NMT |
|---|---|---|
| **Основной подход** | Вероятностный (P(f\|e)) | Нейронная сеть (end-to-end) |
| **Минимум параллельных данных** | 1–5 млн пар предложений | 10–50 млн пар предложений |
| **Скорость обучения** | Часы–дни | Дни–недели |
| **Требования к железу** | CPU хватает | Необходим GPU/TPU |
| **Интерпретируемость** | Высокая (видны фразы) | Низкая (черный ящик) |
| **Качество перевода** | Хорошее | Отличное |
| **Плавность речи** | Средняя | Высокая |
| **Контекстность** | Локальная (фраза) | Глобальная (предложение+) |
| **Обработка редких слов** | Копирует как есть | Может ошибиться |
| **Легкость обновления** | Легко (переобучить часть) | Сложно (полное переобучение) |
| **Поддержка множественных пар языков** | Отдельная система на пару | Одна система на множество пар |

### Когда использовать какую систему?

**Выбирайте SMT если:**
- У вас ограниченные вычислительные ресурсы
- Есть меньше 5 млн параллельных пар предложений
- Нужна интерпретируемость и контроль системы
- Требуется быстро обновлять модель

**Выбирайте NMT если:**
- У вас есть доступ к GPU/облачным сервисам
- Есть достаточное количество параллельных данных
- Приоритет — качество перевода и естественность
- Нужно переводить сразу много языковых пар

## Влияние языковых пар {#языковые-пары}

### Основные различия между языками, влияющие на перевод

#### 1. Порядок слов

Разные языки имеют разные стандартные порядки слов в предложении.

**SVO (Subject-Verb-Object): Английский, Русский**
```
"Мальчик читает книгу." (Подлежащее-Глагол-Дополнение)
"The boy reads a book."
```

**SOV (Subject-Object-Verb): Японский, Корейский, Турецкий**
```
Японский: "少年は本を読む。" (Мальчик-Книга-Читать)
```

**VSO (Verb-Subject-Object): Арабский, Ирландский**
```
Арабский: "قرأ الصبي الكتاب" (Читать-Мальчик-Книга)
```

**Влияние на перевод:**
- SMT: Хорошо справляется благодаря фразовым парам, но может поломать структуру на длинных предложениях
- NMT: Механизм внимания помогает переупорядочить слова, но требует больше примеров переупорядочивания

**Пример сложности:**
```
Русский (SVO): "Интересная книга лежит на столе."
Немецкий (V2 — глагол на втором месте в главном предложении):
"Das interessante Buch liegt auf dem Tisch."

Но в придаточном предложении:
Русский: "Я знаю, что интересная книга лежит на столе."
Немецкий: "Ich weiß, dass das interessante Buch auf dem Tisch liegt."
           (Глагол в конце в придаточном!)
```

#### 2. Морфология (падежи, роды, числа)

**Языки с развитой морфологией (Русский, Польский, Финский):**

Слово может изменяться, кодируя грамматическую информацию в окончании.

```
"Кошка" (именительный, женский род)
"Кошки" (родительный, женский род)
"Кошке" (дательный, женский род)
"Кошку" (винительный, женский род)
...
```

**Языки с бедной морфологией (Английский):**

Информация кодируется порядком и служебными словами.

```
"cat" (именительный)
"cat's" (притяжательный — добавляется 's)
"cats" (множественное число)
```

**Влияние на перевод:**

```
Русский → Английский:
"Мне нравится эта красивая большая кошка."
SMT: Может выбрать неправильное окончание, если не видел точную фразу
NMT: Лучше, но требует примеров для редких форм

Английский → Русский:
"The big cat likes the small dog."
Проблема: Английское "the" не кодирует падеж, род, число
         Русской модели нужно "угадать" правильные окончания
```

#### 3. Наличие грамматического рода

**Языки с родом (Русский, Французский, Испанский, Немецкий):**

Каждое существительное имеет род (женский, мужской, средний), и все зависимые слова согласуются.

```
Русский: "Чёрный кот" (мужской род)
         "Чёрная кошка" (женский род)
         "Чёрное полотно" (средний род)
```

**Языки без грамматического рода (Английский, Турецкий, Финский):**

Нет грамматического рода, согласование не требуется.

```
Английский: "The black cat" (одинаково для всех родов)
```

**Влияние на перевод:**

```
Английский → Русский (сложный случай):
"The doctor gave the patient their medicine."

Проблема: Английское "their" не показывает род врача.
Может быть:
- "Врач дал пациенту его лекарство." (врач — мужчина)
- "Врач дала пациенту её лекарство." (врач — женщина)
- "Врач дал/дала пациенту их лекарство." (множественное, если несколько врачей)

NMT может выбрать неправильный род, особенно если контекст неясен.
SMT: Часто копирует род из обучающих данных, может быть неправильно.
```

#### 4. Развитость и типичность языковой пары

**"Богатые" пары (много параллельных данных):**
- Английский ↔ Немецкий: 10–100 млн пар
- Английский ↔ Французский: 50–200 млн пар
- Английский ↔ Русский: 10–50 млн пар
- Английский ↔ Китайский: 5–20 млн пар

**"Бедные" пары (мало параллельных данных):**
- Эстонский ↔ Латышский: <100k пар
- Албанский ↔ Грузинский: <50k пар
- Мордовский ↔ Марийский: практически нет

**Влияние:**

```
Английский → Немецкий (богатая пара):
SMT: Очень хорошо (много фраз в базе)
NMT: Отличное качество (миллионы примеров)

Английский → Мордовский (бедная пара):
SMT: Приемлемо (если есть система)
NMT: Плохо или невозможно (мало обучающих данных)

Решение для бедных пар: Трансферное обучение (transfer learning)
- Предварительно обучить на богатой паре (Английский → Русский)
- Затем адаптировать к бедной паре (Английский → Мордовский)
- Результат: Улучшение благодаря общим принципам кодирования
```

### Примеры успешных и проблемных пар

#### Успешные пары (Европейские языки)

```
Пара: Английский ↔ Немецкий
Сходства:
- Оба германские языки (схожая грамматика)
- Похожая лексика (~ 26% совпадает)
- Оба SVO (в основном)
- Оба имеют артикли

Результат: BLEU ≈ 38–42 для NMT (очень хорошо)

Пример:
"The quick brown fox jumps over the lazy dog."
"Der schnelle braune Fuchs springt über den faulen Hund."
NMT почти идеален
```

#### Проблемные пары (Удаленные языки)

```
Пара: Английский ↔ Финский
Различия:
- Английский: германский, финский: угорский (разные семьи)
- Финский имеет 15 падежей, английский — нет
- Финский: агглютинативный (слова строятся добавлением морфем)
  Пример: "talossanikinko" = "talo+ssa+ni+kin+ko" 
          (в твоих ли домах)
  Английское: "in-your-houses" (требует 3 слова!)

Результат: BLEU ≈ 20–25 для NMT (удовлетворительно)

Проблема:
"I live in a house near the forest." 
Перевод может быть неправильным, потому что падежная система финского 
не соответствует английской предложно-предметной системе.
```

```
Пара: Английский ↔ Китайский
Различия:
- Английский: алфавитный, китайский: иероглифический
- Английский: флективный (изменяются окончания), китайский: изолирующий 
  (слова не меняются, порядок кодирует грамматику)
- Грамматический род есть в английском (some), в китайском отсутствует
- Тон в китайском кодирует смысл

Результат: BLEU ≈ 24–30 для NMT (хорошо, но медленнее)

Проблема тонов:
"ma" (低声, первый тон) = мать
"má" (阳声, второй тон) = пенька
"mǎ" (上声, третий тон) = лошадь
"mà" (去声, четвёртый тон) = ругать

Английская система не кодирует тоны → модель может перепутать
```

#### Критичная проблема: Направление перевода имеет значение

```
Пара A: Немецкий → Английский (BLEU ≈ 35)
"Der Hund ist groß." → "The dog is big." ✓

Пара B: Английский → Немецкий (BLEU ≈ 30)
"The dog is big." → "Der große Hund" (может потеряться слово "is")

Почему разница?
- Немецкий → Английский: Упрощение (падежи → порядок слов)
- Английский → Немецкий: Усложнение (нужно добавить информацию о падежах)

Правило: Перевод на сложный язык (по морфологии) обычно менее качественный.
```

## Будущее машинного перевода {#будущее}

### Перспективные направления

#### 1. Многоязычные модели

**Современное состояние:**
```
Раньше: 200+ отдельных моделей NMT (одна пара = одна модель)
Сейчас: Одна модель → 100+ языков
```

**Примеры:**
- Google's mT5 (75 языков в одной модели)
- Meta's M2M-100 (100 языков, парный перевод без английского посредника)
- WhatsApp Translator (на базе мультиязычных моделей)

**Преимущества:**
✅ Лучше разделяет знания между языками
✅ Улучшает перевод для бедных языковых пар
✅ Экономнее в вычислениях (одна модель вместо 200)

**Пример эффекта передачи:**
```
Обучить модель на 1 млн пар Английский ↔ Испанский
+ 1 млн пар Английский ↔ Французский
↓
Результат: Улучшение Испанский ↔ Французский даже без прямого обучения!
```

#### 2. Адаптивный перевод (Domain Adaptation)

**Проблема:** NMT хорошо работает на новостях, но плохо на медицинских текстах.

**Решение:** Быстрая адаптация модели к конкретной области.

```
Базовая модель (обучена на новостях): BLEU ≈ 30 на медицинских текстах

Адаптация:
1. Взять готовую модель
2. Дообучить на 10k медицинских пар
3. Результат: BLEU ≈ 40+ ✓ (улучшение на 33%)

Метод: Fine-tuning (дообучение с низкой скоростью обучения)
```

#### 3. Интерпретируемый перевод

**Текущая проблема:** Черный ящик NMT не объясняет решения.

**Направления исследований:**

```
Направление 1: Визуализация внимания
- Показать, какое исходное слово "смотрела" система при генерации целевого
- Помогает отладке и пониманию

Пример:
Исходный: "Мне понравилась книга"
Целевой:  "I    liked       the   book"
                ↓ из "понравилась"
                ↓ из "Мне"
                      ↓ из "книга"
                           ↓ из "книга"

Направление 2: Промежуточные представления
- Объяснить, что "кодирует" система на скрытых слоях
- Использовать для диагностики ошибок

Направление 3: Правило-нейронные гибриды
- Комбинировать правила лингвистов с нейронными сетями
- Высокое качество + некоторая интерпретируемость
```

#### 4. Минимально-ресурсный перевод (Low-Resource Translation)

**Проблема:** 7000+ языков в мире, для большинства нет параллельных данных.

**Решения:**

```
1. Нулевой выстрел (Zero-Shot):
   Модель: 50 языков → NMT на базе мультиязычного кодирования
   Просит перевести пару, на которой не обучалась
   Результат: BLEU ≈ 10–20 (скромно, но лучше, чем ничего!)

2. Обратный перевод (Back-Translation):
   - Есть параллельные данные Английский ↔ Испанский
   - Нужно улучшить Испанский → Английский
   
   Процесс:
   a) Обучить Английский → Испанский (есть данные)
   b) Использовать эту модель для создания "синтетических" пар:
      - Взять неразмеченные испанские тексты
      - Перевести их на английский (Испанский → Английский)
      - Получить пары (синтетический English, оригинальный Spanish)
   c) Дообучить Испанский → Английский на этих синтетических парах
   
   Результат: Улучшение на 2–5 пункта BLEU!

3. Трансфертное обучение (Transfer Learning):
   - Обучить на богатой паре (Англ. ↔ Нем.)
   - Адаптировать к бедной паре (Англ. ↔ Монгольский)
   - Модель переносит общие знания
```

#### 5. Контекстуальный перевод

**Текущее состояние:** Переводит предложение независимо.

**Будущее:** Использует весь документ как контекст.

```
Документ: "Мне нужна свежая рыба. Рыба должна быть качественной."

Предложение 1 (изолированно):
"I need fresh fish."

Предложение 1 (с контекстом):
"I need fresh fish." (понимает, что это рыба в кулинарном смысле)

vs

Предложение 2 (с контекстом из первого):
"The fish must be of high quality." (не "The fish must be fresh" — 
избегает повтора слова "fresh" из предыдущего предложения!)

Метод: Document-level NMT с памятью контекста
- Сохранять представления предыдущих предложений
- Использовать их при переводе нового предложения
```

#### 6. Мультимодальный перевод (Multimodal Translation)

**Идея:** Используем текст + изображение + аудио для лучшего перевода.

```
Пример задачи "Перевести подпись к фото":

Исходный: "В музее много интересных картин."
Изображение: (фотография музея с картинами)

Система:
1. Кодирует текст: "много интересных картин"
2. Кодирует изображение: выявляет, что это действительно картины в музее
3. Использует визуальную информацию для выбора лучшего слова (paintings, artworks, etc.)

Результат: "The museum features many fascinating artworks." 
(лучше, чем если бы видела только текст)

Приложения:
- YouTube субтитры (видео + аудио + текст)
- Переводы туристических фото
- Медицинские описания (снимок + текст отчета)
```

### Упражнение 2: Языковая модель и перплексия

**Уровень: Средний**

Вычислите перплексия для двух гипотез перевода и определите, какая более вероятна согласно языковой модели.

```
Предложение на русском языке для оценки языковой модели:
"Кошка спит на диване"

Нужно вычислить вероятность.

Статистика из корпуса (частоты из миллионов слов):
P("Кошка") = 0.003 (3 из 1000)
P("спит" | "Кошка") = 0.45 (в 45% случаев, когда есть "Кошка", идет "спит")
P("на" | "спит") = 0.12
P("диване" | "на") = 0.08

Формула перплексии для предложения:
Perplexity = (1 / (P(w1) * P(w2|w1) * ... * P(wn|wn-1)))^(1/n)

Или через логарифмы (проще):
log-probability = log(P(w1)) + log(P(w2|w1)) + ... + log(P(wn|wn-1))
Perplexity = 2^(-log-probability / n)
```

**Задание:**

Вычислите перплексию для:

**Гипотеза 1:** "Кошка спит на диване"

```
P = 0.003 × 0.45 × 0.12 × 0.08 = 0.0000129
Perplexity = (1/0.0000129)^(1/4) ≈ ?
```

**Гипотеза 2:** "Диван спит кошка на"

```
P("диван") = 0.002
P("спит" | "диван") = 0.01 (маловероятно, диваны не спят!)
P("кошка" | "спит") = 0.3
P("на" | "кошка") = 0.2

P = 0.002 × 0.01 × 0.3 × 0.2 = 0.0000012
Perplexity = (1/0.0000012)^(1/4) ≈ ?
```

**Решение:**

```python
import math

# Гипотеза 1
p1_list = [0.003, 0.45, 0.12, 0.08]
log_p1 = sum(math.log2(p) for p in p1_list)
perp1 = 2 ** (-log_p1 / len(p1_list))
print(f"Гипотеза 1: Perplexity ≈ {perp1:.2f}")  # ≈ 40.7

# Гипотеза 2
p2_list = [0.002, 0.01, 0.3, 0.2]
log_p2 = sum(math.log2(p) for p in p2_list)
perp2 = 2 ** (-log_p2 / len(p2_list))
print(f"Гипотеза 2: Perplexity ≈ {perp2:.2f}")  # ≈ 158.1

# Интерпретация:
# Гипотеза 1 имеет МЕНЬШУЮ перплексию (40.7 < 158.1)
# → Она БОЛЕЕ вероятна согласно языковой модели ✓
# → Система выбирает гипотезу 1 как лучший перевод

print("\n✓ Гипотеза 1 предпочтительна (меньшая перплексия)")
```

```
      - Имена собственные
      - Использовать constrained decoding
      - Результат: +1–3 BLEU на специализированных текстах

   в) Ансамбли:
      - Обучить 3–5 модели NMT с разными случайными инициализациями
      - Объединить предсказания (голосование)
      - Результат: +0.5–1 BLEU

6. ФИНАЛЬНАЯ АРХИТЕКТУРА

   ```
   Входной текст (английский)
           ↓
   [Предварительная обработка: нормализация, терминология]
           ↓
   [NMT модель с трансфертным обучением + Back-Translation]
           ↓
   [Постобработка: исправление ошибок морфологии]
           ↓
   Выходной текст (эстонский)
   ```

ОЖИДАЕМЫЙ РЕЗУЛЬТАТ: BLEU ≈ 35–40, что превышает требование ✓
```

## Глоссарий {#глоссарий}

### Основные термины

**Back-Translation (Обратный перевод)**
Техника создания синтетических параллельных данных путем перевода одноязычных текстов в противоположном направлении.

**BLEU (Bilingual Evaluation Understudy)**
Метрика автоматической оценки качества перевода, основанная на совпадении n-грамм с эталонными переводами (0–100, выше лучше).

**Decoder (Декодировщик)**
Часть NMT архитектуры, которая генерирует целевой текст на основе представления исходного текста.

**Encoder (Кодировщик)**
Часть NMT архитектуры, которая обрабатывает исходный текст и создает его внутреннее представление.

**End-to-End (Сквозное обучение)**
Обучение системы без явного разделения на компоненты, оптимизирует итоговый результат.

**Fine-tuning (Дообучение)**
Адаптация предварительно обученной модели к новой задаче с низкой скоростью обучения.

**GRU (Gated Recurrent Unit)**
Тип рекуррентной нейронной сети, упрощенная версия LSTM.

**Language Model (Языковая модель)**
Модель, кодирующая вероятность последовательности слов в языке: P(w₁, w₂, ..., wₙ).

**LSTM (Long Short-Term Memory)**
Тип рекуррентной нейронной сети, способный учить дальние зависимости.

**Machine Translation (Машинный перевод)**
Автоматическое преобразование текста с одного языка на другой компьютерными алгоритмами.

**Morphology (Морфология)**
Система внутреннего строения слов языка (падежи, роды, спряжения и т.д.).

**NMT (Neural Machine Translation)**
Системы машинного перевода на основе нейронных сетей.

**N-gram**
Последовательность из N слов; например, "машинный перевод" — это bigram (2-gram).

**OOV (Out of Vocabulary)**
Слово, которого нет в словаре системы (для NMT) или в обучающих данных (для SMT).

**Perplexity (Перплексия)**
Метрика оценки качества языковой модели; показывает, насколько "удивлена" модель при виде текста (ниже лучше).

**Phrase-Based SMT**
Система статистического перевода, использующая парные фразы как основные единицы перевода.

**RNN (Recurrent Neural Network)**
Рекуррентная нейронная сеть, обрабатывающая последовательности данных.

**SMT (Statistical Machine Translation)**
Системы машинного перевода на основе вероятностных моделей.

**Source Language (Исходный язык)**
Язык, с которого переводят.

**Target Language (Целевой язык)**
Язык, на который переводят.

**Transfer Learning (Трансфертное обучение)**
Использование знаний, полученных при обучении на одной задаче, для улучшения производительности на другой задаче.

**Transformer**
Архитектура нейронной сети на основе механизма внимания, без рекуррентности; основа современных NMT систем.

**Translation Model (Модель перевода)**
В SMT: модель, кодирующая соответствие между единицами исходного и целевого языков: P(e|f).

**Word Alignment (Выравнивание слов)**
Определение соответствия между словами в параллельных предложениях на разных языках.
