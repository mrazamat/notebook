# Теория автоматической разметки частей речи (POS tagging) и работа с неважными словами

## Введение в POS-теггинг

### Определение POS-теггинга

**POS-теггинг** (Part-of-Speech Tagging) — это процесс автоматического определения грамматической части речи для каждого слова в тексте. Каждому слову присваивается один из стандартных тегов (например, существительное, глагол, прилагательное).

Пример разметки:
```
The       -> DET (Determiner)
quick     -> ADJ (Adjective)
brown     -> ADJ (Adjective)
fox       -> NOUN (Noun)
jumps     -> VERB (Verb)
over      -> ADP (Adposition)
the       -> DET (Determiner)
lazy      -> ADJ (Adjective)
dog       -> NOUN (Noun)
```

### Зачем нужен POS-теггинг в NLP

POS-теггинг является важной задачей в обработке естественного языка, так как он:

- **Улучшает понимание синтаксиса:** Помогает понять структуру предложения
- **Поддерживает другие NLP-задачи:** Необходим для парсинга, извлечения сущностей (NER), анализа тональности, машинного перевода
- **Повышает качество обработки текста:** Позволяет различать омонимы (слова с одинаковым написанием, но разными значениями)
- **Используется в лингвистическом анализе:** Помогает выделять закономерности в тексте

### Основные теги POS

Универсальная система тегов (Universal Dependencies):

| Тег | Описание | Примеры (English/Русский) |
|-----|----------|--------------------------|
| NOUN | Существительное | cat, dog / кот, собака |
| VERB | Глагол | run, write / бежать, писать |
| ADJ | Прилагательное | big, red / большой, красный |
| ADV | Наречие | quickly, slowly / быстро, медленно |
| DET | Артикль, определитель | the, a / этот, тот |
| ADP | Предлог | in, on / в, на |
| CCONJ | Сочинительный союз | and, or / и, или |
| SCONJ | Подчинительный союз | because, if / потому что, если |
| NUM | Числительное | one, 42 / один, 42 |
| PRON | Местоимение | he, she, it / он, она, оно |

## Основные подходы к POS-теггингу

### 1. Подход на основе правил (Rule-Based Approach)

**Основная идея:** Используются явные лингвистические правила, разработанные человеком.

**Преимущества:**
- Прозрачность и интерпретируемость
- Контролируемое поведение
- Полезен для языков с ограниченным количеством обучающих данных

**Недостатки:**
- Трудоёмко разрабатывать правила для всех случаев
- Сложно учитывать все исключения языка
- Низкая точность на сложных текстах

**Пример правила:**
```
IF word ends with "-ing" AND previous_tag = VERB
THEN assign tag = VERB
```

### 2. Статистические подходы

#### Скрытые Марковские модели (HMM)

**Основная идея:** Предполагается, что текст генерируется процессом, в котором каждое слово зависит только от его части речи, а части речи образуют цепь Маркова.

**Ключевые компоненты:**

1. **Вероятность перехода между тегами:** P(tag_i | tag_{i-1})
   - Вероятность того, что после тега tag_{i-1} следует тег tag_i

2. **Вероятность эмиссии:** P(word_i | tag_i)
   - Вероятность того, что слово word_i имеет тег tag_i

**Базовая формула (упрощённо):**

Для последовательности слов w_1, w_2, ..., w_n найти последовательность тегов t_1, t_2, ..., t_n, максимизирующую:

```
P(t_1, t_2, ..., t_n | w_1, w_2, ..., w_n) ∝ P(t_1) × ∏ P(t_i | t_{i-1}) × P(w_i | t_i)
```

**Алгоритм декодирования:** Используется алгоритм Витерби для эффективного поиска наиболее вероятной последовательности тегов.

**Точность:** Обычно 85-92% в зависимости от языка

#### Условные случайные поля (CRF)

**Основная идея:** CRF — это вероятностный графический модель, которая предсказывает последовательность тегов, учитывая наблюдаемые признаки слова и контекст.

**Отличия от HMM:**
- Не требует предположения о независимости (как в HMM)
- Может использовать наблюдаемые признаки (длина слова, суффикс, наличие чисел и т.д.)
- Более гибкая модель для учёта особенностей языка

**Особенности:**
- Работает с линейными цепочками (для последовательной разметки)
- Использует функции признаков (feature functions)
- Обучается методом максимального правдоподобия

**Точность:** Обычно 92-96%

#### Нейросетевые подходы

**RNN/LSTM для POS-теггинга:**

- **Входные данные:** Word embeddings (векторные представления слов)
- **Процесс:** LSTM-слой обрабатывает последовательность слов, захватывая контекст
- **Выходные данные:** Для каждого слова выходит распределение вероятностей по тегам
- **Преимущества:** Автоматически выучивает признаки, лучше обрабатывает контекст

**Современный подход: Transformer-based модели (BERT, RoBERTa):**

- Используют механизм самовнимания (self-attention)
- Достигают точности 97-99%
- Требуют больше вычислительных ресурсов, но обеспечивают SOTA-результаты

**Общая точность нейросетевых моделей:** 96-99%

## Работа с неважными словами (Stop Words)

### Что такое стоп-слова

**Стоп-слова** — это часто встречающиеся слова, которые содержат мало или совсем не содержат смысловой информации в контексте обработки текста.

**Примеры для английского языка:**
- Артикли: `the`, `a`, `an`
- Предлоги: `in`, `on`, `at`, `by`, `of`, `to`, `from`
- Союзы: `and`, `or`, `but`
- Местоимения: `he`, `she`, `it`, `we`, `they`
- Вспомогательные глаголы: `is`, `are`, `was`, `were`, `be`

**Примеры для русского языка:**
- Артикли/частицы: нет специальных артиклей
- Предлоги: `в`, `на`, `из`, `по`, `к`, `от`, `с`
- Союзы: `и`, `или`, `но`, `а`
- Местоимения: `я`, `ты`, `он`, `она`, `оно`, `мы`, `они`
- Вспомогательные глаголы: `быть`, `есть`

### Почему удаляют стоп-слова

**Причины удаления:**

1. **Снижение размера данных:** Стоп-слова составляют 30-50% текста, их удаление значительно сокращает объём данных

2. **Улучшение релевантности:** При поиске информации стоп-слова не указывают на тему документа

3. **Повышение производительности:** Меньше данных для обработки = быстрее выполнение

4. **Улучшение качества моделей:** В некоторых задачах (например, при классификации текста) удаление стоп-слов улучшает качество

**Когда НЕ следует удалять стоп-слова:**

- Анализ тональности (слово "not" кардинально меняет смысл)
- Машинный перевод (грамматическая структура важна)
- Синтаксический парсинг
- Задачи, требующие полного контекста

### Методы удаления стоп-слов

#### Метод 1: Использование готовых списков

Наиболее простой и распространённый подход.

**Библиотеки и ресурсы:**

| Источник | Язык | Размер списка |
|----------|------|---------------|
| NLTK stopwords | 20+ языков | 100-500 слов |
| spaCy stopwords | 10+ языков | 200-400 слов |
| sklearn TfidfVectorizer | Встроенные списки | 100-300 слов |
| Стопслова на GitHub | Кастомные | Варьируется |

#### Метод 2: Частотно-основанный подход

Удаляются N наиболее частых слов из текста. Основан на предположении, что очень часто встречающиеся слова — это стоп-слова.

**Процесс:**
1. Подсчитать частоту каждого слова
2. Отсортировать по частоте в убывающем порядке
3. Удалить top-N слов

#### Метод 3: TF-IDF фильтрация

Удаляются слова с низким TF-IDF значением. Слова с высоким TF-IDF более информативны.

**Формула TF-IDF:**
```
TF-IDF(word) = TF(word) × IDF(word)

где:
TF(word) = (количество word в документе) / (всего слов в документе)
IDF(word) = log(всего документов / количество документов с word)
```

#### Метод 4: Лингвистические правила

На основе POS-тегов удаляются определённые части речи (например, все предлоги, артикли).

**Пример:**
```
POS_tags_to_remove = {'DET', 'ADP', 'CCONJ', 'AUX'}
```

## Примеры реализации на Python

### Пример 1: POS-теггинг с использованием NLTK

```python
import nltk
from nltk import word_tokenize, pos_tag
from nltk.corpus import stopwords

# Скачивание необходимых ресурсов (выполнить один раз)
# nltk.download('punkt')
# nltk.download('averaged_perceptron_tagger')
# nltk.download('stopwords')

# Исходный текст
text = "The quick brown fox jumps over the lazy dog. It is a beautiful day."

# Токенизация
tokens = word_tokenize(text)
print("Tokens:", tokens)

# POS-теггинг
pos_tags = pos_tag(tokens)
print("\nPOS Tags:")
for word, tag in pos_tags:
    print(f"  {word:12} -> {tag}")

# Удаление стоп-слов
stop_words = set(stopwords.words('english'))
filtered_tokens = [word for word in tokens if word.lower() not in stop_words]
print("\nFiltered tokens (without stop words):")
print(filtered_tokens)

# Комбинированная обработка: POS-теггинг + удаление стоп-слов
meaningful_tokens = [(word, tag) for word, tag in pos_tags 
                     if word.lower() not in stop_words and tag not in ['POS', 'IN']]
print("\nMeaningful tokens (meaningful POS tags only):")
for word, tag in meaningful_tokens:
    print(f"  {word:12} -> {tag}")
```

**Ожидаемый вывод:**

### Пример 2: POS-теггинг с использованием spaCy

```python
import spacy

# Загрузка англоязычной модели

text = "The company developed a new technology for processing language."

print("\nMeaningful tokens (NOUN, VERB, ADJ, ADV):")
for word, pos in meaningful_tokens:
```

### Пример 4: Использование признаков POS для выполнения задач

```python
import nltk
from nltk import word_tokenize, pos_tag

text = "Natural language processing enables computers to understand human language."

tokens = word_tokenize(text)
pos_tags = pos_tag(tokens)

# Извлечение существительных
nouns = [word for word, tag in pos_tags if tag in ['NN', 'NNS']]
print("Nouns:", nouns)

# Извлечение глаголов
verbs = [word for word, tag in pos_tags if tag in ['VB', 'VBD', 'VBG', 'VBN', 'VBP', 'VBZ']]
print("Verbs:", verbs)

# Извлечение прилагательных
adjectives = [word for word, tag in pos_tags if tag in ['JJ', 'JJR', 'JJS']]
print("Adjectives:", adjectives)

# Поиск именных групп (noun phrases)
noun_phrases = []
for i in range(len(pos_tags) - 1):
    word, tag = pos_tags[i]
    if tag in ['JJ']:  # Если текущее слово — прилагательное
        next_word, next_tag = pos_tags[i + 1]
        if next_tag in ['NN', 'NNS']:  # И следующее — существительное
            noun_phrases.append(f"{word} {next_word}")

print("Noun Phrases:", noun_phrases)
```

**Ожидаемый вывод:**
```
Nouns: ['language', 'processing', 'computers', 'language']
Verbs: ['enables']
Adjectives: ['Natural']
Noun Phrases: ['Natural language']
```

## Справочник терминов и формул

### Ключевые термины

| Термин | Определение |
|--------|------------|
| **Token** | Отдельная единица текста (слово, пунктуация) |
| **POS tag** | Метка части речи для токена |
| **Tokenization** | Разделение текста на токены |
| **Lemma** | Базовая форма слова (например, "running" -> "run") |
| **Morphology** | Наука о структуре слов |
| **Syntax** | Правила построения предложений |
| **Frequency** | Количество появлений слова в тексте |
| **Context** | Слова, окружающие целевое слово |

### Основные формулы

**Вероятность перехода в HMM:**
```
P(tag_i | tag_{i-1}) = Count(tag_{i-1} -> tag_i) / Count(tag_{i-1})
```

**Вероятность эмиссии в HMM:**
```
P(word | tag) = Count(word with tag) / Count(tag)
```

**TF (Term Frequency):**
```
TF(word) = (количество word в документе) / (общее количество слов в документе)
```

**IDF (Inverse Document Frequency):**
```
IDF(word) = log(N / n_word)

где:
N = общее количество документов
n_word = количество документов, содержащих word
```

**Точность (Accuracy):**
```
Accuracy = (количество правильных тегов) / (всего тегов) × 100%
```

## Ключевые выводы

1. **POS-теггинг** — это фундаментальная задача NLP, которая помогает понять структуру и значение текста

2. **Основные подходы** различаются по сложности и точности: от простых правил до нейросетевых моделей

3. **HMM** — классический подход, обеспечивающий хороший баланс между простотой и точностью

4. **CRF** и нейросетевые модели обеспечивают более высокую точность благодаря лучшему моделированию контекста

5. **Стоп-слова** удаляют в большинстве текстовых задач для сокращения размера данных и улучшения качества

6. **Выбор метода** удаления стоп-слов зависит от конкретной задачи и типа текста

7. **Инструменты** (NLTK, spaCy, sklearn) обеспечивают готовые реализации POS-теггинга и удаления стоп-слов
