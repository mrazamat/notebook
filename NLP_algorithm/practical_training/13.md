# Векторы слова: Теория семантического пространства и векторное выражение

## Введение в концепцию векторов слова

В традиционном подходе к обработке естественного языка слова представлялись как отдельные, изолированные единицы без информации об их значении и отношениях. **Векторное представление слов (word embeddings)** решает эту проблему, кодируя каждое слово как точку в многомерном пространстве, где семантически близкие слова находятся рядом друг с другом.

### Основная идея

Вместо того чтобы представлять слово как просто текстовую строку, мы представляем его как вектор чисел:

```
слово "кошка" → [0.25, -0.15, 0.89, 0.12, ..., -0.03]
слово "собака" → [0.28, -0.12, 0.85, 0.15, ..., -0.02]
слово "число" → [-0.45, 0.67, -0.23, 0.91, ..., 0.54]
```

**Ключевое наблюдение**: векторы "кошка" и "собака" похожи (близки в пространстве), потому что эти слова имеют сходные значения и используются в похожих контекстах. Вектор "число" сильно отличается, так как это слово относится к другой семантической области.

### Преимущества векторного представления

1. **Захват семантики**: слова с близким значением имеют близкие векторы
2. **Операции над векторами**: можно выполнять математические операции (сложение, вычитание)
3. **Измерение сходства**: вычисление косинусного расстояния между векторами
4. **Сжатие информации**: вместо больших матриц используем компактные векторы

Пример семантических отношений через векторные операции:

```
vector("король") - vector("мужчина") + vector("женщина") ≈ vector("королева")
```

Эта формула показывает, что вектор "королева" близок к результату этой операции в семантическом пространстве.

## Семантическое пространство

### Что такое семантическое пространство?

**Семантическое пространство** — это математическое пространство высокой размерности, где:
- Каждое слово представлено точкой (вектором)
- Расстояние между точками отражает семантическое различие
- Семантически близкие слова расположены близко друг к другу

### Основные свойства

**Размерность**: обычно 50–300 измерений (в больших моделях до 768 или выше)

**Метрики расстояния**: для измерения сходства между словами используют косинусное расстояние:

$$\text{cosine\_similarity}(\vec{u}, \vec{v}) = \frac{\vec{u} \cdot \vec{v}}{|\vec{u}| \cdot |\vec{v}|}$$

где $\vec{u} \cdot \vec{v}$ — скалярное произведение, $|\vec{u}|$ — норма вектора.

Значение косинусного сходства от -1 до 1:
- **1** означает идентичное направление (полное сходство)
- **0** означает ортогональность (отсутствие сходства)
- **-1** означает противоположное направление

### Аналогии в семантическом пространстве

Векторы слов естественным образом кодируют отношения между словами:

```
vector("Россия") - vector("Москва") + vector("Франция") ≈ vector("Париж")
vector("большой") - vector("big") + vector("высокий") ≈ vector("tall")
```

Это работает потому, что вычитание убирает информацию о конкретной стране, а добавление переводит в новую страну, сохраняя отношение "столица — страна".

## Word2Vec: пропуск-граммы и CBOW

### Основная архитектура

**Word2Vec** — это нейросетевой алгоритм, разработанный Google, для обучения векторов слов. Существует две основные версии:

#### 1. Skip-gram (пропуск-граммы)

**Идея**: предсказать контекстные слова по центральному слову.

```
Предложение: "Быстрая коричневая лиса прыгает через забор"

Окно контекста (размер 2):
- центральное слово: "коричневая"
- контекстные слова: "Быстрая", "лиса", "прыгает", "через"

Модель обучается: "коричневая" → {предсказать "Быстрая", "лиса", "прыгает", "через"}
```

**Архитектура**:
- Входной слой: один-горячее кодирование центрального слова (vocab_size)
- Скрытый слой: embedding (размер embedding_dim)
- Выходной слой: предсказание контекстных слов (vocab_size)

**Преимущества Skip-gram**:
- Хорошо работает с редкими словами
- Более точен для семантических отношений

#### 2. CBOW (Continuous Bag of Words)

**Идея**: предсказать центральное слово по его контексту (противоположность Skip-gram).

```
Предложение: "Быстрая коричневая лиса прыгает через забор"

Окно контекста (размер 2):
- контекстные слова: "Быстрая", "лиса", "прыгает", "через"
- центральное слово (предсказываемое): "коричневая"

Модель обучается: {входные "Быстрая", "лиса", "прыгает", "через"} → "коричневая"
```

**Архитектура**:
- Входной слой: усреднение one-hot векторов контекстных слов
- Скрытый слой: embedding
- Выходной слой: предсказание центрального слова

**Преимущества CBOW**:
- Быстрее обучается (один выход вместо нескольких)
- Лучше работает с большими датасетами
- Требует меньше памяти

### Функция потерь и оптимизация

Word2Vec использует **negative sampling** (отрицательная выборка) для эффективного обучения:

$$L = -\log \sigma(\vec{w}^T \vec{c}) - \sum_{i=1}^{k} \mathbb{E}_{w_i \sim P_n}[\log \sigma(-\vec{w}_i^T \vec{c})]$$

где:
- $\vec{w}$ — вектор целевого слова
- $\vec{c}$ — вектор контекстного слова
- $\sigma$ — сигмоид функция
- $k$ — количество отрицательных примеров
- $P_n$ — распределение отрицательных примеров

### Практический пример: обучение Word2Vec

```python
from gensim.models import Word2Vec
import nltk

# Пример текста на русском
texts = [
    ["быстрая", "коричневая", "лиса", "прыгает", "через", "забор"],
    ["ленивая", "собака", "спит", "на", "полу"],
    ["кошка", "ловит", "мышку", "в", "саду"],
    ["птица", "поёт", "в", "ветвях", "дерева"],
    ["рыба", "плывет", "в", "чистой", "воде"]
]

# Обучение модели Skip-gram
model_skipgram = Word2Vec(
    sentences=texts,
    vector_size=100,      # размерность вектора
    window=2,             # размер окна контекста
    min_count=1,          # минимальное количество повторений слова
    sg=1,                 # 1 для Skip-gram, 0 для CBOW
    epochs=10,            # количество эпох обучения
    workers=4             # количество потоков
)

# Получение вектора слова
vector_lisa = model_skipgram.wv['лиса']
print(f"Вектор слова 'лиса': {vector_lisa}")
print(f"Размерность вектора: {len(vector_lisa)}")

# Поиск похожих слов
similar_words = model_skipgram.wv.most_similar('лиса', topn=3)
print(f"Слова, похожие на 'лиса': {similar_words}")

# Вычисление сходства между двумя словами
similarity = model_skipgram.wv.similarity('лиса', 'собака')
print(f"Сходство между 'лиса' и 'собака': {similarity:.4f}")

# Вычисление сходства между двумя словами
similarity_different = model_skipgram.wv.similarity('лиса', 'рыба')
print(f"Сходство между 'лиса' и 'рыба': {similarity_different:.4f}")
```

**Ожидаемый результат**:
- Слова "лиса" и "собака" имеют высокое сходство (оба животные)
- Слова "лиса" и "рыба" имеют меньшее сходство (разные типы животных)

### Сравнение Skip-gram и CBOW

| Характеристика | Skip-gram | CBOW |
|---|---|---|
| **Задача** | Центральное слово → контекст | Контекст → центральное слово |
| **Скорость обучения** | Медленнее | Быстрее |
| **Редкие слова** | Лучше | Хуже |
| **Размер датасета** | Меньше требуется | Больше требуется |
| **Точность** | Выше на малых данных | Выше на больших данных |

## GloVe: Global Vectors for Word Representation

### История и мотивация

**GloVe** (Global Vectors for Word Representation) был разработан Станфордским университетом как альтернатива Word2Vec. Основная идея: объединить преимущества локальных методов (как Word2Vec) и глобальных методов матричной факторизации.

### Основная концепция

GloVe основана на **матрице совместного появления слов (co-occurrence matrix)**:

```
       кот  собака  красивый  бежит
кот  [  0     15       3        5   ]
собака [15     0       2       12   ]
красивый [ 3    2       0        4   ]
бежит [ 5    12       4        0   ]
```

где $X_{ij}$ — количество раз, слово $i$ появляется в контексте слова $j$.

### Функция потерь GloVe

$$J = \sum_{i,j=1}^{V} f(X_{ij}) (w_i^T w_j + b_i + b_j - \log X_{ij})^2$$

где:
- $w_i, w_j$ — векторы слов
- $b_i, b_j$ — смещения (biases)
- $f(X_{ij})$ — весовая функция, что избегает переобучения на частых парах слов
- $X_{ij}$ — количество совместного появления

**Весовая функция** обычно определяется как:

$$f(x) = \begin{cases}
(x/x_{max})^{3/4} & \text{если } x < x_{max} \\
1 & \text{в противном случае}
\end{cases}$$

Эта функция придает больший вес частым парам, но избегает их доминирования.

### Ключевое преимущество

GloVe явно учитывает статистику корпуса (матрица совместного появления), что делает векторы более согласованными и легче интерпретируемыми.

### Практический пример: использование предобученных GloVe

```python
import numpy as np

# Загрузка предобученных GloVe векторов
# Предположим, что файл glove.6B.100d.txt уже скачан

def load_glove_vectors(filepath, word_to_index_size=None):
    """Загрузка GloVe векторов из файла"""
    embeddings_dict = {}
    
    with open(filepath, 'r', encoding='utf-8') as f:
        for line in f:
            values = line.split()
            word = values[0]
            vector = np.array(values[1:], dtype='float32')
            embeddings_dict[word] = vector
    
    return embeddings_dict

# Пример использования (если файл доступен)
# glove_vectors = load_glove_vectors('glove.6B.100d.txt')

# Для демонстрации создадим простую имитацию
np.random.seed(42)

def simulate_glove_vectors(words, dim=100):
    """Имитация GloVe векторов для примера"""
    return {word: np.random.randn(dim) for word in words}

words = ['король', 'королева', 'мужчина', 'женщина', 'принц', 'принцесса']
glove_sim = simulate_glove_vectors(words)

# Вычисление сходства между словами
def cosine_similarity(vec1, vec2):
    """Вычисление косинусного сходства"""
    dot_product = np.dot(vec1, vec2)
    norm1 = np.linalg.norm(vec1)
    norm2 = np.linalg.norm(vec2)
    return dot_product / (norm1 * norm2)

# Демонстрация векторных операций
print("=== Семантические аналогии через GloVe ===")
print(f"Сходство 'король' и 'королева': {cosine_similarity(glove_sim['король'], glove_sim['королева']):.4f}")
print(f"Сходство 'король' и 'принц': {cosine_similarity(glove_sim['король'], glove_sim['принц']):.4f}")
print(f"Сходство 'король' и 'собака': {cosine_similarity(glove_sim['король'], glove_sim['королева']):.4f}")

# Демонстрация векторной операции (аналогия)
print("\n=== Векторная аналогия ===")
print("Операция: король - мужчина + женщина")

# Выполнение операции
analogy_result = glove_sim['король'] - glove_sim['мужчина'] + glove_sim['женщина']

# Поиск ближайшего слова к результату
print("\nТоп-3 слова, ближайшие к результату операции:")
similarities = {}
for word in words:
    sim = cosine_similarity(analogy_result, glove_sim[word])
    similarities[word] = sim

sorted_words = sorted(similarities.items(), key=lambda x: x[1], reverse=True)
for word, sim in sorted_words[:3]:
    print(f"  {word}: {sim:.4f}")
```

**Объяснение**:
- GloVe учитывает глобальную статистику корпуса
- Результаты более согласованы с лингвистическими отношениями
- Векторы можно интерпретировать через оси семантического пространства

## Сравнение Word2Vec и GloVe

| Параметр | Word2Vec | GloVe |
|---|---|---|
| **Метод** | Предиктивный (локальный контекст) | Факторизация (глобальная статистика) |
| **Основа** | Нейросетевая модель | Матричная факторизация |
| **Скорость обучения** | Средняя | Зависит от размера матрицы |
| **Использование памяти** | Низкое | Среднее/высокое |
| **Качество на малых данных** | Хорошее | Зависит от настроек |
| **Интерпретируемость** | Средняя | Высокая (глобальная статистика) |
| **Предобученные модели** | Много доступно | Много доступно (особенно для английского) |

## Практическое применение

### Пример 1: Анализ сходства документов

```python
from gensim.models import Word2Vec
import numpy as np

# Текстовые документы
documents = [
    "быстрая коричневая лиса прыгает через забор",
    "ленивая собака спит на полу",
    "дикая кошка охотится в лесу",
    "домашняя собака играет с мячом"
]

# Токенизация
texts = [doc.split() for doc in documents]

# Обучение модели
model = Word2Vec(texts, vector_size=50, window=2, min_count=1, sg=1)

# Вычисление вектора документа как среднее его слов
def get_document_vector(words, model):
    vectors = []
    for word in words:
        if word in model.wv:
            vectors.append(model.wv[word])
    
    if not vectors:
        return np.zeros(model.vector_size)
    
    return np.mean(vectors, axis=0)

# Векторы документов
doc_vectors = [get_document_vector(text, model) for text in texts]

# Вычисление сходства между документами
def cosine_similarity(vec1, vec2):
    dot_product = np.dot(vec1, vec2)
    norm1 = np.linalg.norm(vec1)
    norm2 = np.linalg.norm(vec2)
    
    if norm1 == 0 or norm2 == 0:
        return 0
    
    return dot_product / (norm1 * norm2)

# Сравнение первого документа со всеми остальными
print("=== Сходство документов ===")
print(f"Базовый документ: '{documents[0]}'")
for i in range(1, len(documents)):
    sim = cosine_similarity(doc_vectors[0], doc_vectors[i])
    print(f"  Сходство с документом {i}: {sim:.4f}")
    print(f"    Текст: '{documents[i]}'")
```

### Пример 2: Классификация слов по семантическим группам

```python
from gensim.models import Word2Vec
import numpy as np

# Тренировочные данные
training_texts = [
    "кошка сидит на коврике",
    "собака бегает в парке",
    "тигр охотится в джунглях",
    "лошадь скачет по полю",
    "помидор красный и спелый",
    "яблоко зелёное и твёрдое",
    "морковь оранжевая и сладкая",
    "груша жёлтая и сочная"
]

# Токенизация
texts = [text.split() for text in training_texts]

# Обучение модели
model = Word2Vec(texts, vector_size=50, window=2, min_count=1, sg=0, epochs=20)

# Определение категорий
animals = ['кошка', 'собака', 'тигр', 'лошадь']
vegetables = ['помидор', 'морковь']
fruits = ['яблоко', 'груша']

# Вычисление центра категории
def get_category_center(words, model):
    vectors = [model.wv[word] for word in words if word in model.wv]
    return np.mean(vectors, axis=0)

animal_center = get_category_center(animals, model)
fruit_center = get_category_center(fruits, model)
vegetable_center = get_category_center(vegetables, model)

# Классификация новых слов
def cosine_similarity(vec1, vec2):
    dot_product = np.dot(vec1, vec2)
    norm1 = np.linalg.norm(vec1)
    norm2 = np.linalg.norm(vec2)
    return dot_product / (norm1 * norm2)

def classify_word(word, model, categories):
    """Классифицировать слово по самой близкой категории"""
    if word not in model.wv:
        return "Unknown"
    
    word_vector = model.wv[word]
    
    distances = {}
    for category_name, center in categories.items():
        sim = cosine_similarity(word_vector, center)
        distances[category_name] = sim
    
    return max(distances, key=distances.get)

# Классификация животного
test_word = 'кошка'
categories = {
    'животные': animal_center,
    'фрукты': fruit_center,
    'овощи': vegetable_center
}

print(f"=== Классификация слова '{test_word}' ===")
predicted_category = classify_word(test_word, model, categories)
print(f"Предсказанная категория: {predicted_category}")
```

## Ограничения и особенности

### Word2Vec

**Ограничения**:
- Не учитывает морфологию (одинаковые векторы для всех форм слова в базовой версии)
- Размер окна контекста фиксирован
- Полисемия (многозначные слова): одно слово имеет один вектор, но может иметь разные значения в разных контекстах

**Особенности**:
- Хорошо работает с редкими словами в Skip-gram версии
- Быстро обучается
- Требует меньше данных чем GloVe

### GloVe

**Ограничения**:
- Требует больше памяти для хранения матрицы совместного появления
- Обучение может быть медленнее на очень больших корпусах
- Требует больше данных для хорошего качества

**Особенности**:
- Более согласованные результаты
- Лучше захватывает глобальные статистические свойства
- Интерпретируемые оси в пространстве

print("Слова, похожие на 'машина':")
for word, score in similar:
    print(f"  {word}: {score:.4f}")

# 4. Вычисление сходства
sim = model.wv.similarity('машина', 'велосипед')
print(f"\nСходство 'машина' и 'велосипед': {sim:.4f}")

print(f"Сходство 'машина' и 'автомобиль': {sim2:.4f}")

```python
import numpy as np

def solve_analogy(word_a, word_b, word_c, model):
    """Решение семантической аналогии"""
    # Проверка наличия слов в модели
    required_words = [word_a, word_b, word_c]
    for word in required_words:
        if word not in model.wv:
            return None
    
    # Вычисление вектора для аналогии
    vector_a = model.wv[word_a]
    vector_b = model.wv[word_b]
    vector_c = model.wv[word_c]
    
    # Формула: result_vector = word_c - word_a + word_b
    result_vector = vector_c - vector_a + vector_b
    
    # Нормализация
    result_vector = result_vector / np.linalg.norm(result_vector)
    
    # Поиск ближайшего слова (исключая исходные слова)
    similarities = {}
    for word in model.wv.index_to_key:
        if word not in required_words:
            sim = np.dot(result_vector, model.wv[word] / np.linalg.norm(model.wv[word]))
            similarities[word] = sim
    
    # Возвращение слова с максимальным сходством
    best_word = max(similarities, key=similarities.get)
    return best_word
```

**Решение**:

```python
from gensim.models import Word2Vec
from sklearn.cluster import KMeans
import numpy as np

# Подготовка корпуса
corpus = [
    "кошка спит на диване",
    "собака играет в парке",
    "тигр охотится в джунглях",
    "помидор растет на грядке",
    "яблоко висит на дереве",
    "морковь оранжевая и сладкая",
]

texts = [text.split() for text in corpus]
model = Word2Vec(texts, vector_size=50, window=2, min_count=1, sg=1)

words = ['кошка', 'собака', 'тигр', 'помидор', 'яблоко', 'морковь']

# Получение векторов
word_vectors = np.array([model.wv[word] for word in words])

# Кластеризация
kmeans = KMeans(n_clusters=2, random_state=42, n_init=10)
clusters = kmeans.fit_predict(word_vectors)

# Вывод результатов
print("=== Кластеризация слов ===")
for i, word in enumerate(words):
    print(f"{word}: кластер {clusters[i]}")

# Интерпретация кластеров
print("\nИнтерпретация кластеров:")
for cluster_id in set(clusters):
    cluster_words = [word for word, cluster in zip(words, clusters) if cluster == cluster_id]
    print(f"Кластер {cluster_id}: {', '.join(cluster_words)}")
```

## Ключевые термины и определения

**Embedding (вложение)** — представление слова в виде вектора вещественных чисел

**Co-occurrence (совместное появление)** — количество раз два слова появляются в одном контексте

**Semantic space (семантическое пространство)** — многомерное пространство, где каждое измерение соответствует семантическому признаку

**Context window (окно контекста)** — набор слов вокруг целевого слова, используемых для обучения

**Skip-gram** — архитектура Word2Vec, предсказывающая контекст из центрального слова

**CBOW** — архитектура Word2Vec, предсказывающая центральное слово из контекста

**Negative sampling (отрицательная выборка)** — метод оптимизации, использующий случайные отрицательные примеры

**GloVe** — алгоритм, объединяющий факторизацию матрицы совместного появления с предсказательными методами

**Cosine similarity (косинусное сходство)** — мера сходства между векторами, основанная на углу между ними

**Word analogy (словесная аналогия)** — нахождение слова D для аналогии "A относится к B как C относится к D"
