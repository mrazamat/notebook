# Тематическое моделирование: Лингвистический анализ и вероятностные модели

**Область:** Компьютерная лингвистика  
**Предмет:** Алгоритмы NLP  

## Введение

Тематическое моделирование — это методология автоматического выявления скрытых тем в коллекции текстовых документов. Представьте себе стопку газетных статей: задача тематического моделирования состоит в том, чтобы выяснить, о чём говорится в этих статьях, без предварительного ручного прочтения каждой из них.

**Ключевая идея:** Предполагается, что каждый документ состоит из смеси нескольких тем, а каждая тема состоит из смеси слов.

**Практическое применение:**
- Анализ новостных потоков
- Категоризация научных публикаций
- Выявление трендов в социальных сетях
- Рекомендационные системы
- Историческое анализ корпусов текстов

## Основные концепции

### Документ, корпус и словарь

**Документ** — отдельный текст (статья, твит, рецензия, письмо).

**Корпус** — коллекция документов, которую мы анализируем.

**Словарь** — множество всех уникальных слов, встречающихся в корпусе. Если в корпусе есть слова "кот", "собака", "животное", то словарь содержит эти три слова.

**Пример:**
```
Корпус:
  Документ 1: "Кот спит на диване"
  Документ 2: "Собака прыгает в парке"
  Документ 3: "Кот и собака — друзья"

Словарь: {кот, спит, на, диване, собака, прыгает, в, парке, и, друзья}
```

### Темы и представление документов

**Тема** — вероятностное распределение над словами. Например, тема "Животные" может быть представлена так:
- кот: 0.15
- собака: 0.15
- лапы: 0.10
- мяукать: 0.08
- лаять: 0.08
- (остальные слова: меньшие вероятности)

**Представление документа через темы:** Каждый документ характеризуется смесью тем. Например:
- Документ 1: 80% тема "Животные", 20% тема "Мебель"
- Документ 2: 70% тема "Животные", 30% тема "Парк и природа"

Эти смеси моделируют, "о чём" говорит документ.

## Вероятностные основы

### Условная вероятность

Напомним, что **условная вероятность** P(A|B) — это вероятность события A при условии, что произошло событие B.

$$P(A|B) = \frac{P(A \cap B)}{P(B)}$$

**Пример:** Если в корпусе есть 100 документов, и в 30 из них говорится о "кошках", то P(тема = кошки) = 0.30.

### Байесовская логика

Тематическое моделирование основано на теореме Байеса:

$$P(topic|word) = \frac{P(word|topic) \cdot P(topic)}{P(word)}$$

Смысл: если мы видим слово "кот", какова вероятность, что документ относится к теме "Животные"?

### Порождающий процесс (Generative Process)

Вероятностные модели работают через описание того, как *порождаются* документы:

1. Выбираем распределение тем для документа
2. Для каждого слова в документе:
   - Случайно выбираем тему согласно распределению
   - Выбираем слово согласно распределению слов в этой теме

Задача моделирования — обратить процесс: имея готовые документы, восстановить скрытые темы.

## Модель Latent Dirichlet Allocation (LDA)

### Что такое LDA?

**Latent Dirichlet Allocation (LDA)** — наиболее популярная модель тематического моделирования. Слово "latent" означает "скрытый" — темы предполагаются скрытыми, их нужно выявить. "Dirichlet" — это название вероятностного распределения, используемого в модели.

### Математическое описание LDA

LDA предполагает следующий порождающий процесс для каждого документа d:

**Шаг 1.** Выбрать распределение тем для документа:
$$\theta_d \sim \text{Dirichlet}(\alpha)$$

Здесь $\theta_d$ — вектор вероятностей тем для документа d, а α — параметр концентрации.

**Шаг 2.** Для каждого слова $w_n$ в документе d (где n = 1, 2, ..., N_d):
- Выбрать тему: $z_n \sim \text{Categorical}(\theta_d)$
- Выбрать слово: $w_n \sim \text{Categorical}(\beta_{z_n})$

Здесь:
- $z_n$ — тема для n-го слова
- $\beta_k$ — распределение слов в теме k

### Гиперпараметры LDA

**α (alpha)** — гиперпараметр, контролирующий разреженность распределения тем в документе.
- Малое α (например, 0.01) → документы содержат несколько главных тем
- Большое α (например, 10) → все темы равномерно распределены в документе

**β (beta)** — гиперпараметр, контролирующий разреженность распределения слов в теме.
- Малое β → каждая тема содержит несколько ключевых слов
- Большое β → слова распределены равномерно в теме

## Вывод параметров модели

### Задача вывода

Имея корпус документов, мы хотим найти:
- **θ** — распределение тем для каждого документа
- **β** — распределение слов для каждой темы
- **z** — скрытые темы для каждого слова

Прямой расчет вероятностей невозможен из-за сложной зависимости переменных. Используются методы приближённого вывода.

### Gibbs Sampling (гиббсовская выборка)

**Gibbs Sampling** — итеративный алгоритм, который последовательно переопределяет тему каждого слова, основываясь на:
- Текущих темах соседних слов в документе
- Частотности слова в теме
- Частотности темы в документе

**Интуиция:** Если слово "кот" часто встречается в теме "Животные" и документ много говорит о животных, то вероятность, что текущее слово "кот" относится к теме "Животные", высока.

**Алгоритм (упрощённо):**

```
ИНИЦИАЛИЗАЦИЯ: Случайно назначить тему каждому слову

ИТЕРАЦИИ (повторять T раз):
  для каждого слова w_n в каждом документе:
    Вычислить P(z_n = k | z_{-n}, w, d, α, β)
    Выбрать новую тему z_n согласно распределению
```

После достаточного числа итераций модель сходится к хорошему решению.

### Вариационный вывод (Variational Inference)

Альтернативный метод, чаще используемый в практических приложениях для большых корпусов:
- Быстрее, чем Gibbs Sampling
- Хорошо масштабируется
- Основан на минимизации расстояния между истинным и приближённым распределением (KL-дивергенция)

## Лингвистическая подготовка текстов

Перед применением LDA текст должен быть обработан.

### Токенизация

**Токенизация** — разбиение текста на отдельные слова (токены).

**Пример:**
```
Входной текст: "Кот спит на диване."
Токены: ["Кот", "спит", "на", "диване", "."]
```

### Нормализация

**Нормализация** — приведение слов к единому виду.

**Приёмы:**
- **Приведение к нижнему регистру:** "Кот" → "кот"
- **Удаление пунктуации:** "диване." → "диване"
- **Лемматизация:** "спит", "спал", "спать" → "спать" (основная форма)
- **Стемминг:** "кошка", "кошачий" → "кош" (корень)

**Пример полной обработки:**
```
Исходный текст: "Кот спит на диване. Кошка очень усталая!"
После токенизации: ["Кот", "спит", "на", "диване", "Кошка", "очень", "усталая"]
После нормализации: ["кот", "спит", "на", "диван", "кошка", "очень", "усталый"]
После лемматизации: ["кот", "спать", "на", "диван", "кот", "очень", "усталый"]
```

### Удаление стоп-слов

**Стоп-слова** — частые служебные слова без существенного смысла (предлоги, союзы, артикли).

**Примеры русских стоп-слов:** и, в, на, что, как, если, или, так, это, это, которые, их...

**Зачем удалять?**
- Они занимают место в словаре, но не помогают различать темы
- Пример: если тема "Спорт" содержит слова "в", "на", "и" наравне с "футбол", "гол", это снижает информативность

**После удаления стоп-слов:**
```
Исходный текст: "Кот спит на диване. Кошка очень усталая!"
После удаления стоп-слов: ["кот", "спать", "диван", "кот", "очень", "усталый"]
```

## Практический пример: LDA на русском корпусе

### Подготовка данных

```python
import numpy as np
from collections import defaultdict
import re

# Стоп-слова на русском
russian_stopwords = {
    'и', 'в', 'на', 'что', 'как', 'если', 'или', 'так', 'это', 
    'по', 'которые', 'их', 'для', 'от', 'к', 'со', 'с', 'а', 'но'
}

# Пример корпуса
corpus = [
    "Кот спит на диване в гостиной",
    "Собака прыгает в парке на траве",
    "Кот и собака — верные друзья животных",
    "Диван стоит в углу комнаты гостиной",
    "В парке растут красивые деревья и цветы"
]

# Токенизация и нормализация
def preprocess_document(text):
    # Приведение к нижнему регистру
    text = text.lower()
    
    # Удаление пунктуации
    text = re.sub(r'[^\w\s]', '', text)
    
    # Токенизация
    tokens = text.split()
    
    # Удаление стоп-слов
    tokens = [t for t in tokens if t not in russian_stopwords]
    
    return tokens

# Обработка корпуса
processed_corpus = [preprocess_document(doc) for doc in corpus]

print("Обработанный корпус:")
for i, doc in enumerate(processed_corpus):
    print(f"Документ {i+1}: {doc}")
```

**Вывод:**
```
Обработанный корпус:
Документ 1: ['кот', 'спит', 'диване', 'гостиной']
Документ 2: ['собака', 'прыгает', 'парке', 'траве']
Документ 3: ['кот', 'собака', 'верные', 'друзья', 'животных']
Документ 4: ['диван', 'стоит', 'углу', 'комнаты', 'гостиной']
Документ 5: ['парке', 'растут', 'красивые', 'деревья', 'цветы']
```

### Создание словаря и документ-термин матрицы

```python
# Построение словаря
vocabulary = {}
word_id = 0
for doc in processed_corpus:
    for word in doc:
        if word not in vocabulary:
            vocabulary[word] = word_id
            word_id += 1

print(f"Размер словаря: {len(vocabulary)} слов")
print(f"Словарь: {vocabulary}")

# Создание документ-термин матрицы
num_docs = len(processed_corpus)
num_words = len(vocabulary)
doc_term_matrix = np.zeros((num_docs, num_words))

for doc_id, doc in enumerate(processed_corpus):
    for word in doc:
        word_id = vocabulary[word]
        doc_term_matrix[doc_id, word_id] += 1

print(f"\nДокумент-термин матрица (размер {num_docs}x{num_words}):")
print(doc_term_matrix)
```

### Применение LDA

```python
# Используем библиотеку gensim для LDA
from gensim import corpora
from gensim.models import LdaModel

# Создание корпуса в формате gensim
# Каждый документ представляется списком кортежей (word_id, count)
gensim_corpus = []
for doc in processed_corpus:
    bow = defaultdict(int)
    for word in doc:
        word_id = vocabulary[word]
        bow[word_id] += 1
    gensim_corpus.append([(word_id, count) 
                          for word_id, count in bow.items()])

print("Корпус в формате bag-of-words:")
for i, doc in enumerate(gensim_corpus):
    print(f"Документ {i+1}: {doc}")

# Обучение LDA модели
num_topics = 2  # Предполагаем 2 темы
lda_model = LdaModel(
    corpus=gensim_corpus,
    id2word={v: k for k, v in vocabulary.items()},
    num_topics=num_topics,
    random_state=42,
    passes=10,
    alpha='auto',
    per_word_topics=True
)

# Вывод тем
print(f"\nОбнаруженные {num_topics} темы:")
for topic_id in range(num_topics):
    print(f"\nТема {topic_id}:")
    terms = lda_model.show_topic(topic_id, topn=5)
    for term, prob in terms:
        print(f"  {term}: {prob:.4f}")
```

**Ожидаемый вывод:**
```
Обнаруженные 2 темы:

Тема 0:
  собака: 0.0523
  парке: 0.0498
  прыгает: 0.0485
  траве: 0.0472
  друзья: 0.0451

Тема 1:
  кот: 0.0615
  диван: 0.0594
  гостиной: 0.0578
  спит: 0.0551
  комнаты: 0.0524
```

## Оценка качества модели

### Перплексия (Perplexity)

**Перплексия** — стандартная метрика для оценки качества вероятностной модели. Она показывает, насколько "удивлена" модель, видя тестовый документ.

$$\text{Perplexity} = \exp\left(-\frac{\sum_{d=1}^{M} \log P(w_d | \text{model})}{N}\right)$$

где:
- M — количество документов
- N — общее число слов
- P(w_d | model) — вероятность документа d согласно модели

**Интерпретация:**
- Низкая перплексия → модель хорошо объясняет данные
- Высокая перплексия → модель плохо соответствует данным

```python
# Расчёт перплексии
from gensim.models.perplexity import PerplexityModel

# На тестовом наборе данных
test_corpus = gensim_corpus  # В реальности — отдельный тест-набор
perplexity = lda_model.log_perplexity(test_corpus)

print(f"Перплексия модели: {np.exp(-perplexity):.4f}")
```

### Когерентность (Coherence)

**Когерентность** — мера того, насколько "связаны" слова в каждой теме. Связанные слова указывают на осмысленную тему.

```python
from gensim.models import CoherenceModel

coherence_model = CoherenceModel(
    model=lda_model,
    corpus=gensim_corpus,
    dictionary=lda_model.id2word,
    coherence='u_mass'
)

coherence_score = coherence_model.get_coherence()
print(f"Когерентность модели: {coherence_score:.4f}")
```

**Интерпретация:**
- Значения ближе к 0 → хорошая когерентность
- Значения сильно отрицательные → плохая когерентность

## Выбор числа тем

Число тем K — важный гиперпараметр, который нужно выбрать перед обучением.

### Метод локтя (Elbow Method)

Обучаем модели с разным числом тем и смотрим, как меняется метрика качества:

```python
# Поиск оптимального числа тем
coherence_scores = []
perplexity_scores = []
topic_range = range(2, 11)

for num_topics in topic_range:
    model = LdaModel(
        corpus=gensim_corpus,
        id2word={v: k for k, v in vocabulary.items()},
        num_topics=num_topics,
        random_state=42,
        passes=10
    )
    
    # Когерентность
    coherence = CoherenceModel(
        model=model,
        corpus=gensim_corpus,
        dictionary=model.id2word,
        coherence='u_mass'
    ).get_coherence()
    coherence_scores.append(coherence)
    
    # Перплексия
    perplexity = np.exp(-model.log_perplexity(gensim_corpus))
    perplexity_scores.append(perplexity)
    
    print(f"Темы: {num_topics}, "
          f"Когерентность: {coherence:.4f}, "
          f"Перплексия: {perplexity:.4f}")
```

**Рекомендация:** Выбираем число тем, где когерентность начинает стабилизироваться (точка "локтя").

## Интерпретация результатов

### Качественный анализ тем

1. **Посмотрите на слова в каждой теме** — Имеют ли они смысл вместе?
2. **Проверьте распределение тем в документах** — Документы похожи по теме?
3. **Сравните с вашими ожиданиями** — Соответствуют ли найденные темы реальной структуре корпуса?

### Распространённые проблемы

**Проблема: Темы не интерпретируемы**
- Решение: Удалить больше стоп-слов, применить лемматизацию, увеличить минимальную частоту слова

**Проблема: Много тем содержат одни и те же слова**
- Решение: Увеличить гиперпараметр β (или уменьшить коэффициент регуляризации), уменьшить число тем

**Проблема: Одна тема содержит почти все слова**
- Решение: Уменьшить гиперпараметр α, увеличить число итераций обучения

## Основные формулы и обозначения

| Обозначение | Описание |
|-------------|----------|
| $D$ | Число документов |
| $V$ | Размер словаря |
| $K$ | Число тем |
| $\theta_d$ | Распределение тем в документе d |
| $\beta_k$ | Распределение слов в теме k |
| $z_n$ | Тема для n-го слова в документе |
| $w_n$ | n-е слово в документе |
| $\alpha$ | Гиперпараметр для распределения тем |
| $\eta$ | Гиперпараметр для распределения слов |
| $P(w\|z)$ | Вероятность слова при данной теме |
| $P(z\|d)$ | Вероятность темы при данном документе |

## Практические советы для работы с LDA

**Советы по предобработке:**
- Всегда удаляйте стоп-слова
- Применяйте лемматизацию для морфологически богатых языков (русский, немецкий и т.д.)
- Убирайте редкие слова (появляющиеся менее 2-3 раз)
- Убирайте очень частые слова (появляющиеся в более чем 80% документов)

**Советы по настройке параметров:**
- Начните с α = 50/K и η = 200/V
- Увеличивайте число итераций (passes), пока модель не сходится (20-100 итераций)
- Используйте несколько случайных инициализаций (random_state) и выбирайте лучшую

**Советы по интерпретации:**
- Изучите документы с высокой вероятностью определённой темы
- Используйте визуализацию (например, pyLDAvis) для исследования тем
- Не верьте слепо автоматическим метрикам — всегда проверяйте вручную

**В: Что делать, если число тем неизвестно?**
О: Используйте метод локтя или метод силуэта. Обучите несколько моделей с разными K и выберите оптимальное.

**В: Можно ли применить LDA на других языках, кроме русского?**
О: Да, но нужна правильная предобработка для каждого языка. Для русского — лемматизация средствами mystem или pymorphy2.

**В: Как обновить модель на новых документах?**
О: Используйте online LDA. Это позволяет добавлять документы пакетами без переобучения всей модели.

**В: Насколько большим должен быть корпус?**
О: Минимум 100-200 документов, но для надёжных результатов рекомендуется 1000+ документов.

## Дополнительные ресурсы и расширения

### Модели, связанные с LDA

- **Hierarchical LDA (hLDA)** — иерархическое расположение тем
- **Dynamic LDA** — для анализа изменения тем во времени
- **Correlated Topic Model (CTM)** — моделирование корреляций между темами
- **Author-Topic Model** — учитывает авторов документов

### Библиотеки и инструменты

- **gensim** — основная библиотека для LDA на Python
- **pyLDAvis** — интерактивная визуализация тем
- **mallet** — быстрая реализация на Java
- **pymorphy2** — морфологический анализ для русского языка
