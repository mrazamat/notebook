# Определение и классификация типов текста

## Машинное обучение и теория нейронных моделей

**Курс:** Алгоритмы NLP  
**Специальность:** Компьютерная лингвистика

## Введение

Текст — это последовательность символов и слов, содержащая информацию. В контексте машинного обучения и нейронных сетей классификация текста становится фундаментальной задачей обработки естественного языка. Понимание типов текста помогает выбрать правильную модель и подход к решению конкретной задачи.

## Основные определения

### Что такое тип текста?

Тип текста — это категория текстового материала, определяемая по функциональным, структурным и стилистическим признакам. В машинном обучении тип текста служит целевым признаком (target) для обучения классификатора.

### Что такое классификация текста?

Классификация текста — это задача автоматического распределения текстовых документов по заранее определённым категориям.

Это задача обучения с учителем, где:
- **Входные данные**: текст (строка, документ, последовательность)
- **Выходные данные**: категория или метка (label)
- **Модель**: функция, которая отображает текст в категорию

## Уровни классификации текста

### Бинарная классификация

Текст классифицируется в одну из двух категорий.

**Примеры:**
- Анализ тональности: позитивный или негативный отзыв
- Фильтрация спама: спам или не спам
- Определение языка: русский или английский текст

```
f(text) → {0, 1} или {Класс_A, Класс_B}
```

### Мультиклассовая классификация

Текст классифицируется в одну категорию из более чем двух доступных.

**Примеры:**
- Определение жанра текста: новость, рецепт, инструкция, статья
- Классификация по языкам: русский, английский, французский, испанский
- Определение стиля: научный, художественный, разговорный, официальный

```
f(text) → {Класс_1, Класс_2, ..., Класс_n}
```

### Мультиметочная классификация

Один текст может принадлежать к нескольким категориям одновременно.

**Примеры:**
- Определение тем статьи: [политика, экономика, экология]
- Разметка ключевых слов в документе
- Классификация по жанрам фильма: [драма, детектив, криминал]

```
f(text) → {(Класс_1, вероятность_1), (Класс_2, вероятность_2), ...}
```

## Классификация типов текстов по функциональным признакам

### 1. По функциональному стилю

**Научный текст**

Характеристика: точность, объективность, специальные термины, структурированность.

```
Пример: "Процесс трансформации входной последовательности токенов в 
вектор представления осуществляется посредством многослойного перцептрона 
с функцией активации ReLU."
```

**Художественный текст**

Характеристика: эмоциональность, образность, метафоры, субъективность.

```
Пример: "Закат разливался по небу золотистыми красками, словно 
художник спешил запечатлеть последние моменты дня."
```

**Разговорный текст**

Характеристика: неформальность, простота, сленг, восклицания.

```
Пример: "Класс! Новая версия работает намного быстрее. Я в восторге!"
```

**Официально-деловой текст**

Характеристика: формальность, стандартизованность, точность, отсутствие эмоций.

```
Пример: "Уведомляем Вас об утверждении протокола заседания от 21 марта 2026 года."
```

### 2. По коммуникативной цели

**Информационный текст**

Основная цель: передача информации, сообщение фактов.

Примеры: новостные статьи, справочники, инструкции, доклады

**Убеждающий текст**

Основная цель: воздействие на читателя, изменение мнения.

Примеры: реклама, политические речи, рецензии, аргументированные статьи

**Развлекательный текст**

Основная цель: развлечение, создание эмоциональной реакции.

Примеры: художественные произведения, юмористические статьи, истории

## Подготовка данных для классификации

### Этап 1: Сбор и разметка данных

Необходимо собрать датасет текстов и вручную разметить каждый текст меткой класса.

```python
# Пример датасета
dataset = [

        "type": "scientific"
    },
    {

        "type": "spam"
    },
    {
        "text": "Утром я встретил старого друга в парке",
        "type": "artistic"
    },
    {
        "text": "Прошу направить счёт на оплату услуг",
        "type": "formal"
    },
]
```

### Этап 2: Нормализация и предобработка

Приведение текста к единообразному формату:

```python
import re

def preprocess_text(text):
    # Приведение к нижнему регистру
    text = text.lower()
    # Удаление пунктуации
    text = re.sub(r'[^\w\s]', '', text)
    # Удаление лишних пробелов
    text = ' '.join(text.split())
    return text

texts = ["Привет! Как дела?", "МАШИННОЕ ОБУЧЕНИЕ..."]
cleaned = [preprocess_text(t) for t in texts]
print(cleaned)
# Выход: ['привет как дела', 'машинное обучение']
```

### Этап 3: Векторизация текста

Преобразование текста в числовые представления для обработки моделью:

**Подход 1: Bag-of-Words (BoW)**

```python
from sklearn.feature_extraction.text import CountVectorizer

texts = ["токены и леммы", "обработка текста"]
vectorizer = CountVectorizer()
X = vectorizer.fit_transform(texts)

print("Словарь:", vectorizer.get_feature_names_out())
print("Матрица частот:\n", X.toarray())
```

**Подход 2: TF-IDF**

```python
from sklearn.feature_extraction.text import TfidfVectorizer

vectorizer = TfidfVectorizer(max_features=100)
X = vectorizer.fit_transform(texts)
print("TF-IDF матрица формы:", X.shape)
# (2, 5) — 2 документа, 5 уникальных слов
```

**Подход 3: Word Embeddings**

Встраивание слов в многомерное пространство, где семантически близкие слова находятся рядом.

```python
from gensim.models import Word2Vec

sentences = [["машинное", "обучение"], ["глубокое", "обучение"]]
model = Word2Vec(sentences=sentences, vector_size=100, window=5)

# Получение вектора для слова
vector = model.wv['обучение']
print("Размер вектора:", len(vector))  # 100
```

## Нейронные модели для классификации текста

### 1. Полносвязная нейронная сеть (Feedforward)

**Архитектура:**

```
Входной слой → Эмбеддинг слов → Flatten → Dense слои → Softmax
```

**Преимущества:**
- Простая в реализации
- Быстрая обработка

**Недостатки:**
- Игнорирует порядок слов
- Требует больше памяти

```python
import tensorflow as tf
from tensorflow import keras

model = keras.Sequential([
    keras.layers.Embedding(input_dim=1000, output_dim=128),
    keras.layers.Flatten(),
    keras.layers.Dense(64, activation='relu'),
    keras.layers.Dropout(0.2),
    keras.layers.Dense(4, activation='softmax')  # 4 класса
])

model.compile(optimizer='adam', loss='categorical_crossentropy', 
              metrics=['accuracy'])
```

### 2. Рекуррентная нейронная сеть (RNN/LSTM)

**Архитектура:**

```
Входной слой → Эмбеддинг → LSTM/GRU → Dense → Softmax
```

**Особенности:**
- Обрабатывает последовательности
- Сохраняет контекст между словами
- Может работать с текстами переменной длины

```python
model = keras.Sequential([
    keras.layers.Embedding(input_dim=1000, output_dim=128),
    keras.layers.LSTM(64, return_sequences=True),
    keras.layers.LSTM(32),
    keras.layers.Dense(64, activation='relu'),
    keras.layers.Dense(4, activation='softmax')
])
```

### 3. Сверточная нейронная сеть (CNN)

**Архитектура:**

```
Входной слой → Эмбеддинг → Conv1D → MaxPooling → Flatten → Dense
```

**Особенности:**
- Эффективно ловит локальные паттерны (n-граммы)
- Быстрее, чем RNN
- Хорошо работает для коротких текстов

```python
model = keras.Sequential([
    keras.layers.Embedding(input_dim=1000, output_dim=128),
    keras.layers.Conv1D(128, 5, activation='relu'),
    keras.layers.MaxPooling1D(5),
    keras.layers.Conv1D(128, 5, activation='relu'),
    keras.layers.GlobalMaxPooling1D(),
    keras.layers.Dense(64, activation='relu'),
    keras.layers.Dense(4, activation='softmax')
])
```

### 4. Трансформер (Transformer)

**Архитектура:**

```
Входной слой → Токенизация → Эмбеддинг → Multi-Head Attention → 
Feed-Forward → Выходной слой
```

**Особенности:**
- Параллельная обработка всей последовательности
- Механизм внимания (attention) улавливает долгосрочные зависимости
- SOTA результаты на большинстве задач

```python
from transformers import pipeline

classifier = pipeline(
    "text-classification", 
    model="cointegrated/rubert-tiny"
)
result = classifier("Отличный фильм, очень понравился!")
print(result)
# [{'label': 'POSITIVE', 'score': 0.95}]
```

## Практический пример: Классификация типов текста

### Задача
Классифицировать русские тексты на 4 типа: научный, художественный, разговорный, официальный.

### Реализация

```python
import numpy as np
from sklearn.model_selection import train_test_split
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.naive_bayes import MultinomialNB
from sklearn.metrics import accuracy_score, classification_report

# Шаг 1: Подготовка данных
X = [
    "Процесс фотосинтеза в растениях осуществляется в хлоропластах",
    "Я очень люблю гулять по лесу и наслаждаться природой",
    "Привет! Как прошёл твой день?",
    "Уведомляем об обязательной проверке документов",
    "Квантовые компьютеры используют принцип суперпозиции",
    "Ночью луна сияла в небе, как серебряная монета",
    "Ты уже видел новый фильм? Он просто огонь!",
    "Прошу направить результаты анализа на указанный адрес",
]

y = [
    "scientific", "artistic", "conversational", "formal",
    "scientific", "artistic", "conversational", "formal"
]

# Шаг 2: Разделение на обучающую и тестовую выборки
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, random_state=42
)

# Шаг 3: Векторизация текстов
vectorizer = TfidfVectorizer(lowercase=True, max_features=100)
X_train_vec = vectorizer.fit_transform(X_train)
X_test_vec = vectorizer.transform(X_test)

# Шаг 4: Обучение модели
model = MultinomialNB()
model.fit(X_train_vec, y_train)

# Шаг 5: Прогнозирование
y_pred = model.predict(X_test_vec)

# Шаг 6: Оценка
print("Точность:", accuracy_score(y_test, y_pred))
print("\nДетальный отчёт:")
print(classification_report(y_test, y_pred))

# Шаг 7: Классификация нового текста
new_text = "Рекомендуем внимательно изучить приложенное приложение"
new_vec = vectorizer.transform([new_text])
prediction = model.predict(new_vec)
probability = model.predict_proba(new_vec)

print(f"\nТекст: '{new_text}'")
print(f"Предсказанный класс: {prediction[0]}")
print(f"Уверенность: {max(probability[0]):.2f}")
```

## Метрики оценки качества

### Точность (Accuracy)

Процент правильно классифицированных примеров:

```
Accuracy = (TP + TN) / (TP + TN + FP + FN)
```

### Полнота (Recall)

Доля найденных объектов из всех объектов класса:

```
Recall = TP / (TP + FN)
```

### Точность класса (Precision)

Доля правильно предсказанных объектов от всех предсказанных:

```
Precision = TP / (TP + FP)
```

### F1-мера

Гармоническое среднее между точностью и полнотой:

```
F1 = 2 * (Precision * Recall) / (Precision + Recall)
```

## Вызовы и ограничения

### 1. Дисбаланс классов

Когда примеры одного класса значительно превосходят другие.

**Решение:**
- Взвешивание классов (class_weight)
- SMOTE (Synthetic Minority Over-sampling Technique)
- Stratified sampling

### 2. Многозначность слов

Одно слово может иметь разные значения в разных контекстах.

**Решение:**
- Контекстные эмбеддинги (BERT, ELMo)
- Лемматизация и нормализация

### 3. Нехватка данных

Для обучения хороших моделей нужны большие датасеты.

**Решение:**
- Transfer learning (предтренированные модели)
- Data augmentation (увеличение данных)
- Few-shot learning

## Справочник ключевых терминов

| Термин | Определение |
|--------|---|
| Эмбеддинг | Числовое представление текста в многомерном пространстве |
| Токен | Минимальная единица текста (слово, символ, подслово) |
| TF-IDF | Мера важности слова в документе относительно всего корпуса |
| Bag-of-Words | Модель представления текста как набор слов без учета порядка |
| LSTM | Long Short-Term Memory, тип рекуррентной нейронной сети |
| Attention | Механизм, позволяющий модели фокусироваться на важных частях |
| Softmax | Функция активации для многоклассовой классификации |
| Dropout | Техника регуляризации для предотвращения переобучения |
| Transfer learning | Использование знаний с одной задачи для решения другой |
| Transformer | Архитектура на базе механизма внимания без рекуррентности |
