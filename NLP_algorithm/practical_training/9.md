# Определение именных единиц (NER) и анализ настроения текста (Sentiment Analysis)

## Введение

В этом семинаре мы изучим два важных направления обработки естественного языка:

1. **Named Entity Recognition (NER)** — определение и классификация именных единиц (имена людей, названия мест, организации и т.д.)
2. **Sentiment Analysis** — определение эмоциональной окраски текста (позитивное, негативное, нейтральное настроение)

Эти методы лежат в основе многих практических приложений: анализ отзывов, информационный поиск, обработка социальных сетей.

## Часть 1: Определение именных единиц (NER)

### Что такое NER?

**Named Entity Recognition (NER)** — это задача автоматического выделения и классификации значимых объектов в тексте.

Пример:
```
Текст: "Иван Петров работает в компании Google в Лондоне"
    ↓
NER результат:
- "Иван Петров" → PERSON (Персона)
- "Google" → ORGANIZATION (Организация)  
- "Лондон" → LOCATION (Место)
```

### Основные классы именных единиц

| Класс | Описание | Примеры |
|-------|---------|---------|
| PERSON | Имена людей | Пушкин, Достоевский, Мария |
| LOCATION | Географические объекты | Москва, Европа, Волга |
| ORGANIZATION | Названия организаций | Газпром, МГУ, ООН |
| PRODUCT | Названия продуктов/товаров | iPhone, Windows, Tesla |
| DATE | Даты и периоды времени | 15 марта, 2024 год |
| MONEY | Денежные суммы | 1000 рублей, $50 |

### Подходы к NER

#### 1. Правило-ориентированный подход

Использование регулярных выражений и словарей:

```python
import re

def simple_ner_regex(text):
    # Простое выделение по шаблонам
    persons = re.findall(r'\b[А-Я][а-я]+ [А-Я][а-я]+\b', text)
    emails = re.findall(r'\b[a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,}\b', text)
    
    return {
        'PERSON': persons,
        'EMAIL': emails
    }

text = "Контакт: Иван Сидоров ivan.sidorov@mail.ru"
result = simple_ner_regex(text)
print(result)
# {'PERSON': ['Иван Сидоров'], 'EMAIL': ['ivan.sidorov@mail.ru']}
```

**Достоинства:** быстро, понятно, не требует обучающих данных

**Недостатки:** плохо работает на сложных случаях, требует ручной работы

#### 2. Статистический подход (машинное обучение)

Использование моделей типа CRF (Conditional Random Fields):

```python
# Пример с библиотекой spaCy (готовая предобученная модель)

```

**Достоинства:** точнее на сложных примерах, обучается на данных

**Недостатки:** требует обучающего набора, медленнее правил

#### 3. Нейросетевой подход

Использование глубокого обучения (LSTM, Transformers):


**Недостатки:** требуют больших данных и вычислительных ресурсов

### Практическое задание 1: Простой NER на Python

```python
# Задание: создать простой NER для выделения дат

def extract_dates(text):
    """
    Выделяет даты в формате: день месяца, год
    Примеры: 15 марта 2024, 3 января
    """
    import re
    
    months = ['января', 'февраля', 'марта', 'апреля', 'мая', 'июня',
              'июля', 'августа', 'сентября', 'октября', 'ноября', 'декабря']
    
    month_pattern = '|'.join(months)
    date_pattern = rf'\d{{1,2}}\s+({month_pattern})(?:\s+\d{{4}})?'
    
    dates = re.findall(date_pattern, text)
    return dates

# Тестирование
texts = [
    "Встреча запланирована на 15 марта 2024",
    "События: 3 января и 25 декабря",
    "С 1 июня по 30 августа 2024"
]

for text in texts:
    dates = extract_dates(text)
    print(f"Текст: {text}")
    print(f"Найденные даты: {dates}\n")
```

**Ожидаемый результат:**
```
Текст: Встреча запланирована на 15 марта 2024
Найденные даты: ['марта']

Текст: События: 3 января и 25 декабря
Найденные даты: ['января', 'декабря']

Текст: С 1 июня по 30 августа 2024
Найденные даты: ['июня', 'августа']
```

## Часть 2: Анализ настроения (Sentiment Analysis)

### Что такое Sentiment Analysis?

**Sentiment Analysis** — это определение эмоциональной окраски текста: позитивное (положительное) или негативное (отрицательное) отношение автора.

Пример:
```
"Отличный фильм! Очень понравился!" → POSITIVE (Позитивное)
"Скучный и нудный фильм" → NEGATIVE (Негативное)
"Фильм вышел в кинотеатрах" → NEUTRAL (Нейтральное)
```

### Классы настроения

| Класс | Описание | Примеры фраз |
|-------|---------|------------|
| POSITIVE | Положительное отношение | Отлично, люблю, восхищение, спасибо |
| NEGATIVE | Отрицательное отношение | Плохо, ненавижу, разочарован, ужас |
| NEUTRAL | Нейтральное отношение | Сообщение факта, без эмоций |

### Подходы к Sentiment Analysis

#### 1. Лексиконный подход (Lexicon-based)

Использование словарей с размеченными словами:

```python
def sentiment_lexicon(text):
    """
    Простой анализ настроения на основе словаря
    """
    positive_words = {
        'хорошо', 'отлично', 'прекрасно', 'спасибо', 'люблю',
        'восхищение', 'замечательно', 'классный', 'super', 'awesome'
    }
    
    negative_words = {
        'плохо', 'ужас', 'ненавижу', 'разочарован', 'скучно',
        'ужасно', 'кошмар', 'хуже', 'bad', 'terrible', 'awful'
    }
    
    words = text.lower().split()
    
    positive_count = sum(1 for w in words if w in positive_words)
    negative_count = sum(1 for w in words if w in negative_words)
    
    if positive_count > negative_count:
        return 'POSITIVE'
    elif negative_count > positive_count:
        return 'NEGATIVE'
    else:
        return 'NEUTRAL'

# Тестирование
texts = [
    "Отличный фильм! Очень понравился!",
    "Скучный и плохой фильм",
    "Фильм вышел в кинотеатрах в пятницу"
]

for text in texts:
    sentiment = sentiment_lexicon(text)
    print(f"Текст: {text}")
    print(f"Настроение: {sentiment}\n")
```

**Ожидаемый результат:**
```
Текст: Отличный фильм! Очень понравился!
Настроение: POSITIVE

Текст: Скучный и плохой фильм
Настроение: NEGATIVE

Текст: Фильм вышел в кинотеатрах в пятницу
Настроение: NEUTRAL
```

**Достоинства:** быстро, понятно, не требует обучения

**Недостатки:** не учитывает контекст, не работает с иронией

#### 2. Статистический подход (Machine Learning)

Использование классификаторов на основе Naive Bayes или SVM:

```python
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.naive_bayes import MultinomialNB
from sklearn.pipeline import Pipeline

# Обучающие данные (примеры)
train_texts = [
    "Отличный фильм, очень понравился",
    "Прекрасная кинолента, рекомендую всем",
    "Скучный и нудный фильм",
    "Ужасно плохо, не рекомендую",
    "Фильм интересный, хорошо снят",
    "Кошмарная история и актёры"
]

train_labels = [
    'POSITIVE',
    'POSITIVE',
    'NEGATIVE',
    'NEGATIVE',
    'POSITIVE',
    'NEGATIVE'
]

# Создание конвейера
pipeline = Pipeline([
    ('tfidf', TfidfVectorizer(lowercase=True)),
    ('classifier', MultinomialNB())
])

# Обучение
pipeline.fit(train_texts, train_labels)

# Предсказание
test_texts = [
    "Классный фильм, понравился",
    "Плохой фильм, не смотрите"
]

predictions = pipeline.predict(test_texts)
for text, pred in zip(test_texts, predictions):
    print(f"'{text}' → {pred}")

# Вывод:
# 'Классный фильм, понравился' → POSITIVE
# 'Плохой фильм, не смотрите' → NEGATIVE
```

**Достоинства:** учитывает контекст, лучше чем лексиконный подход

**Недостатки:** требует обучающих данных

### Практическое задание 2: Анализ настроения с учётом отрицания

```python
def sentiment_with_negation(text):
    """
    Анализ настроения с учётом отрицания
    Отрицание изменяет знак: "не хорошо" → отрицательно
    """
    positive_words = {'хорошо', 'отлично', 'классно', 'замечательно', 'люблю'}
    negative_words = {'плохо', 'ужасно', 'скучно', 'ненавижу'}
    negations = {'не', 'нет', 'никогда', 'ничего'}
    
    words = text.lower().split()
    sentiment_score = 0
    
    for i, word in enumerate(words):
        # Проверяем наличие отрицания перед словом
        has_negation = i > 0 and words[i-1] in negations
        
        if word in positive_words:
            sentiment_score += -1 if has_negation else 1
        elif word in negative_words:
            sentiment_score += 1 if has_negation else -1
    
    if sentiment_score > 0:
        return 'POSITIVE', sentiment_score
    elif sentiment_score < 0:
        return 'NEGATIVE', sentiment_score
    else:
        return 'NEUTRAL', sentiment_score

# Примеры
examples = [
    "Отличный фильм",
    "Не отличный фильм",
    "Плохой фильм",
    "Не плохой фильм"
]

for text in examples:
    sentiment, score = sentiment_with_negation(text)
    print(f"'{text}' → {sentiment} (score: {score})")

# Вывод:
# 'Отличный фильм' → POSITIVE (score: 1)
# 'Не отличный фильм' → NEGATIVE (score: -1)
# 'Плохой фильм' → NEGATIVE (score: -1)
# 'Не плохой фильм' → POSITIVE (score: 1)
```

## Часть 3: Интеграция NER и Sentiment Analysis

Часто эти две задачи решаются вместе для извлечения мнений об определённых объектах.

```python
def analyze_opinions_about_entities(text):
    """
    Определяет мнения о конкретных объектах
    Пример: "Apple выпустила классный iPhone, но дорогой Samsung"
    """
    import re
    
    entities = {
        'Apple': 'COMPANY',
        'Samsung': 'COMPANY',
        'iPhone': 'PRODUCT',
        'фильм': 'PRODUCT'
    }
    
    positive_words = {'классный', 'отличный', 'прекрасный', 'люблю'}
    negative_words = {'плохой', 'дорогой', 'ненавижу', 'ужасный'}
    
    results = []
    
    for entity, entity_type in entities.items():
        # Ищем слова около сущности
        pattern = rf'(\w+)\s+{entity}|{entity}\s+(\w+)'
        matches = re.finditer(pattern, text, re.IGNORECASE)
        
        for match in matches:
            nearby_word = (match.group(1) or match.group(2)).lower()
            
            if nearby_word in positive_words:
                sentiment = 'POSITIVE'
            elif nearby_word in negative_words:
                sentiment = 'NEGATIVE'
            else:
                sentiment = 'NEUTRAL'
            
            results.append({
                'entity': entity,
                'type': entity_type,
                'sentiment': sentiment,
                'context': nearby_word
            })
    
    return results

# Пример использования
text = "Apple выпустила отличный iPhone, но дорогой Samsung"
opinions = analyze_opinions_about_entities(text)

for opinion in opinions:
    print(f"{opinion['entity']} ({opinion['type']}): {opinion['sentiment']}")
    print(f"  Контекст: '{opinion['context']}'")
```

## Ключевые концепции и определения

| Термин | Определение |
|--------|------------|
| Named Entity Recognition | Задача выделения и классификации именных единиц в тексте |
| Entity | Именная единица — объект, имеющий имя в тексте |

| CRF (Conditional Random Fields) | Вероятностная модель для последовательной классификации |
| Transformer | Архитектура нейросети, основанная на механизме внимания |
