# Синтаксическая связь и синтаксический анализ: Теория конституции и грамматики связи

## 1. Введение

**Синтаксический анализ** — это процесс определения грамматической структуры предложения на основе его компонентов и связей между ними. Это один из ключевых этапов обработки естественного языка, который позволяет компьютерам понимать, как слова организованы в предложении и как они влияют друг на друга.

В этом материале мы рассмотрим две основные теории синтаксического анализа:

1. **Теория конституции (Phrase Structure Grammar)** — описывает, как слова группируются в синтаксические единицы
2. **Грамматика связи (Dependency Grammar)** — описывает отношения между отдельными словами

## 2. Основные понятия

### 2.1 Синтаксическая единица (Constituent)

Синтаксическая единица — это группа слов, которые функционируют как одно целое в предложении.

**Примеры:**
- Существительная фраза (NP): *большой красный дом*
- Глагольная фраза (VP): *читает книгу*
- Предложная фраза (PP): *в парке*

### 2.2 Синтаксическая связь (Syntactic Relation)

Синтаксическая связь — это отношение между двумя словами, показывающее, как одно слово грамматически связано с другим.

**Примеры типов связей:**
- **Субъект (nsubj)**: подлежащее глагола
- **Объект (obj)**: прямое дополнение глагола
- **Предикат (cop)**: глагол-связка
- **Модификатор (amod)**: прилагательное, модифицирующее существительное

### 2.3 Синтаксическое дерево (Parse Tree)

Синтаксическое дерево — это древовидная структура, которая показывает иерархическую организацию синтаксических единиц в предложении.

**Компоненты дерева:**
- **Узлы (Nodes)**: синтаксические категории (S, NP, VP и т.д.)
- **Листья (Leaves)**: слова предложения
- **Рёбра (Edges)**: отношения между узлами

## 3. Теория конституции (Phrase Structure Grammar)

### 3.1 Основные принципы

Теория конституции основана на предположении, что предложение состоит из иерархически организованных синтаксических единиц. Каждая единица состоит из более мелких единиц.

**Грамматические правила (Production Rules):**

```
S    → NP VP
NP   → Det N | Det Adj N | Det Adj Adj N
VP   → V NP | V NP PP
PP   → P NP
Det  → the | a | an
N    → dog | cat | house | book
V    → chases | eats | reads
Adj  → big | brown | old
P    → in | on | under
```

Где:
- `S` — предложение (Sentence)
- `NP` — именная фраза (Noun Phrase)
- `VP` — глагольная фраза (Verb Phrase)
- `PP` — предложная фраза (Prepositional Phrase)
- `Det` — детерминатив (Determiner)
- `N` — существительное (Noun)
- `V` — глагол (Verb)
- `Adj` — прилагательное (Adjective)
- `P` — предлог (Preposition)

### 3.2 Пример синтаксического дерева (конституция)

**Предложение:** *The big cat chases a brown mouse*

```
                    S
                   / \
                  /   \
                NP     VP
               /|\      /\
              / | \    /  \
            Det Adj  N  V  NP
             |   |  |  |  /|\
            the big cat chases a brown mouse
                             |   |   |   |
                            Det Adj  N   ...
```

### 3.3 Восходящий анализ (Bottom-Up Parsing)

При восходящем анализе начинаем со слов и строим дерево снизу вверх.

**Алгоритм Shift-Reduce:**

1. Сдвиг (Shift) — добавить слово в стек
2. Свёртка (Reduce) — заменить элементы стека согласно грамматическому правилу

**Пример для предложения "cat chases mouse":**

```
Шаг 1: Shift "cat"          → [cat]
Шаг 2: Reduce N → "cat"     → [N]
Шаг 3: Reduce NP → Det N    → [NP]
Шаг 4: Shift "chases"       → [NP, chases]
Шаг 5: Reduce V → "chases"  → [NP, V]
Шаг 6: Shift "mouse"        → [NP, V, mouse]
Шаг 7: Reduce N → "mouse"   → [NP, V, N]
Шаг 8: Reduce NP → N        → [NP, V, NP]
Шаг 9: Reduce VP → V NP     → [NP, VP]
Шаг 10: Reduce S → NP VP    → [S]
```

### 3.4 Нисходящий анализ (Top-Down Parsing)

При нисходящем анализе начинаем с корня (S) и пытаемся развернуть правила до тех пор, пока не получим слова.

**Алгоритм LL(1) Parsing:**

1. Начнём с S
2. Применяем правило: S → NP VP
3. Для NP ищем подходящее правило
4. Продолжаем, пока не покроем все слова

## 4. Грамматика связи (Dependency Grammar)

### 4.1 Основные принципы

Грамматика связи предполагает, что синтаксическая структура состоит из **двоичных отношений** между словами. Каждое слово имеет ровно одного **главного** (head) слова, кроме корня предложения.

**Основные характеристики:**
- Каждое слово связано с одним главным словом
- Связи не пересекаются (это называется **непроективностью**)
- Граф связей является ориентированным деревом

### 4.2 Типы синтаксических отношений

| Обозначение | Название | Описание | Пример |
|-------------|----------|---------|---------|
| nsubj | nominal subject | подлежащее | *dog* barks |
| obj | direct object | прямое дополнение | sees *cat* |
| iobj | indirect object | косвенное дополнение | gives *him* a book |
| amod | adjectival modifier | прилагательное-модификатор | *big* house |
| nmod | nominal modifier | модификатор существительного | house *in* the city |
| det | determiner | детерминатив | *the* cat |
| case | case marker | показатель падежа | book *of* John |
| cop | copula | глагол-связка | is *tired* |
| nmod:tmod | temporal modifier | временной модификатор | runs *every day* |

### 4.3 Граф зависимостей (Dependency Graph)

Граф зависимостей показывает отношения между словами в виде ориентированного дерева.

**Пример для предложения:** *The cat chases a mouse in the garden*

```
          chases (ROOT)
           /    |    \
         /      |      \
       cat      mouse    garden
      / |       / |      / |
    the  (det) a (det) the (det)
              (det)    (case) in
    
Связи:
- chases → cat (nsubj)
- chases → mouse (obj)
- chases → garden (nmod)
- cat → the (det)
- mouse → a (det)
- garden → the (det)
- garden → in (case)
```

### 4.4 Алгоритм синтаксического анализа зависимостей

#### Переходной алгоритм (Transition-Based Parsing)

Этот алгоритм использует стек и буфер для постепенного построения графа зависимостей.

**Компоненты:**
- **Стек (Stack)**: слова, которые мы рассматриваем
- **Буфер (Buffer)**: оставшиеся слова для обработки
- **Зависимости (Dependencies)**: список построенных связей

**Переходы (Transitions):**

1. **Shift** — переместить слово из буфера в стек
2. **Left-Arc** — создать связь: верхушка буфера ← верхушка стека
3. **Right-Arc** — создать связь: верхушка стека → верхушка буфера

**Пример для предложения "The cat chases":**

```
Начальное состояние:
Стек: []
Буфер: [The, cat, chases]
Зависимости: []

Шаг 1: Shift
Стек: [The]
Буфер: [cat, chases]

Шаг 2: Shift
Стек: [The, cat]
Буфер: [chases]

Шаг 3: Left-Arc (cat → The)
Стек: [cat]
Буфер: [chases]
Зависимости: [(cat, det, The)]

Шаг 4: Shift
Стек: [cat, chases]
Буфер: []

Шаг 5: Right-Arc (chases → cat)
Стек: [chases]
Буфер: []
Зависимости: [(chases, nsubj, cat), (cat, det, The)]
```

#### Граф-базированный алгоритм (Graph-Based Parsing)

Этот подход ищет максимальное остовное дерево в полносвязном графе, где рёбра имеют веса.

**Алгоритм Chu-Liu-Edmonds:**

1. Для каждого слова найти входящее ребро с максимальным весом
2. Если граф ациклический — готово
3. Если есть цикл — обнаружить и сжать его в один узел
4. Рекурсивно решить для сжатого графа
5. Развернуть результат

## 5. Сравнение конституции и связей

| Аспект | Конституция | Связь |
|--------|-----------|--------|
| **Структура** | Иерархические фразы | Двоичные отношения |
| **Граф** | Дерево с внутренними узлами | Дерево только с листовыми узлами |
| **Сложность** | O(n³) для CYK | O(n²) или O(n³) для зависимостей |
| **Применение** | Анализ структуры, компиляция | NLP задачи, синтез речи |
| **Неоднозначность** | Несколько деревьев возможны | Обычно одно дерево |

## 6. Практические примеры на Python

### 6.1 Простой анализатор зависимостей

```python
class DependencyNode:
    def __init__(self, word, pos, index):
        self.word = word
        self.pos = pos  # Part of Speech
        self.index = index
        self.head = None
        self.deprel = None
        self.children = []

class SimpleDependencyParser:
    def __init__(self):
        # Простые правила для определения главного слова
        self.rules = {
            ('V', 'N'): 0,      # глагол — главное слово для существительного
            ('N', 'Adj'): 0,    # существительное — главное для прилагательного
            ('V', 'Adv'): 0,    # глагол — главное для наречия
            ('N', 'Det'): 0,    # существительное — главное для детерминатива
        }
    
    def parse(self, words, pos_tags):
        """
        Простой алгоритм определения зависимостей
        
        Args:
            words: список слов
            pos_tags: список частей речи
        
        Returns:
            список узлов с установленными зависимостями
        """
        nodes = [DependencyNode(word, pos, i) 
                 for i, (word, pos) in enumerate(zip(words, pos_tags))]
        
        # Первое слово (обычно глагол) — корень
        root = nodes[0]
        root.head = -1
        root.deprel = 'ROOT'
        
        # Для остальных слов найти главное слово
        for i in range(1, len(nodes)):
            node = nodes[i]
            
            # Ищем ближайшее слово слева, которое может быть главным
            for j in range(i - 1, -1, -1):
                candidate = nodes[j]
                rule_key = (candidate.pos, node.pos)
                
                if rule_key in self.rules:
                    node.head = j
                    node.deprel = self._get_deprel(candidate.pos, node.pos)
                    candidate.children.append(node)
                    break
            
            # Если не нашли главное слово, связываем с первым словом
            if node.head is None:
                node.head = 0
                node.deprel = 'nmod'
                root.children.append(node)
        
        return nodes
    
    def _get_deprel(self, head_pos, dep_pos):
        """Определить тип зависимости на основе частей речи"""
        if head_pos == 'V':
            if dep_pos == 'N':
                return 'nsubj'
            elif dep_pos == 'Adv':
                return 'advmod'
        elif head_pos == 'N':
            if dep_pos == 'Adj':
                return 'amod'
            elif dep_pos == 'Det':
                return 'det'
        return 'nmod'
    
    def visualize(self, nodes):
        """Визуализация дерева зависимостей"""
        print("\nДерево зависимостей:")
        for node in nodes:
            if node.head == -1:
                print(f"  {node.word} ({node.pos}) — ROOT")
            else:
                head_word = nodes[node.head].word
                print(f"  {node.word} ({node.pos}) —[{node.deprel}]→ {head_word}")

# Пример использования
parser = SimpleDependencyParser()

# Пример 1: английское предложение
words_en = ["The", "big", "cat", "chases", "a", "mouse"]
pos_en = ["Det", "Adj", "N", "V", "Det", "N"]

nodes = parser.parse(words_en, pos_en)
parser.visualize(nodes)

# Пример 2: русское предложение
words_ru = ["Большая", "рыжая", "кошка", "ловит", "маленькую", "мышку"]
pos_ru = ["Adj", "Adj", "N", "V", "Adj", "N"]

nodes_ru = parser.parse(words_ru, pos_ru)
parser.visualize(nodes_ru)
```

### 6.2 Проверка проективности

```python
def is_projective(nodes):
    """
    Проверить, является ли граф зависимостей проективным
    
    Граф проективен, если дуги не пересекаются
    
    Args:
        nodes: список узлов с установленными зависимостями
    
    Returns:
        True если проективен, False иначе
    """
    for i, node1 in enumerate(nodes):
        if node1.head == -1:
            continue
        
        for j, node2 in enumerate(nodes):
            if node2.head == -1 or node1.head == -1:
                continue
            
            # Проверить пересечение дуг
            head1, dep1 = node1.head, i
            head2, dep2 = node2.head, j
            
            # Убедиться, что head1 < dep1 и head2 < dep2
            if head1 > dep1:
                head1, dep1 = dep1, head1
            if head2 > dep2:
                head2, dep2 = dep2, head2
            
            # Проверить пересечение
            if (head1 < head2 < dep1 < dep2) or (head2 < head1 < dep2 < dep1):
                return False
    
    return True

# Пример использования
nodes = parser.parse(words_en, pos_en)
if is_projective(nodes):
    print("Граф зависимостей проективен")
else:
    print("Граф зависимостей не проективен")
```

### 6.3 Вычисление метрик оценки

```python
def calculate_metrics(predicted_nodes, gold_nodes):
    """
    Вычислить метрики для оценки качества синтаксического анализа
    
    Args:
        predicted_nodes: предсказанные узлы с зависимостями
        gold_nodes: эталонные узлы с зависимостями
    
    Returns:
        словарь с метриками UAS и LAS
    """
    total = len(predicted_nodes)
    uas_correct = 0
    las_correct = 0
    
    for pred, gold in zip(predicted_nodes, gold_nodes):
        # UAS (Unlabeled Attachment Score) — правильно ли определено главное слово
        if pred.head == gold.head:
            uas_correct += 1
            
            # LAS (Labeled Attachment Score) — правильны ли главное слово и тип связи
            if pred.deprel == gold.deprel:
                las_correct += 1
    
    uas = (uas_correct / total) * 100 if total > 0 else 0
    las = (las_correct / total) * 100 if total > 0 else 0
    
    return {
        'UAS': round(uas, 2),
        'LAS': round(las, 2),
        'total': total,
        'uas_correct': uas_correct,
        'las_correct': las_correct
    }

# Пример использования
gold_nodes = parser.parse(words_en, pos_en)
predicted_nodes = parser.parse(words_en, pos_en)

metrics = calculate_metrics(predicted_nodes, gold_nodes)
print(f"\nМетрики оценки:")
print(f"  UAS (правильность главного): {metrics['UAS']}%")
print(f"  LAS (правильность связей): {metrics['LAS']}%")
```

## 8. Ключевые термины (Глоссарий)

| Термин | Объяснение |
|--------|-----------|
| **Синтаксический анализ** | Процесс определения грамматической структуры предложения |
| **Конституция** | Группа слов, функционирующих как одно целое |
| **Зависимость** | Отношение между главным и зависимым словом |
| **Синтаксическое дерево** | Иерархическая структура синтаксических единиц |
| **Граф зависимостей** | Дерево отношений между словами |
| **Head (главное слово)** | Слово, от которого зависит другое слово |
| **Deprel (тип зависимости)** | Название грамматического отношения |
| **Проективность** | Свойство, при котором дуги в графе не пересекаются |
| **Нисходящий анализ** | Анализ от корня к листьям |
| **Восходящий анализ** | Анализ от листьев к корню |
| **UAS** | Unlabeled Attachment Score — метрика без учёта типов зависимостей |
| **LAS** | Labeled Attachment Score — метрика с учётом типов зависимостей |
