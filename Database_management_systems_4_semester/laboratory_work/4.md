## Теоретическая часть

### 1. Оператор WHERE

Оператор `WHERE` используется для фильтрации строк в запросе. Только те записи, для которых условие истинно, попадают в результат.

**Синтаксис:**
```sql
SELECT column1, column2, ...
FROM table_name
WHERE condition;
```

### 2. Операторы сравнения

| Оператор    | Описание         | Пример                               |
|-------------|------------------|--------------------------------------|
| `=`         | Равно            | `WHERE language = 'Russian'`         |
| `<>` / `!=` | Не равно         | `WHERE type <> 'Dialect'`            |
| `>`         | Больше           | `WHERE year_founded > 1900`          |
| `<`         | Меньше           | `WHERE num_speakers < 1000000`       |
| `>=`        | Больше или равно | `WHERE rating >= 4`                  |
| `<=`        | Меньше или равно | `WHERE difficulty <= 3`              |

### 3. Логические операторы

- **AND** — оба условия должны быть истинными
- **OR** — хотя бы одно условие истинно
- **NOT** — инвертирует условие

### 4. Оператор LIKE

Используется для поиска по шаблону в строковых полях.

| Символ | Значение                        | Пример                        |
|--------|---------------------------------|-------------------------------|
| `%`    | Любое количество любых символов | `LIKE 'Sla%'` → Slavic        |
| `_`    | Ровно один любой символ         | `LIKE 'Rus___n'` → Russian    |

### 5. Оператор BETWEEN

Фильтрует значения в заданном диапазоне (включительно).

```sql
WHERE year_founded BETWEEN 1800 AND 1950
```

### 6. Оператор IN

Проверяет принадлежность значения списку.

```sql
WHERE family IN ('Indo-European', 'Turkic', 'Semitic')
```

### 7. Операторы IS NULL / IS NOT NULL

Проверяют наличие или отсутствие значения в поле.

```sql
WHERE writing_system IS NULL
WHERE writing_system IS NOT NULL
```

## Практическая часть

### Создание учебной базы данных

Для лабораторной работы используется база данных **«Языки мира»**, содержащая сведения о языках, диалектах и лингвистических семьях.

**Создание таблиц:**

```sql
-- Таблица языковых семей
CREATE TABLE language_families (
    family_id       INTEGER PRIMARY KEY,
    name            VARCHAR(100) NOT NULL,
    region          VARCHAR(100),
    num_languages   INTEGER
);

-- Таблица языков
CREATE TABLE languages (
    language_id     INTEGER PRIMARY KEY,
    name            VARCHAR(100) NOT NULL,
    family          VARCHAR(100),
    family_id       INTEGER REFERENCES language_families(family_id),
    standard_year   INTEGER,
    num_speakers    BIGINT,
    writing_system  VARCHAR(50),
    status          VARCHAR(50),
    difficulty      INTEGER  -- от 1 (простой) до 5 (очень сложный)
);

-- Таблица диалектов
CREATE TABLE dialects (
    dialect_id    INTEGER PRIMARY KEY,
    name          VARCHAR(100) NOT NULL,
    language_id   INTEGER REFERENCES languages(language_id),
    region        VARCHAR(100),
    is_active     BOOLEAN
);
```

### Задания

#### Задание 1. Фильтрация с оператором WHERE и операторами сравнения

**1.1.** Вывести все живые языки.

```sql
SELECT name, family, num_speakers
FROM languages
WHERE status = 'Living';
```

**1.2.** Вывести языки со сложностью выше среднего (больше 3).

```sql
SELECT name, family, difficulty
FROM languages
WHERE difficulty > 3
ORDER BY difficulty DESC;
```

**1.3.** Вывести языки с числом носителей более 100 миллионов.

```sql
SELECT name, num_speakers, status
FROM languages
WHERE num_speakers > 100000000
ORDER BY num_speakers DESC;
```

---

#### Задание 2. Логические операторы AND, OR, NOT

**2.1.** Вывести живые языки со сложностью 4 или 5.

```sql
SELECT name, family, difficulty, status
FROM languages
WHERE status = 'Living'
  AND difficulty >= 4;
```

**2.2.** Вывести языки индоевропейской или тюркской семьи.

```sql
SELECT name, family, num_speakers
FROM languages
WHERE family = 'Indo-European'
   OR family = 'Turkic';
```

**2.3.** Вывести все языки, кроме мёртвых и имеющих сложность 5.

```sql
SELECT name, family, difficulty, status
FROM languages
WHERE NOT (status = 'Dead' OR difficulty = 5);
```

#### Задание 3. Оператор LIKE — поиск по шаблону

**3.1.** Найти все языки, название которых начинается на «R».

```sql
SELECT name, family
FROM languages
WHERE name LIKE 'R%';
```

**3.2.** Найти диалекты, в названии которых есть суффикс «ern».

```sql
SELECT name, region
FROM dialects
WHERE name LIKE '%ern';
```

**3.3.** Найти языки, в названии которых ровно 6 букв (используя `_`).

```sql
SELECT name, family
FROM languages

```

#### Задание 4. Оператор BETWEEN — диапазоны

**4.1.** Вывести языки, у которых стандарт был принят между 1800 и 1960 годами.

```sql
SELECT name, standard_year, family
FROM languages
WHERE standard_year BETWEEN 1800 AND 1960
ORDER BY standard_year;
```

**4.2.** Вывести языки со средней сложностью (2–4).

```sql
SELECT name, difficulty, family
FROM languages
WHERE difficulty BETWEEN 2 AND 4
ORDER BY difficulty;
```

#### Задание 5. Оператор IN — список значений

**5.1.** Вывести языки из нескольких конкретных семей.

```sql
SELECT name, family, num_speakers
FROM languages
WHERE family IN ('Turkic', 'Uralic', 'Semitic');
```

**5.2.** Вывести диалекты, привязанные к русскому или арабскому языку (id 1 и 3).

```sql
SELECT name, region, is_active
FROM dialects
WHERE language_id IN (1, 3);
```

#### Задание 6. Операторы IS NULL / IS NOT NULL

**6.1.** Вывести языки без установленного года стандартизации.

```sql
SELECT name, status, family
FROM languages
WHERE standard_year IS NULL;
```

**6.2.** Вывести языки, у которых письменность не указана.

```sql
SELECT name, status
FROM languages
WHERE writing_system IS NULL;
```

**6.3.** Вывести все языки с известной письменностью.

```sql
SELECT name, writing_system, family
FROM languages
WHERE writing_system IS NOT NULL
ORDER BY writing_system;
```

---

#### Задание 7. Комбинированные запросы (повышенная сложность)

**7.1.** Вывести живые языки индоевропейской семьи с более чем 50 миллионами носителей.

```sql
SELECT name, num_speakers, writing_system
FROM languages
WHERE status = 'Living'
  AND family = 'Indo-European'
  AND num_speakers > 50000000
ORDER BY num_speakers DESC;
```

**7.2.** Найти все активные диалекты языков со сложностью 3 и выше.

```sql
SELECT d.name       AS dialect,
       l.name       AS language,
       l.difficulty,
       d.region
FROM dialects d
JOIN languages l ON d.language_id = l.language_id
WHERE d.is_active = TRUE
  AND l.difficulty >= 3
ORDER BY l.difficulty DESC;
```

**7.3.** Найти языки, стандартизированные после 1900 года, с числом носителей от 10 до 500 миллионов, у которых указана письменность.

```sql
SELECT name,
       standard_year,
       num_speakers,
       writing_system
FROM languages
WHERE standard_year > 1900
  AND num_speakers BETWEEN 10000000 AND 500000000
  AND writing_system IS NOT NULL
ORDER BY num_speakers DESC;
```
