## Теоретическая часть

### Что такое объединение таблиц?

Реляционные базы данных хранят данные в отдельных таблицах, связанных между собой ключами. **Объединение (JOIN)** — это операция, позволяющая получить данные из двух и более таблиц одновременно на основе общего поля.

Синтаксис общего вида:

```sql
SELECT столбцы
FROM таблица1
[ТИП_JOIN] JOIN таблица2
ON таблица1.ключ = таблица2.ключ;
```


### Виды объединений

#### 1. INNER JOIN (Внутреннее объединение)

Возвращает только те строки, для которых найдено совпадение в **обеих** таблицах.

```
Таблица A       Таблица B         INNER JOIN
┌───┬────┐     ┌───┬────┐        ┌───┬────┐
│ 1 │ A  │     │ 1 │ X  │   →    │ 1 │A, X│
│ 2 │ B  │     │ 3 │ Y  │        │ 3 │C, Y│
│ 3 │ C  │     │ 4 │ Z  │        └───┴────┘
└───┴────┘     └───┴────┘
```

```sql
SELECT *
FROM таблица1
INNER JOIN таблица2
ON таблица1.id = таблица2.id;
```


#### 2. LEFT JOIN (Левое внешнее объединение)

Возвращает **все строки** из левой таблицы и совпадающие строки из правой. Если совпадения нет — `NULL`.

```
Таблица A       Таблица B         LEFT JOIN
┌───┬────┐     ┌───┬────┐        ┌───┬────────┐
│ 1 │ A  │     │ 1 │ X  │   →    │ 1 │ A, X   │
│ 2 │ B  │     │ 3 │ Y  │        │ 2 │ B, NULL│
│ 3 │ C  │     └───┴────┘        │ 3 │ C, Y   │
└───┴────┘                       └───┴────────┘
```

```sql
SELECT *
FROM таблица1
LEFT JOIN таблица2
ON таблица1.id = таблица2.id;
```


#### 3. RIGHT JOIN (Правое внешнее объединение)

Возвращает **все строки** из правой таблицы и совпадающие строки из левой. Если совпадения нет — `NULL`.

```sql
SELECT *
FROM таблица1
RIGHT JOIN таблица2
ON таблица1.id = таблица2.id;
```

---

#### 4. FULL OUTER JOIN (Полное внешнее объединение)

Возвращает **все строки** из обеих таблиц. Где нет совпадений — `NULL`.

```sql
SELECT *
FROM таблица1
FULL OUTER JOIN таблица2
ON таблица1.id = таблица2.id;
```



#### 5. CROSS JOIN (Перекрёстное объединение)

Декартово произведение: каждая строка из первой таблицы соединяется с каждой строкой из второй.

```sql
SELECT *
FROM таблица1
CROSS JOIN таблица2;
```


#### 6. SELF JOIN (Самообъединение)

Таблица объединяется сама с собой (используется псевдоним).

```sql
SELECT a.слово AS слово1, b.слово AS слово2
FROM слова a, слова b
WHERE a.корень = b.корень AND a.id <> b.id;
```


## Практическая часть

### Предметная область

Лингвистическая база данных для анализа текстов. Используется для задач компьютерной лингвистики: анализ частотности слов, работа с корпусами, словарями и морфологическими данными.


### Схема базы данных

```
┌─────────────────┐       ┌──────────────────┐
│     words       │       │   word_forms     │
├─────────────────┤       ├──────────────────┤
│ word_id  PK     │──┐    │ form_id  PK      │
│ lemma           │  └───>│ word_id  FK      │
│ pos_id   FK     │       │ form             │
│ frequency       │       │ case_id  FK      │
└─────────────────┘       └──────────────────┘
        │
        │  ┌──────────────────┐
        │  │  parts_of_speech │
        └─>├──────────────────┤
           │ pos_id   PK      │
           │ pos_name         │
           │ description      │
           └──────────────────┘

┌─────────────────┐       ┌──────────────────┐
│    texts        │       │   text_words     │
├─────────────────┤       ├──────────────────┤
│ text_id  PK     │──┐    │ id       PK      │
│ title           │  └───>│ text_id  FK      │
│ language        │       │ word_id  FK      │
│ author          │       │ position         │
└─────────────────┘       └──────────────────┘
```



### Создание таблиц

```sql
-- Части речи
CREATE TABLE parts_of_speech (
    pos_id   SERIAL PRIMARY KEY,
    pos_name VARCHAR(50) NOT NULL,
    description TEXT
);

-- Словарные единицы
CREATE TABLE words (
    word_id   SERIAL PRIMARY KEY,
    lemma     VARCHAR(100) NOT NULL,
    pos_id    INT REFERENCES parts_of_speech(pos_id),
    frequency INT DEFAULT 0
);

-- Словоформы
CREATE TABLE word_forms (
    form_id  SERIAL PRIMARY KEY,
    word_id  INT REFERENCES words(word_id),
    form     VARCHAR(100) NOT NULL,
    case_id  INT
);

-- Тексты
CREATE TABLE texts (
    text_id  SERIAL PRIMARY KEY,
    title    VARCHAR(200),
    language VARCHAR(50),
    author   VARCHAR(100)
);

-- Слова в текстах
CREATE TABLE text_words (
    id       SERIAL PRIMARY KEY,
    text_id  INT REFERENCES texts(text_id),
    word_id  INT REFERENCES words(word_id),
    position INT
);
```

## Задания



### Задание 1 — INNER JOIN

**Условие:** Вывести все слова вместе с названием их части речи.

```sql
SELECT
    w.lemma         AS слово,
    p.pos_name      AS часть_речи,
    w.frequency     AS частота
FROM words w
INNER JOIN parts_of_speech p
    ON w.pos_id = p.pos_id
ORDER BY w.frequency DESC;
```

**Ожидаемый результат:**

### Задание 3 — Многотабличное объединение

**Условие:** Вывести, в каких текстах встречается каждое слово, с указанием части речи.

```sql
SELECT
    t.title         AS текст,
    w.lemma         AS слово,
    p.pos_name      AS часть_речи,
    tw.position     AS позиция
FROM text_words tw
INNER JOIN words w
    ON tw.word_id = w.word_id
INNER JOIN texts t
    ON tw.text_id = t.text_id
INNER JOIN parts_of_speech p
    ON w.pos_id = p.pos_id
ORDER BY t.title, tw.position;
```



### Задание 4 — LEFT JOIN с фильтрацией

**Условие:** Найти тексты, в которых **не встречается** ни одного существительного.

```sql
SELECT DISTINCT
    t.title     AS текст,
    t.language  AS язык
FROM texts t
LEFT JOIN text_words tw
    ON t.text_id = tw.text_id
LEFT JOIN words w
    ON tw.word_id = w.word_id
LEFT JOIN parts_of_speech p
    ON w.pos_id = p.pos_id AND p.pos_name = 'существительное'
WHERE p.pos_id IS NULL;
```



### Задание 5 — SELF JOIN

**Условие:** Найти пары слов одной части речи с одинаковой первой буквой леммы.

```sql
SELECT
    a.lemma     AS слово_1,
    b.lemma     AS слово_2,
    p.pos_name  AS часть_речи
FROM words a
INNER JOIN words b
    ON a.pos_id = b.pos_id
   AND LEFT(a.lemma, 1) = LEFT(b.lemma, 1)
   AND a.word_id < b.word_id
INNER JOIN parts_of_speech p
    ON a.pos_id = p.pos_id
ORDER BY p.pos_name;
```



### Задание 6 — Агрегация с JOIN

**Условие:** Подсчитать количество уникальных слов каждой части речи в каждом тексте.

```sql
SELECT
    t.title             AS текст,
    p.pos_name          AS часть_речи,
    COUNT(DISTINCT w.word_id) AS кол_во_слов
FROM texts t
INNER JOIN text_words tw
    ON t.text_id = tw.text_id
INNER JOIN words w
    ON tw.word_id = w.word_id
INNER JOIN parts_of_speech p
    ON w.pos_id = p.pos_id
GROUP BY t.title, p.pos_name
ORDER BY t.title, кол_во_слов DESC;
```

## Дополнительное задание (повышенная сложность)

Разработайте запрос, который возвращает **топ-3 наиболее частотных слова для каждой части речи**, встречающихся хотя бы в одном тексте на русском языке.

```sql
SELECT
    pos_name    AS часть_речи,
    lemma       AS слово,
    frequency   AS частота
FROM (
    SELECT
        p.pos_name,
        w.lemma,
        w.frequency,
        RANK() OVER (PARTITION BY p.pos_id ORDER BY w.frequency DESC) AS rnk
    FROM words w
    INNER JOIN parts_of_speech p
        ON w.pos_id = p.pos_id
    WHERE w.word_id IN (
        SELECT DISTINCT tw.word_id
        FROM text_words tw
        INNER JOIN texts t ON tw.text_id = t.text_id
        WHERE t.language = 'русский'
    )
) ranked
WHERE rnk <= 3
ORDER BY pos_name, frequency DESC;
```



## Итоги лабораторной работы

В ходе работы были изучены:

- **INNER JOIN** — выборка пересечения двух таблиц
- **LEFT / RIGHT JOIN** — сохранение всех строк одной из таблиц
- **FULL OUTER JOIN** — объединение всех строк обеих таблиц
- **CROSS JOIN** — декартово произведение
- **SELF JOIN** — самообъединение таблицы
- Многотабличные объединения (3 и более таблиц)
- Агрегация данных в связке с `JOIN`

Эти операции являются основой для работы с лингвистическими корпусами, словарными базами данных и любыми реляционными данными в области компьютерной лингвистики.
