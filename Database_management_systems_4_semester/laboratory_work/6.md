## Теоретическая часть

### Что такое агрегатные функции?

**Агрегатные функции** — это функции SQL, которые выполняют вычисления над набором значений и возвращают единственное результирующее значение. Они широко используются для статистического анализа данных и формирования сводных отчётов.

### Основные агрегатные функции

| Функция   | Описание                              | Пример                    |
|-----------|---------------------------------------|---------------------------|
| `COUNT()` | Подсчёт количества строк или значений | `COUNT(*), COUNT(column)` |
| `SUM()`   | Сумма числовых значений               | `SUM(salary)`             |
| `AVG()`   | Среднее арифметическое значений       | `AVG(age)`                |
| `MIN()`   | Минимальное значение                  | `MIN(price)`              |
| `MAX()`   | Максимальное значение                 | `MAX(score)`              |

### Синтаксис

```sql
SELECT агрегатная_функция(столбец)
FROM таблица
[WHERE условие]
[GROUP BY столбец(цы)]
[HAVING условие_для_групп]
[ORDER BY столбец];
```

### Ключевые конструкции

**GROUP BY** — группирует строки с одинаковыми значениями в указанных столбцах. Агрегатные функции применяются к каждой группе отдельно.

**HAVING** — фильтрует результаты после группировки (аналог `WHERE`, но для агрегатных функций).

> ⚠️ **Важно:** `WHERE` применяется **до** группировки, `HAVING` — **после**.

### Подготовка базы данных

Для выполнения лабораторной работы используется база данных, моделирующая **лингвистический корпус текстов** с информацией об авторах, текстах и лексике.

#### Создание таблиц

```sql
-- Таблица авторов
CREATE TABLE authors (
    author_id   INTEGER PRIMARY KEY,
    name        VARCHAR(100) NOT NULL,
    language    VARCHAR(50),   -- родной язык автора
    birth_year  INTEGER
);

-- Таблица текстов
CREATE TABLE texts (
    text_id     INTEGER PRIMARY KEY,
    author_id   INTEGER REFERENCES authors(author_id),
    title       VARCHAR(200),
    genre       VARCHAR(50),
    word_count  INTEGER,
    created_year INTEGER
);

-- Таблица лексических единиц
CREATE TABLE lexemes (
    lexeme_id   INTEGER PRIMARY KEY,
    text_id     INTEGER REFERENCES texts(text_id),
    word        VARCHAR(100),
    frequency   INTEGER,
    part_of_speech VARCHAR(30)
);
```

#### Задание 1. Функция COUNT — подсчёт количества

**Задача:** Подсчитать общее количество текстов в базе данных.

```sql
SELECT COUNT(*) AS total_texts
FROM texts;
```

**Ожидаемый результат:**

| total_texts |
|-------------|
| 12          |

**Задача:** Подсчитать количество текстов каждого жанра.

```sql
SELECT genre,
       COUNT(*) AS text_count
```

|---------|------------|

#### Задание 2. Функция SUM — суммирование

**Задача:** Вычислить суммарный объём текстов (в словах) для каждого автора.

```sql
SELECT a.name,
       SUM(t.word_count) AS total_words
```

#### Задание 3. Функции AVG, MIN, MAX

**Задача:** Определить среднее, минимальное и максимальное количество слов в текстах по каждому жанру.

```sql
SELECT genre,
       ROUND(AVG(word_count), 0) AS avg_words,
       MIN(word_count)           AS min_words,
       MAX(word_count)           AS max_words
FROM texts
GROUP BY genre
ORDER BY avg_words DESC;
```

**Ожидаемый результат:**

#### Задание 4. Конструкция HAVING — фильтрация групп

**Задача:** Найти авторов, суммарный объём текстов которых превышает 200 000 слов.

```sql
SELECT a.name,
       COUNT(t.text_id)          AS num_texts,
       SUM(t.word_count)         AS total_words
FROM authors a
JOIN texts t ON a.author_id = t.author_id
GROUP BY a.author_id, a.name
HAVING SUM(t.word_count) > 200000
ORDER BY total_words DESC;
```

**Ожидаемый результат:**

| name              | num_texts | total_words |
|-------------------|-----------|-------------|
| Лев Толстой       | 3         | 982 000     |
| Фёдор Достоевский | 3         | 805 000     |
| Николай Гоголь    | 2         | 215 000     |



#### Задание 5. Комплексный запрос

**Задача:** Для каждого автора вывести количество текстов, суммарную и среднюю частоту встречаемости лексем, а также самое частотное слово (по максимальной частоте). Отобрать только авторов, у которых более 2 лексем в базе.

```sql
SELECT a.name,
       COUNT(DISTINCT t.text_id)      AS num_texts,
       COUNT(l.lexeme_id)             AS num_lexemes,
       SUM(l.frequency)               AS total_frequency,
       ROUND(AVG(l.frequency), 1)     AS avg_frequency,
       MAX(l.frequency)               AS max_frequency
FROM authors a
JOIN texts   t ON a.author_id = t.author_id
JOIN lexemes l ON t.text_id   = l.text_id
GROUP BY a.author_id, a.name
HAVING COUNT(l.lexeme_id) > 2
ORDER BY total_frequency DESC;
```



#### Задание 6. COUNT DISTINCT — подсчёт уникальных значений

**Задача:** Подсчитать количество уникальных частей речи, зафиксированных в каждом тексте.

```sql
SELECT t.title,
       COUNT(DISTINCT l.part_of_speech) AS unique_pos_count
FROM texts t
JOIN lexemes l ON t.text_id = l.text_id
GROUP BY t.text_id, t.title
ORDER BY unique_pos_count DESC;
```


#### Задание 7*. Вложенный запрос с агрегатной функцией

**Задача:** Найти тексты, объём которых превышает средний объём по всем текстам базы.

```sql
SELECT title,
       genre,
       word_count
FROM texts
WHERE word_count > (
    SELECT AVG(word_count)
    FROM texts
)
ORDER BY word_count DESC;
```

## Краткие итоги

- Агрегатные функции (`COUNT`, `SUM`, `AVG`, `MIN`, `MAX`) обобщают множество строк в одно значение.
- `GROUP BY` разбивает результат на группы; агрегатные функции работают в рамках каждой группы.
- `HAVING` фильтрует группы по условию на агрегатную функцию; `WHERE` — строки до группировки.
- `COUNT(*)` считает все строки, `COUNT(col)` — строки без `NULL` в столбце `col`.
- `COUNT(DISTINCT col)` позволяет подсчитать уникальные значения.
