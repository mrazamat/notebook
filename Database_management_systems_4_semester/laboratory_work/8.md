## Теоретическая часть

### 1. Что такое выражение в SQL?

**Выражение** — это комбинация значений, столбцов, операторов и функций, которая вычисляется и возвращает результат. Выражения могут использоваться в секциях `SELECT`, `WHERE`, `ORDER BY`, `GROUP BY`, `HAVING`.

```sql
-- Простейшие примеры выражений
SELECT 2 + 2;                        -- арифметическое выражение → 4
SELECT 'Привет' || ', мир!';          -- строковое выражение → 'Привет, мир!'
SELECT 10 > 5;                        -- логическое выражение → TRUE
```


### 2. Арифметические выражения

Применяются к числовым данным. Поддерживают стандартные математические операции:

| Оператор | Назначение         | Пример   | Результат    |
|----------|--------------------|----------|--------------|
| `+`      | Сложение           | `3 + 5`  | `8`          |
| `-`      | Вычитание          | `10 - 4` | `6`          |
| `*`      | Умножение          | `6 * 7`  | `42`         |
| `/`      | Деление            | `15 / 4` | `3` (целое!) |
| `%`      | Остаток от деления | `15 % 4` | `3`          |

> ⚠️ **Важно:** При делении двух целых чисел результат будет целым. Чтобы получить дробное значение, приведите одно из чисел к типу `FLOAT`: `15.0 / 4` или `CAST(15 AS FLOAT) / 4`.


### 3. Строковые функции и выражения

Крайне важны для задач компьютерной лингвистики — обработки текстов, лемматизации, морфологического анализа.

| Функция                    | Описание                         | Пример                                    |
|----------------------------|----------------------------------|-------------------------------------------|
| `\|\|`                     | Конкатенация строк               | `'Привет' \|\| ' ' \|\| 'мир'`            |
| `CONCAT(s1, s2, ...)`      | Конкатенация (альтернатива)      | `CONCAT(имя, ' ', фамилия)`               |
| `LENGTH(s)`                | Длина строки в символах          | `LENGTH('лингвистика')` → `11`            |
| `UPPER(s)`                 | Перевод в верхний регистр        | `UPPER('слово')` → `'СЛОВО'`              |
| `LOWER(s)`                 | Перевод в нижний регистр         | `LOWER('СЛОВО')` → `'слово'`              |
| `TRIM(s)`                  | Удаление пробелов с обеих сторон | `TRIM('  текст  ')` → `'текст'`           |
| `LTRIM(s)` / `RTRIM(s)`    | Удаление пробелов слева/справа   | `LTRIM('  слово')`                        |
| `SUBSTRING(s, start, len)` | Извлечение подстроки             | `SUBSTRING('лингвист', 1, 5)` → `'лингв'` |
| `REPLACE(s, old, new)`     | Замена подстроки                 | `REPLACE('словоформа', 'слово', 'морфо')` |
| `POSITION(sub IN s)`       | Позиция вхождения подстроки      | `POSITION('ист' IN 'лингвист')` → `6`     |
| `REVERSE(s)`               | Переворот строки                 | `REVERSE('кот')` → `'ток'`                |

### 4. Логические выражения и операторы сравнения

| Оператор             | Описание                                                   |
|----------------------|------------------------------------------------------------|
| `=`                  | Равенство                                                  |
| `<>` или `!=`        | Неравенство                                                |
| `<`, `>`, `<=`, `>=` | Сравнение                                                  |
| `BETWEEN a AND b`    | Значение в диапазоне [a, b]                                |
| `IN (список)`        | Значение входит в список                                   |
| `NOT IN (список)`    | Значение не входит в список                                |
| `LIKE 'шаблон'`      | Сопоставление по шаблону (`%` — любые символы, `_` — один) |
| `IS NULL`            | Значение является NULL                                     |
| `IS NOT NULL`        | Значение не является NULL                                  |
| `AND`, `OR`, `NOT`   | Логические связки                                          |

### 5. Выражение CASE

Позволяет реализовать условную логику прямо внутри запроса:

```sql
-- Форма 1: CASE с условиями (поисковая)
CASE
    WHEN условие1 THEN результат1
    WHEN условие2 THEN результат2
    ELSE результат_по_умолчанию
END

-- Форма 2: CASE с равенствами (простая)
CASE столбец
    WHEN 'значение1' THEN результат1
    WHEN 'значение2' THEN результат2
    ELSE результат_по_умолчанию
END
```

---

### 6. Псевдонимы (Aliases)

Выражениям можно давать имена с помощью ключевого слова `AS`:

```sql
SELECT
    длина_текста / количество_слов AS средняя_длина_слова,
    UPPER(лемма)                   AS лемма_верхний_регистр
FROM словарь;
```

### Задание 1. Арифметические выражения

**Задача:** вычислить среднюю длину слова в каждом тексте, а также оценить «плотность» текста.

```sql
-- 1.1. Средняя длина слова в символах
SELECT
    название,
    автор,
    кол_слов,
    кол_символов,
    кол_символов / кол_слов          AS средняя_длина_целая,
    ROUND(кол_символов::FLOAT / кол_слов, 2) AS средняя_длина_точная
FROM тексты
ORDER BY средняя_длина_точная DESC;
```

```sql
-- 1.2. Приблизительный объём текста в условных страницах (250 слов на страницу)
SELECT
    название,
    автор,
    кол_слов,
    кол_слов / 250                   AS страниц_приблизительно,
    кол_слов % 250                   AS слов_на_последней_странице
FROM тексты
ORDER BY кол_слов DESC;
```

```sql
-- 1.3. Насколько текст длиннее среднего по корпусу?
SELECT
    название,
    автор,
    кол_слов,
    ROUND(кол_слов * 100.0 / AVG(кол_слов) OVER (), 1) AS процент_от_среднего
FROM тексты
ORDER BY процент_от_среднего DESC;
```

### Задание 2. Строковые функции

**Задача:** нормализовать и анализировать текстовые данные словаря.

```sql
-- 2.1. Базовый анализ лексики: длина леммы и словоформы
SELECT
    лемма,
    словоформа,
    LENGTH(лемма)                     AS длина_леммы,
    LENGTH(словоформа)                AS длина_словоформы,
    LENGTH(словоформа) - LENGTH(лемма) AS разница_символов
FROM словарь
ORDER BY длина_леммы DESC;
```

```sql
-- 2.2. Нормализация регистра и извлечение префикса (первые 3 символа)
SELECT
    лемма,
    UPPER(лемма)                      AS лемма_верхний,
    LOWER(лемма)                      AS лемма_нижний,
    SUBSTRING(лемма, 1, 3)            AS префикс,
    REVERSE(лемма)                    AS лемма_наоборот
FROM словарь
ORDER BY лемма;
```

```sql
-- 2.3. Конкатенация: формирование полного описания словоформы
SELECT
    лемма || ' → ' || словоформа AS пара_лемма_форма,
    '[' || часть_речи || ']'     AS грамм_тег,
    'Частота: ' || частота       AS статистика,
    лемма || ' (' || часть_речи || ', ' || частота || ')' AS полное_описание
FROM словарь
ORDER BY частота DESC;
```

```sql
-- 2.4. Поиск слов по шаблону — все леммы на определённую букву
SELECT лемма, словоформа, часть_речи, частота
FROM словарь
WHERE лемма LIKE 'с%'        -- начинается на 'с'
   OR лемма LIKE '%ть'       -- оканчивается на '-ть' (инфинитивы)
ORDER BY лемма;
```

```sql
-- 2.5. Замена символов — очистка данных
SELECT
    лемма,
    REPLACE(лемма, 'ё', 'е')   AS лемма_без_ё,
    TRIM(словоформа)            AS словоформа_обрезанная,
    POSITION('а' IN лемма)     AS позиция_буквы_а
FROM словарь
WHERE LENGTH(лемма) >= 5;
```

### Задание 5. Комплексные выражения

**Задача:** объединить несколько типов выражений в одном запросе.

```sql
-- 5.1. Полный аналитический отчёт по лексике каждого текста
SELECT
    т.название,
    т.автор,
    т.год,
    COUNT(с.id)                                       AS уникальных_лемм,
    SUM(с.частота)                                    AS суммарная_частота,
    ROUND(AVG(с.частота), 1)                          AS средняя_частота,
    ROUND(AVG(LENGTH(с.лемма)), 2)                    AS средняя_длина_леммы,
    MAX(с.частота)                                    AS макс_частота,
    -- Вычисляемая метрика: лексическое богатство
    ROUND(COUNT(DISTINCT с.лемма)::FLOAT /
          NULLIF(т.кол_слов, 0) * 1000, 3)            AS лекс_богатство_на_1000,
    -- Классификация
    CASE
        WHEN COUNT(с.id) >= 5 THEN 'Богатая лексика'
        WHEN COUNT(с.id) >= 3 THEN 'Средняя лексика'
        ELSE 'Малая выборка'
    END AS лекс_оценка
FROM тексты т
LEFT JOIN словарь с ON с.текст_id = т.id
GROUP BY т.id, т.название, т.автор, т.год, т.кол_слов
ORDER BY уникальных_лемм DESC;
```

```sql
-- 5.2. Анализ предложений с вычисляемыми характеристиками
SELECT
    т.название,
    п.содержание,
    п.кол_слов,
    п.кол_символов,
    ROUND(п.кол_символов::FLOAT / NULLIF(п.кол_слов, 0), 2) AS средняя_длина_слова,
    CASE
        WHEN п.кол_слов <= 5  THEN 'Короткое'
        WHEN п.кол_слов <= 10 THEN 'Среднее'
        ELSE 'Длинное'
    END AS тип_предложения,
    UPPER(SUBSTRING(п.содержание, 1, 20)) || '...' AS фрагмент
FROM предложения п
JOIN тексты т ON п.текст_id = т.id
ORDER BY средняя_длина_слова DESC;
```

## Типичные ошибки и как их избежать

| Ошибка                      | Неверно                   | Верно                                          |
|-----------------------------|---------------------------|------------------------------------------------|
| Целочисленное деление       | `кол_символов / кол_слов` | `кол_символов::FLOAT / кол_слов`               |
| Сравнение с NULL через `=`  | `WHERE поле = NULL`       | `WHERE поле IS NULL`                           |
| Псевдоним в WHERE           | `WHERE средняя_длина > 5` | Повторить выражение или использовать подзапрос |
| Регистрозависимое сравнение | `WHERE лемма = 'Слово'`   | `WHERE LOWER(лемма) = 'слово'`                 |
| Деление на ноль             | `1 / кол_слов`            | `1 / NULLIF(кол_слов, 0)`                      |

## Вывод

В ходе лабораторной работы были изучены основные типы выражений в SQL:

- **Арифметические выражения** позволяют вычислять числовые показатели непосредственно в запросе: средние значения, доли, нормированные метрики.
- **Строковые функции** (`LENGTH`, `UPPER`, `LOWER`, `SUBSTRING`, `REPLACE`, `LIKE`) дают возможность нормализовать, анализировать и трансформировать текстовые данные — что критически важно в лингвистических задачах.
- **Логические выражения** обеспечивают гибкую фильтрацию данных с использованием составных условий.
- **Выражение `CASE WHEN`** реализует условную логику прямо внутри запроса и позволяет создавать категориальные признаки.

Сочетание этих инструментов делает SQL мощным языком для анализа лингвистических корпусов, частотных словарей и аннотированных баз данных.
