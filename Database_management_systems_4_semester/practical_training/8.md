## Теоретический материал

### Что такое агрегатные функции?

Агрегатные функции принимают набор строк и возвращают **одно** итоговое значение. Они используются для подсчёта, суммирования, нахождения среднего, минимума и максимума по столбцам.

### Основные агрегатные функции

| Функция                  | Описание                                    | Пример                         |
|--------------------------|---------------------------------------------|--------------------------------|
| `COUNT(*)`               | Количество всех строк                       | `COUNT(*)` → 150               |
| `COUNT(column)`          | Количество строк, где значение **не NULL**  | `COUNT(frequency)` → 148       |
| `COUNT(DISTINCT column)` | Количество уникальных значений              | `COUNT(DISTINCT pos_tag)` → 12 |
| `SUM(column)`            | Сумма числовых значений (NULL игнорируется) | `SUM(frequency)` → 45200       |
| `AVG(column)`            | Среднее арифметическое (NULL игнорируется)  | `AVG(frequency)` → 301.3       |
| `MIN(column)`            | Минимальное значение                        | `MIN(word_length)` → 1         |
| `MAX(column)`            | Максимальное значение                       | `MAX(word_length)` → 24        |

### Синтаксис с GROUP BY

```sql
SELECT column1, AGG_FUNCTION(column2)
FROM table_name
WHERE condition
GROUP BY column1
HAVING AGG_FUNCTION(column2) condition
ORDER BY AGG_FUNCTION(column2);
```

### Порядок выполнения запроса (логический)

```
FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY
```

Это важно: `WHERE` фильтрует строки **до** группировки, `HAVING` — **после**.

### Правила GROUP BY

- Все столбцы в `SELECT`, не обёрнутые в агрегатную функцию, **обязаны** быть перечислены в `GROUP BY`
- В `HAVING` можно использовать как агрегатные функции, так и столбцы из `GROUP BY`
- `ORDER BY` может ссылаться на агрегатную функцию или её псевдоним

### Поведение NULL

- `COUNT(*)` считает все строки, включая те, где есть NULL
- `COUNT(column)` **не считает** строки, где значение в этом столбце равно NULL
- `SUM`, `AVG`, `MIN`, `MAX` **игнорируют** NULL
- Если все значения NULL, `SUM` и `AVG` вернут NULL; `COUNT(column)` вернёт 0

### Псевдонимы и вложенность

```sql
-- Псевдоним для агрегата
SELECT pos_tag, COUNT(*) AS word_count
FROM lemmas
GROUP BY pos_tag;

-- Фильтрация через HAVING с псевдонимом (MySQL поддерживает)
SELECT pos_tag, COUNT(*) AS word_count
FROM lemmas
GROUP BY pos_tag
HAVING word_count > 10;
```

## Схема базы данных

Для всех упражнений используется следующая схема лингвистического корпуса:

```sql
CREATE TABLE authors (
    author_id   INT PRIMARY KEY AUTO_INCREMENT,
    full_name   VARCHAR(100) NOT NULL,
    birth_year  INT,
    nationality VARCHAR(50)
);

CREATE TABLE texts (
    text_id     INT PRIMARY KEY AUTO_INCREMENT,
    title       VARCHAR(200) NOT NULL,
    author_id   INT,
    genre       VARCHAR(50),
    pub_year    INT,
    word_count  INT,
    FOREIGN KEY (author_id) REFERENCES authors(author_id)
);

CREATE TABLE lemmas (
    lemma_id    INT PRIMARY KEY AUTO_INCREMENT,
    lemma       VARCHAR(100) NOT NULL,
    pos_tag     VARCHAR(20),
    is_stopword TINYINT(1) DEFAULT 0
);

CREATE TABLE word_forms (
    form_id     INT PRIMARY KEY AUTO_INCREMENT,
    form        VARCHAR(100) NOT NULL,
    lemma_id    INT,
    FOREIGN KEY (lemma_id) REFERENCES lemmas(lemma_id)
);

CREATE TABLE corpus_entries (
    entry_id    INT PRIMARY KEY AUTO_INCREMENT,
    text_id     INT,
    lemma_id    INT,
    frequency   INT NOT NULL DEFAULT 0,
    tf_idf      DECIMAL(10, 6),
    FOREIGN KEY (text_id) REFERENCES texts(text_id),
    FOREIGN KEY (lemma_id) REFERENCES lemmas(lemma_id)
);
```

## Разобранные примеры

### Пример 1. Сколько текстов в корпусе по каждому жанру?

```sql
SELECT genre,
       COUNT(*) AS text_count
FROM texts
GROUP BY genre
ORDER BY text_count DESC;
```

**Результат:**

| genre   | text_count |
|---------|------------|
| Роман   | 7          |
| Повесть | 4          |
| Рассказ | 2          |
| Пьеса   | 1          |
| Поэма   | 1          |

**Комментарий:** `COUNT(*)` считает каждую строку в группе. `ORDER BY text_count DESC` выводит жанры от самого представленного к наименее.

### Пример 2. Суммарный объём текстов по каждому автору

```sql
SELECT a.full_name,
       COUNT(t.text_id)   AS total_texts,
       SUM(t.word_count)  AS total_words,
       AVG(t.word_count)  AS avg_words
FROM authors a
JOIN texts t ON a.author_id = t.author_id
GROUP BY a.author_id, a.full_name
ORDER BY total_words DESC;
```

**Комментарий:** Здесь `author_id` и `full_name` попадают в `GROUP BY` вместе, так как `full_name` функционально зависит от `author_id` — это стандартная практика.

### Пример 3. Части речи с более чем пятью леммами в корпусе

```sql
SELECT pos_tag,
       COUNT(*) AS lemma_count
FROM lemmas
WHERE is_stopword = 0
GROUP BY pos_tag
HAVING lemma_count > 2
ORDER BY lemma_count DESC;
```

**Комментарий:** `WHERE is_stopword = 0` исключает стоп-слова **до** группировки. `HAVING lemma_count > 2` фильтрует уже сформированные группы.

### Пример 4. Максимальная и минимальная частота леммы по тексту

```sql
SELECT t.title,
       MAX(ce.frequency) AS max_freq,
       MIN(ce.frequency) AS min_freq,
       MAX(ce.frequency) - MIN(ce.frequency) AS freq_range
FROM texts t
JOIN corpus_entries ce ON t.text_id = ce.text_id
GROUP BY t.text_id, t.title
ORDER BY max_freq DESC;
```

### Пример 5. Количество форм на каждую лемму

```sql
SELECT l.lemma,
       l.pos_tag,
       COUNT(wf.form_id) AS form_count
FROM lemmas l
LEFT JOIN word_forms wf ON l.lemma_id = wf.lemma_id
GROUP BY l.lemma_id, l.lemma, l.pos_tag
ORDER BY form_count DESC;
```

**Комментарий:** `LEFT JOIN` сохранит леммы, у которых нет форм в таблице — `COUNT(wf.form_id)` вернёт 0 для них, тогда как `COUNT(*)` вернул бы 1.

**Задача 2.** Исследуйте «лексическое богатство» каждого текста: найдите соотношение количества уникальных лемм к общему числу вхождений (сумма частот). Чем выше это значение, тем разнообразнее лексика. Выведите название текста и вычисленный коэффициент, отсортируйте по убыванию.

**Задача 3.** Определите, у какого автора наибольшая «средняя частотность» лемм в его текстах (среднее по всем вхождениям в `corpus_entries`, связанным с его текстами). Сравните результаты для знаменательных слов (`is_stopword = 0`) и стоп-слов.

**Задача 4.** Сформируйте «частотный словарь»: для каждой леммы (не стоп-слова) найдите суммарную частоту по всему корпусу. Выведите топ-10 самых частотных лемм с их частью речи и суммарной частотой.
