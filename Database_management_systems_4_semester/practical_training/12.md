## Строковые функции в MySQL

**Предмет:** Системы управления базами данных  
**Специальность:** Компьютерная лингвистика  
**Тема:** Функции работы со строками

## Цели работы

- Освоить основные строковые функции MySQL для обработки текстовых данных
- Научиться применять функции поиска, замены, извлечения и форматирования строк
- Отработать использование строковых функций в контексте лингвистических баз данных
- Научиться комбинировать несколько строковых функций в одном запросе

## Теоретический материал

### 1. Функции определения длины и регистра

| Функция | Описание | Пример |
|---|---|---|
| `LENGTH(str)` | Длина строки в байтах | `LENGTH('слово')` → 10 |
| `CHAR_LENGTH(str)` | Длина строки в символах | `CHAR_LENGTH('слово')` → 5 |
| `UPPER(str)` | Перевод в верхний регистр | `UPPER('лемма')` → `ЛЕММА` |
| `LOWER(str)` | Перевод в нижний регистр | `LOWER('ЛЕММА')` → `лемма` |

> **Важно:** Для кириллицы используйте `CHAR_LENGTH`, а не `LENGTH`, так как кириллические символы занимают 2 байта в UTF-8.

### 2. Функции извлечения подстрок

| Функция | Описание | Пример |
|---|---|---|
| `SUBSTRING(str, pos, len)` | Извлекает подстроку начиная с позиции `pos` длиной `len` | `SUBSTRING('лингвистика', 1, 5)` → `лингв` |
| `LEFT(str, len)` | Первые `len` символов | `LEFT('префикс', 3)` → `пре` |
| `RIGHT(str, len)` | Последние `len` символов | `RIGHT('суффикс', 3)` → `икс` |
| `MID(str, pos, len)` | Псевдоним SUBSTRING | `MID('корень', 2, 3)` → `оре` |

### 3. Функции поиска

| Функция | Описание | Пример |
|---|---|---|
| `LOCATE(substr, str)` | Позиция первого вхождения (0 если не найдено) | `LOCATE('лог', 'филология')` → 5 |
| `INSTR(str, substr)` | Аналогично LOCATE, аргументы поменяны местами | `INSTR('морфология', 'лог')` → 7 |
| `FIND_IN_SET(str, strlist)` | Поиск в списке значений через запятую | `FIND_IN_SET('сущ', 'гл,сущ,прил')` → 2 |

### 4. Функции замены и вставки

| Функция | Описание | Пример |
|---|---|---|
| `REPLACE(str, from_str, to_str)` | Замена всех вхождений | `REPLACE('Ивановъ', 'ъ', '')` → `Иванов` |
| `INSERT(str, pos, len, newstr)` | Замена фрагмента строки | `INSERT('лемма', 3, 2, 'ин')` → `леина` |
| `REVERSE(str)` | Обратный порядок символов | `REVERSE('abc')` → `cba` |

### 5. Функции удаления пробелов и дополнения

| Функция | Описание | Пример |
|---|---|---|
| `TRIM(str)` | Удаление пробелов с обоих концов | `TRIM('  слово  ')` → `слово` |
| `LTRIM(str)` | Удаление ведущих пробелов | `LTRIM('  слово')` → `слово` |
| `RTRIM(str)` | Удаление завершающих пробелов | `RTRIM('слово  ')` → `слово` |
| `LPAD(str, len, padstr)` | Дополнение строки слева | `LPAD('01', 5, '0')` → `00001` |
| `RPAD(str, len, padstr)` | Дополнение строки справа | `RPAD('ABC', 6, '-')` → `ABC---` |

### 6. Функции объединения строк

| Функция | Описание | Пример |
|---|---|---|
| `CONCAT(s1, s2, ...)` | Объединение строк (NULL → NULL) | `CONCAT('при', 'ставка')` → `приставка` |
| `CONCAT_WS(sep, s1, s2, ...)` | Объединение с разделителем (пропускает NULL) | `CONCAT_WS('-', 'А', NULL, 'Б')` → `А-Б` |
| `REPEAT(str, count)` | Повторение строки | `REPEAT('*', 5)` → `*****` |
| `SPACE(n)` | Строка из n пробелов | `SPACE(3)` → `   ` |

### 7. Прочие строковые функции

| Функция | Описание | Пример |
|---|---|---|
| `FORMAT(x, d)` | Форматирование числа с разделителями | `FORMAT(12345.6, 2)` → `12,345.60` |
| `ASCII(str)` | ASCII-код первого символа | `ASCII('A')` → 65 |
| `CHAR(n)` | Символ по ASCII-коду | `CHAR(65)` → `A` |
| `STRCMP(s1, s2)` | Сравнение строк (-1, 0, 1) | `STRCMP('abc', 'abd')` → -1 |
| `SOUNDEX(str)` | Фонетическое представление строки | `SOUNDEX('Smith')` → `S530` |

## Схема базы данных

В этой лабораторной работе используется лингвистический корпус текстов русской классической литературы.

```sql
-- Авторы произведений
CREATE TABLE authors (
    author_id    INT PRIMARY KEY AUTO_INCREMENT,
    last_name    VARCHAR(60)  NOT NULL,
    first_name   VARCHAR(60)  NOT NULL,
    patronymic   VARCHAR(60),
    birth_year   INT,
    death_year   INT,
    notes        TEXT
);

-- Произведения
CREATE TABLE works (
    work_id      INT PRIMARY KEY AUTO_INCREMENT,
    author_id    INT NOT NULL,
    title        VARCHAR(200) NOT NULL,
    genre        VARCHAR(50),
    year_written INT,
    FOREIGN KEY (author_id) REFERENCES authors(author_id)
);

-- Токены (отдельные словоформы из текста)
CREATE TABLE tokens (
    token_id     INT PRIMARY KEY AUTO_INCREMENT,
    work_id      INT NOT NULL,
    word_form    VARCHAR(100) NOT NULL,
    lemma        VARCHAR(100),
    pos_tag      VARCHAR(20),
    position     INT,
    FOREIGN KEY (work_id) REFERENCES works(work_id)
);

-- Морфемы
CREATE TABLE morphemes (
    morpheme_id   INT PRIMARY KEY AUTO_INCREMENT,
    morpheme_text VARCHAR(50) NOT NULL,
    morph_type    VARCHAR(20) NOT NULL,  -- 'prefix', 'root', 'suffix', 'ending', 'postfix'
    language      VARCHAR(20) DEFAULT 'Russian'
);

-- Связь токенов с морфемами
CREATE TABLE token_morphemes (
    token_id    INT NOT NULL,
    morpheme_id INT NOT NULL,
    position    INT,
    PRIMARY KEY (token_id, morpheme_id),
    FOREIGN KEY (token_id)    REFERENCES tokens(token_id),
    FOREIGN KEY (morpheme_id) REFERENCES morphemes(morpheme_id)
);
```

## Практические задания

### Блок 1. Длина строк и регистр

**Задание 1.** Выведите все словоформы из таблицы `tokens` вместе с количеством символов в каждой. Отсортируйте результат от самой длинной словоформы к самой короткой.

```sql
SELECT
    word_form,
    CHAR_LENGTH(word_form) AS char_count
FROM tokens
ORDER BY char_count DESC;
```

**Задание 2.** Найдите все токены, у которых количество символов в `word_form` и в `lemma` различается. Это может указывать на изменение формы слова при лемматизации.

```sql
SELECT
    word_form,
    lemma,
    CHAR_LENGTH(word_form) AS wf_length,
    CHAR_LENGTH(lemma)     AS lemma_length,
    CHAR_LENGTH(word_form) - CHAR_LENGTH(lemma) AS diff
FROM tokens
WHERE word_form <> lemma
ORDER BY ABS(CHAR_LENGTH(word_form) - CHAR_LENGTH(lemma)) DESC;
```

**Задание 3.** Сформируйте «карточку автора»: полное имя в формате «ФАМИЛИЯ, Имя Отчество». Фамилию выведите в верхнем регистре, имя и отчество — как есть.

```sql
SELECT
    CONCAT(UPPER(last_name), ', ', first_name, ' ', patronymic) AS author_card

WHERE patronymic IS NOT NULL;
```

**Задание 4.** Выведите список уникальных частей речи (`pos_tag`) в нижнем регистре. Исключите NULL-значения.


**Задание 5.** Найдите авторов, у которых длина фамилии превышает 7 символов. Выведите фамилию, имя и длину фамилии.

### Блок 2. Извлечение подстрок

**Задание 6.** Извлеките первые 3 символа каждой леммы в таблице `tokens`. Это может быть полезно для анализа начальных буквосочетаний (триграмм).

```sql
SELECT
    lemma,

    RIGHT(lemma, 3)          AS trigram_end,
    CHAR_LENGTH(lemma)       AS total_len
FROM tokens
WHERE lemma IS NOT NULL;
```

**Задание 7.** Для каждой морфемы выведите её текст, тип и первый символ. Сгруппируйте результат по первому символу и типу морфемы, посчитав количество морфем в каждой группе.

```sql
SELECT
    LEFT(morpheme_text, 1)  AS first_char,
    morph_type,
    COUNT(*)                AS count
FROM morphemes
GROUP BY first_char, morph_type
ORDER BY morph_type, first_char;
```

**Задание 8.** Определите «середину» каждого заголовка произведения: извлеките подстроку, начиная с позиции `(длина / 2)`, длиной 5 символов. Это упражнение демонстрирует вычисляемые позиции в `SUBSTRING`.

```sql
SELECT
```

### Блок 6. Комплексные задания

**Задание 23.** Напишите запрос, который для каждого произведения:
- Подсчитывает количество уникальных лемм
- Выводит самую длинную лемму (`MAX` + подзапрос или группировка)
- Выводит среднюю длину лемм (округлить до 2 знаков)
- Формирует итоговую строку: `«Название»: N лемм, макс. M симв.`

```sql
SELECT
    w.title,
    COUNT(DISTINCT t.lemma)                    AS unique_lemmas,
    MAX(CHAR_LENGTH(t.lemma))                  AS max_lemma_len,
    ROUND(AVG(CHAR_LENGTH(t.lemma)), 2)        AS avg_lemma_len,
    CONCAT(
        '«', w.title, '»: ',
        COUNT(DISTINCT t.lemma), ' лемм, макс. ',
        MAX(CHAR_LENGTH(t.lemma)), ' симв.'
    )                                          AS summary
FROM works w
JOIN tokens t ON w.work_id = t.work_id
GROUP BY w.work_id, w.title
ORDER BY unique_lemmas DESC;
```

**Задание 24.** Найдите «потенциальные опечатки»: пары (word_form, lemma), где:
- Длина `word_form` и `lemma` одинакова
- Первые 2 символа совпадают
- Последние 2 символа совпадают
- Но `word_form` ≠ `lemma`

Это упражнение имитирует простой алгоритм нечёткого сравнения.

```sql
SELECT
    word_form,
    lemma,
    CHAR_LENGTH(word_form) AS len
FROM tokens
WHERE word_form <> lemma
  AND CHAR_LENGTH(word_form) = CHAR_LENGTH(lemma)
  AND LEFT(word_form, 2)  = LEFT(lemma, 2)
  AND RIGHT(word_form, 2) = RIGHT(lemma, 2);
```

**Задание 26.** Сформируйте «лингвистический паспорт» каждого автора в одну строку. Формат:
`ID:[author_id] | [ФАМИЛИЯ И.О.] | Годы: [birth]-[death] | Произведений: [N]`
Если год смерти неизвестен, написать «наст.вр.».

```sql
SELECT
    CONCAT(
        'ID:', a.author_id, ' | ',
        UPPER(a.last_name), ' ', LEFT(a.first_name, 1), '.', LEFT(COALESCE(a.patronymic, ''), 1),
        CASE WHEN a.patronymic IS NOT NULL THEN '.' ELSE '' END, ' | ',
        'Годы: ', a.birth_year, '–', COALESCE(CAST(a.death_year AS CHAR), 'наст.вр.'), ' | ',

    ) AS passport
FROM authors a
LEFT JOIN works w ON a.author_id = w.author_id
GROUP BY a.author_id, a.last_name, a.first_name, a.patronymic, a.birth_year, a.death_year
ORDER BY a.last_name;
```

**Задание 27 ★.** Напишите запрос, который «переворачивает» каждую лемму (функция `REVERSE`) и проверяет, является ли она палиндромом (перевёрнутое слово совпадает с оригиналом). Выведите все леммы, помечая палиндромы.

```sql
SELECT
    lemma,
    REVERSE(lemma)                                    AS reversed,
    IF(lemma = REVERSE(lemma), '✓ палиндром', '✗')   AS is_palindrome
FROM tokens
WHERE lemma IS NOT NULL
GROUP BY lemma
ORDER BY is_palindrome DESC, lemma;
```

**Задание 28 ★.** Используя оконную функцию `ROW_NUMBER()` в сочетании со строковыми функциями, пронумеруйте все токены в рамках каждого произведения и сформируйте «адрес» токена в формате: `[краткий заголовок]-[номер]`, где краткий заголовок — первые 5 символов заголовка произведения.

```sql
SELECT
    w.title,
    t.word_form,
    ROW_NUMBER() OVER (PARTITION BY t.work_id ORDER BY t.position) AS token_num,
    CONCAT(
        LEFT(w.title, 5), '-',
        LPAD(ROW_NUMBER() OVER (PARTITION BY t.work_id ORDER BY t.position), 4, '0')
    ) AS token_address
FROM tokens t
JOIN works w ON t.work_id = w.work_id
ORDER BY w.work_id, t.position;
```

**Задание 29 ★.** Создайте «облако частот» морфемных типов: для каждого типа морфемы сформируйте визуальную полосу из символов `█`, длина которой пропорциональна количеству морфем этого типа (масштаб: 1 символ = 1 морфема).

**Задание 30 ★.** Реализуйте простую функцию «нормализации» словоформ: приведите все слова к нижнему регистру, удалите ведущие и завершающие пробелы, замените «ё» на «е». Проверьте, сколько словоформ изменилось после нормализации.

```sql
SELECT
    word_form                                              AS original,
    REPLACE(TRIM(LOWER(word_form)), 'ё', 'е')             AS normalized,

    )                                                      AS status

```
