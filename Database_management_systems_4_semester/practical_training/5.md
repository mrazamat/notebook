# Практическое занятие: Оператор INSERT в SQL

## Теоретический справочник

### Что такое INSERT?

Оператор `INSERT` используется для добавления новых строк в таблицу базы данных. Это один из основных операторов языка DML (Data Manipulation Language).

### Синтаксис

**Вставка одной строки с указанием столбцов:**
```sql
INSERT INTO имя_таблицы (столбец1, столбец2, столбец3)
VALUES (значение1, значение2, значение3);
```

**Вставка одной строки без указания столбцов** (значения должны идти в том же порядке, что и столбцы в таблице):
```sql
INSERT INTO имя_таблицы
VALUES (значение1, значение2, значение3);
```

**Вставка нескольких строк за один запрос:**
```sql
INSERT INTO имя_таблицы (столбец1, столбец2)
VALUES
    (значение1а, значение2а),
    (значение1б, значение2б),
    (значение1в, значение2в);
```

**Вставка данных из другой таблицы (INSERT ... SELECT):**
```sql
INSERT INTO имя_таблицы (столбец1, столбец2)
SELECT столбец_а, столбец_б
FROM другая_таблица
WHERE условие;
```

### Важные правила

- Если столбец объявлен как `NOT NULL` и не имеет значения по умолчанию, его необходимо указать явно.
- Строковые значения заключаются в одинарные кавычки: `'текст'`.
- Числовые значения пишутся без кавычек: `42`, `3.14`.
- Для пропуска необязательного столбца используется ключевое слово `NULL` или столбец просто не упоминается в списке.
- Порядок значений в `VALUES` должен строго соответствовать порядку столбцов в списке.

### Типы данных (напоминание)

| Тип            | Описание                    | Пример               |
|----------------|-----------------------------|----------------------|
| `INTEGER`      | Целое число                 | `42`                 |
| `NUMERIC(p,s)` | Число с дробной частью      | `3.14`               |
| `VARCHAR(n)`   | Строка переменной длины     | `'лемма'`            |
| `TEXT`         | Текст без ограничения длины | `'длинный текст...'` |
| `BOOLEAN`      | Логическое значение         | `TRUE` / `FALSE`     |
| `DATE`         | Дата                        | `'2024-01-15'`       |


## Схема базы данных для занятия

На этом занятии мы работаем с учебной базой данных **лингвистического корпуса русской классической литературы**. Выполните следующие команды для создания таблиц:

```sql
-- Таблица авторов
CREATE TABLE authors (
    author_id   INTEGER PRIMARY KEY,
    full_name   VARCHAR(150) NOT NULL,
    birth_year  INTEGER,
    death_year  INTEGER
);

-- Таблица текстов корпуса
CREATE TABLE texts (
    text_id     INTEGER PRIMARY KEY,
    author_id   INTEGER REFERENCES authors(author_id),
    title       VARCHAR(255) NOT NULL,
    genre       VARCHAR(50),
    written_year INTEGER
);

-- Таблица частей речи
CREATE TABLE pos_tags (
    pos_id      INTEGER PRIMARY KEY,
    pos_code    VARCHAR(10) NOT NULL UNIQUE,
    pos_name_ru VARCHAR(50) NOT NULL,
    pos_name_en VARCHAR(50)
);

-- Таблица лемм
CREATE TABLE lemmas (
    lemma_id    INTEGER PRIMARY KEY,
    lemma_form  VARCHAR(100) NOT NULL,
    pos_id      INTEGER REFERENCES pos_tags(pos_id),
    is_stopword BOOLEAN DEFAULT FALSE
);

-- Таблица словоформ
CREATE TABLE wordforms (
    wordform_id  INTEGER PRIMARY KEY,
    lemma_id     INTEGER REFERENCES lemmas(lemma_id),
    wordform     VARCHAR(100) NOT NULL,
    gram_info    VARCHAR(150)
);

-- Таблица частотных данных
CREATE TABLE word_frequencies (
    freq_id      INTEGER PRIMARY KEY,
    text_id      INTEGER REFERENCES texts(text_id),
    lemma_id     INTEGER REFERENCES lemmas(lemma_id),
    frequency    INTEGER NOT NULL,
    relative_freq NUMERIC(10, 6)
);
```
