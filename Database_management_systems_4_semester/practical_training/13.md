# Практическое занятие: Функции даты, времени и интервала в MySQL

**Предмет:** Системы управления базами данных  
**Специальность:** Компьютерная лингвистика  
**СУБД:** MySQL 8.x

## Цели занятия

- Освоить встроенные функции MySQL для работы с датой и временем
- Научиться вычислять временные интервалы между событиями
- Применять функции дат для анализа лингвистических корпусов и связанных данных
- Форматировать и преобразовывать значения дат под конкретные задачи

## Теоретический справочник

### Типы данных для даты и времени

| Тип         | Формат                | Диапазон                | Пример                  |
|-------------|-----------------------|-------------------------|-------------------------|
| `DATE`      | `YYYY-MM-DD`          | 1000-01-01 — 9999-12-31 | `'1868-09-09'`          |
| `TIME`      | `HH:MM:SS`            | −838:59:59 — 838:59:59  | `'14:30:00'`            |
| `DATETIME`  | `YYYY-MM-DD HH:MM:SS` | 1000-01-01 — 9999-12-31 | `'1868-09-09 10:00:00'` |
| `TIMESTAMP` | `YYYY-MM-DD HH:MM:SS` | 1970-01-01 — 2038-01-19 | `'2024-03-15 09:45:00'` |
| `YEAR`      | `YYYY`                | 1901 — 2155             | `1868`                  |

### Функции получения текущей даты и времени

| Функция           | Описание                    | Пример результата     |
|-------------------|-----------------------------|-----------------------|
| `NOW()`           | Текущая дата и время        | `2024-11-20 14:32:10` |
| `CURDATE()`       | Текущая дата                | `2024-11-20`          |
| `CURTIME()`       | Текущее время               | `14:32:10`            |
| `SYSDATE()`       | Дата/время вызова функции   | `2024-11-20 14:32:10` |
| `UTC_DATE()`      | Текущая дата по UTC         | `2024-11-20`          |
| `UTC_TIMESTAMP()` | Текущие дата и время по UTC | `2024-11-20 11:32:10` |

### Функции извлечения компонентов даты

| Функция                | Извлекает                | Пример                            | Результат |
|------------------------|--------------------------|-----------------------------------|-----------|
| `YEAR(d)`              | Год                      | `YEAR('1868-09-09')`              | `1868`    |
| `MONTH(d)`             | Месяц (1–12)             | `MONTH('1868-09-09')`             | `9`       |
| `DAY(d)`               | День месяца              | `DAY('1868-09-09')`               | `9`       |
| `HOUR(t)`              | Час                      | `HOUR('14:30:00')`                | `14`      |
| `MINUTE(t)`            | Минуты                   | `MINUTE('14:30:00')`              | `30`      |
| `SECOND(t)`            | Секунды                  | `SECOND('14:30:45')`              | `45`      |
| `DAYOFWEEK(d)`         | День недели (1=вс, 7=сб) | `DAYOFWEEK('2024-11-20')`         | `4`       |
| `DAYOFYEAR(d)`         | Порядковый день года     | `DAYOFYEAR('2024-03-01')`         | `61`      |
| `WEEK(d)`              | Номер недели             | `WEEK('2024-11-20')`              | `46`      |
| `QUARTER(d)`           | Квартал (1–4)            | `QUARTER('2024-08-15')`           | `3`       |
| `EXTRACT(unit FROM d)` | Любой компонент          | `EXTRACT(YEAR FROM '1868-09-09')` | `1868`    |

### Функции форматирования и преобразования

| Функция                 | Описание                  | Пример                                                 |
|-------------------------|---------------------------|--------------------------------------------------------|
| `DATE_FORMAT(d, fmt)`   | Форматирование по маске   | `DATE_FORMAT('1868-09-09', '%d.%m.%Y')` → `09.09.1868` |
| `STR_TO_DATE(str, fmt)` | Строка → дата             | `STR_TO_DATE('09.09.1868', '%d.%m.%Y')`                |
| `DATE(dt)`              | Извлечь дату из datetime  | `DATE('1868-09-09 10:00:00')` → `1868-09-09`           |
| `TIME(dt)`              | Извлечь время из datetime | `TIME('1868-09-09 10:00:00')` → `10:00:00`             |
| `UNIX_TIMESTAMP(d)`     | Перевод в Unix-время      | целое число секунд                                     |
| `FROM_UNIXTIME(n)`      | Unix-время → DATETIME     | стандартный формат                                     |

**Спецификаторы формата `DATE_FORMAT`:**

| Спецификатор | Описание                   | Пример      |
|--------------|----------------------------|-------------|
| `%Y`         | Год 4 цифры                | `1868`      |
| `%y`         | Год 2 цифры                | `68`        |
| `%m`         | Месяц 01–12                | `09`        |
| `%M`         | Полное название месяца     | `September` |
| `%d`         | День 01–31                 | `09`        |
| `%D`         | День с суффиксом           | `9th`       |
| `%H`         | Час 00–23                  | `14`        |
| `%i`         | Минуты 00–59               | `30`        |
| `%s`         | Секунды 00–59              | `00`        |
| `%W`         | Полное название дня недели | `Wednesday` |
| `%j`         | День года 001–366          | `252`       |

### Арифметические функции с датами

| Функция                        | Описание                      | Пример                                            |
|--------------------------------|-------------------------------|---------------------------------------------------|
| `DATE_ADD(d, INTERVAL n unit)` | Прибавить интервал            | `DATE_ADD('1868-09-09', INTERVAL 10 YEAR)`        |
| `DATE_SUB(d, INTERVAL n unit)` | Вычесть интервал              | `DATE_SUB('1910-11-07', INTERVAL 3 MONTH)`        |
| `DATEDIFF(d1, d2)`             | Разность в днях (d1 − d2)     | `DATEDIFF('1910-11-07', '1828-09-09')`            |
| `TIMESTAMPDIFF(unit, d1, d2)`  | Разность в указанных единицах | `TIMESTAMPDIFF(YEAR, '1828-09-09', '1910-11-07')` |
| `TIMEDIFF(t1, t2)`             | Разность временны́х значений  | `TIMEDIFF('18:00:00', '10:30:00')`                |
| `ADDDATE(d, n)`                | Прибавить дни                 | `ADDDATE('1868-09-09', 30)`                       |
| `SUBDATE(d, n)`                | Вычесть дни                   | `SUBDATE('1910-11-07', 7)`                        |
| `LAST_DAY(d)`                  | Последний день месяца         | `LAST_DAY('2024-02-01')` → `2024-02-29`           |

**Единицы интервала (`unit`):**  
`MICROSECOND`, `SECOND`, `MINUTE`, `HOUR`, `DAY`, `WEEK`, `MONTH`, `QUARTER`, `YEAR`

### Прочие полезные функции

| Функция               | Описание                                  |
|-----------------------|-------------------------------------------|
| `DAYNAME(d)`          | Название дня недели на английском         |
| `MONTHNAME(d)`        | Название месяца на английском             |
| `MAKEDATE(year, day)` | Дата из года и порядкового дня            |
| `MAKETIME(h, m, s)`   | Время из компонентов                      |
| `PERIOD_DIFF(p1, p2)` | Разность в месяцах между периодами YYYYMM |
| `TO_DAYS(d)`          | Число дней от 0-го года                   |
| `FROM_DAYS(n)`        | Дата по числу дней                        |

## Структура базы данных

Для занятия используется база данных, содержащая информацию о писателях, их произведениях и лингвистическом корпусе, собранном на основе этих текстов.

```sql
-- Авторы русской литературы
CREATE TABLE authors (
    author_id   INT PRIMARY KEY AUTO_INCREMENT,
    full_name   VARCHAR(100) NOT NULL,
    birth_date  DATE,
    death_date  DATE,
    birth_place VARCHAR(100)
);

-- Литературные произведения
CREATE TABLE works (
    work_id        INT PRIMARY KEY AUTO_INCREMENT,
    author_id      INT NOT NULL,
    title          VARCHAR(200) NOT NULL,
    written_date   DATE,
    published_date DATE,
    genre          VARCHAR(50),
    word_count     INT,
    FOREIGN KEY (author_id) REFERENCES authors(author_id)
);

-- Корпусные тексты (фрагменты, добавленные в корпус)
CREATE TABLE corpus_texts (
    text_id        INT PRIMARY KEY AUTO_INCREMENT,
    work_id        INT NOT NULL,
    fragment_text  TEXT,
    added_at       DATETIME DEFAULT NOW(),
    last_updated   TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    token_count    INT,
    annotated      TINYINT(1) DEFAULT 0,
    FOREIGN KEY (work_id) REFERENCES works(work_id)
);

-- Сеансы аннотирования (работа лингвистов-аннотаторов)
CREATE TABLE annotation_sessions (
    session_id    INT PRIMARY KEY AUTO_INCREMENT,
    text_id       INT NOT NULL,
    annotator     VARCHAR(80),
    started_at    DATETIME,
    finished_at   DATETIME,
    tags_added    INT DEFAULT 0,
    FOREIGN KEY (text_id) REFERENCES corpus_texts(text_id)
);

-- Лексические единицы (леммы) с датой первой фиксации
CREATE TABLE lemmas (
    lemma_id       INT PRIMARY KEY AUTO_INCREMENT,
    lemma_form     VARCHAR(100) NOT NULL,
    pos_tag        VARCHAR(20),
    first_recorded DATE,
    frequency      INT DEFAULT 0
);
```

## Практические упражнения

### Блок 1. Текущая дата и время

**Упражнение 1.1.** Выведите текущие дату и время, только текущую дату и только текущее время тремя отдельными запросами.

```sql
SELECT NOW();
SELECT CURDATE();
SELECT CURTIME();
```

**Упражнение 1.2.** Выведите фразу в виде: `Сегодня: 20.11.2024, время: 14:32` — используя `NOW()` и `DATE_FORMAT`.

```sql
SELECT CONCAT(
    'Сегодня: ',
    DATE_FORMAT(NOW(), '%d.%m.%Y'),
    ', время: ',
    DATE_FORMAT(NOW(), '%H:%i')
) AS current_moment;
```

**Упражнение 1.3.** Для каждого фрагмента корпуса выведите его `text_id` и количество дней с момента добавления до сегодняшнего дня.

```sql
SELECT
    text_id,
    added_at,
    DATEDIFF(CURDATE(), DATE(added_at)) AS days_since_added
FROM corpus_texts
ORDER BY days_since_added DESC;
```

### Блок 2. Извлечение компонентов даты

**Упражнение 2.1.** Выведите имя каждого автора, год и месяц рождения отдельными столбцами.

```sql
SELECT
    full_name,
    YEAR(birth_date)  AS birth_year,
    MONTH(birth_date) AS birth_month,
    DAY(birth_date)   AS birth_day
FROM authors
ORDER BY birth_year;
```

**Упражнение 2.2.** Определите, в каком квартале года родился каждый автор.

```sql
SELECT
    full_name,
    birth_date,
    QUARTER(birth_date) AS birth_quarter
FROM authors
ORDER BY birth_quarter, full_name;
```

**Упражнение 2.3.** Выведите имя каждого автора и название дня недели, в который он родился (используйте `DAYNAME`).

```sql
SELECT
    full_name,
    birth_date,
    DAYNAME(birth_date) AS day_of_week
FROM authors;
```

**Упражнение 2.4.** Для каждой леммы выведите `lemma_form` и год первой фиксации. Отфильтруйте только те леммы, которые впервые зафиксированы до 1810 года.

```sql
SELECT
    lemma_form,
    pos_tag,
    first_recorded,
    YEAR(first_recorded) AS first_year
FROM lemmas
WHERE YEAR(first_recorded) < 1810
ORDER BY first_recorded;
```

**Упражнение 2.5.** Для каждого фрагмента корпуса определите, в какой час суток (`HOUR`) он был добавлен. Сгруппируйте по часу и подсчитайте количество фрагментов.

```sql
SELECT
    HOUR(added_at)       AS hour_of_day,
    COUNT(*)             AS texts_added
FROM corpus_texts
GROUP BY hour_of_day
ORDER BY hour_of_day;
```

**Упражнение 2.6.** Используя `EXTRACT`, получите из даты публикации произведений: год, месяц и квартал. Добавьте столбец с именем автора, используя `JOIN`.

```sql
SELECT
    a.full_name,
    w.title,
    EXTRACT(YEAR    FROM w.published_date) AS pub_year,
    EXTRACT(MONTH   FROM w.published_date) AS pub_month,
    EXTRACT(QUARTER FROM w.published_date) AS pub_quarter
FROM works w
JOIN authors a ON w.author_id = a.author_id
ORDER BY pub_year;
```

### Блок 3. Форматирование дат

**Упражнение 3.1.** Выведите дату рождения каждого автора в формате `«09 сентября 1828 года»`. Для названия месяца используйте `MONTHNAME` или `DATE_FORMAT` с `%M` (и учтите, что результат будет на английском — это нормально для MySQL без локализации).

```sql
SELECT
    full_name,
    DATE_FORMAT(birth_date, '%d %M %Y') AS formatted_birth
FROM authors
ORDER BY birth_date;
```

**Упражнение 3.2.** Для сеансов аннотирования выведите `annotator`, время начала в формате `ЧЧ:ММ ДД.ММ.ГГГГ` и время окончания в том же формате. Для незавершённых сеансов вывести `'не завершён'`.

```sql
SELECT
    annotator,
    DATE_FORMAT(started_at,  '%H:%i %d.%m.%Y') AS started_formatted,
    COALESCE(
        DATE_FORMAT(finished_at, '%H:%i %d.%m.%Y'),
        'не завершён'
    ) AS finished_formatted
FROM annotation_sessions
ORDER BY started_at;
```

**Упражнение 3.3.** Добавьте в вывод списка произведений столбец `written_century`, содержащий номер века написания произведения, вычисленный через `YEAR`.

```sql
SELECT
    title,
    written_date,
    YEAR(written_date)                            AS written_year,
    CONCAT(FLOOR((YEAR(written_date) - 1) / 100) + 1, ' век') AS written_century
FROM works
ORDER BY written_date;
```

**Упражнение 3.4.** Преобразуйте строку `'07-11-1910'` в тип `DATE` с помощью `STR_TO_DATE`. Затем выведите из неё год, месяц и день отдельно.

```sql
SELECT
    STR_TO_DATE('07-11-1910', '%d-%m-%Y')                      AS converted_date,
    YEAR(STR_TO_DATE('07-11-1910', '%d-%m-%Y'))                AS y,
    MONTH(STR_TO_DATE('07-11-1910', '%d-%m-%Y'))               AS m,
    DAY(STR_TO_DATE('07-11-1910', '%d-%m-%Y'))                 AS d;
```

### Блок 4. Арифметика с датами и интервалами

**Упражнение 4.1.** Вычислите возраст каждого автора на момент смерти в полных годах. Для живых (NULL в `death_date`) выведите `'жив'`.

```sql
SELECT
    full_name,
    birth_date,
    death_date,
    CASE
        WHEN death_date IS NULL THEN 'жив'
        ELSE CAST(TIMESTAMPDIFF(YEAR, birth_date, death_date) AS CHAR)
    END AS age_at_death
FROM authors
ORDER BY birth_date;
```

**Упражнение 4.2.** Для каждого произведения вычислите разрыв между датой написания и датой публикации в днях. Назовите столбец `days_to_publish`.

```sql
SELECT
    title,
    written_date,
    published_date,
    DATEDIFF(published_date, written_date) AS days_to_publish
FROM works
ORDER BY days_to_publish DESC;
```

**Упражнение 4.3.** Вычислите продолжительность каждого сеанса аннотирования в минутах. Незавершённые сеансы пропустите (`WHERE finished_at IS NOT NULL`).

```sql
SELECT
    annotator,
    started_at,
    finished_at,
    TIMESTAMPDIFF(MINUTE, started_at, finished_at) AS duration_minutes
FROM annotation_sessions
WHERE finished_at IS NOT NULL
ORDER BY duration_minutes DESC;
```

**Упражнение 7.1. Оконные функции и ранжирование по дате.**


**Упражнение 7.3. Определение временных «пробелов».**

```sql
SELECT

        ORDER BY w.written_date
```
