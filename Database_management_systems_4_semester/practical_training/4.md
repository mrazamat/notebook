## Команда ORDER BY оператора SELECT
### Предмет: Системы управления базами данных
### Специальность: Компьютерная лингвистика

---

## Цель работы

Научиться применять предложение `ORDER BY` для сортировки результатов SQL-запросов: по одному и нескольким столбцам, в возрастающем и убывающем порядке, с учётом значений `NULL`.

---

## 1. Теоретическая справка

### 1.1 Синтаксис ORDER BY

Предложение `ORDER BY` задаёт порядок строк в результирующей выборке. Оно всегда указывается **последним** в операторе `SELECT`.

```sql
SELECT столбец1, столбец2, ...
FROM таблица
[WHERE условие]
ORDER BY столбец1 [ASC | DESC], столбец2 [ASC | DESC], ...;
```

| Ключевое слово | Значение                      |
|----------------|-------------------------------|
| `ASC`          | По возрастанию (по умолчанию) |
| `DESC`         | По убыванию                   |

### 1.2 Сортировка по нескольким столбцам

Если указано несколько столбцов, сортировка выполняется **последовательно**: сначала по первому столбцу, затем, при совпадении значений — по второму, и т.д.

```sql
ORDER BY part_of_speech ASC, frequency DESC
```

### 1.3 Сортировка по номеру столбца

Вместо имени столбца можно указать его **порядковый номер** в списке `SELECT`:

```sql
SELECT word, frequency, part_of_speech
FROM lexicon
ORDER BY 2 DESC;   -- сортировка по столбцу frequency
```

### 1.4 Поведение NULL при сортировке

Значения `NULL` — особый случай. В большинстве СУБД:

- В PostgreSQL: `NULL` считается **больше** любого значения → при `ASC` выводится **последним**, при `DESC` — **первым**.
- Поведение можно явно задать с помощью `NULLS FIRST` / `NULLS LAST`:

```sql
ORDER BY frequency ASC NULLS LAST;
ORDER BY frequency DESC NULLS FIRST;
```

### 1.5 Сортировка по вычисляемому выражению

```sql
SELECT word, (frequency * 100.0 / total) AS relative_freq
FROM word_stats
ORDER BY relative_freq DESC;
```

## 2. Схема учебной базы данных

В данной лабораторной работе используется лингвистическая база данных **«Корпус русской классической прозы»**.

### Таблица `texts` — тексты корпуса

| Столбец          | Тип          | Описание                        |
|------------------|--------------|---------------------------------|
| `text_id`        | INTEGER      | Идентификатор текста            |
| `title`          | VARCHAR(200) | Название произведения           |
| `author`         | VARCHAR(100) | Автор                           |
| `year_published` | INTEGER      | Год публикации                  |
| `genre`          | VARCHAR(50)  | Жанр (роман, рассказ, повесть…) |
| `word_count`     | INTEGER      | Количество слов в тексте        |

### Таблица `lexicon` — словарные единицы корпуса

| Столбец          | Тип          | Описание                         |
|------------------|--------------|----------------------------------|
| `word_id`        | INTEGER      | Идентификатор леммы              |
| `word`           | VARCHAR(100) | Лемма (словарная форма)          |
| `part_of_speech` | VARCHAR(30)  | Часть речи                       |
| `frequency`      | INTEGER      | Абсолютная частота в корпусе     |
| `text_id`        | INTEGER      | Ссылка на текст (FK)             |
| `syllable_count` | INTEGER      | Количество слогов                |
| `is_borrowed`    | BOOLEAN      | Является ли слово заимствованием |

### Таблица `morphemes` — морфемный состав

| Столбец         | Тип         | Описание                                   |
|-----------------|-------------|--------------------------------------------|
| `morpheme_id`   | INTEGER     | Идентификатор морфемы                      |
| `word_id`       | INTEGER     | Ссылка на лемму (FK)                       |
| `morpheme`      | VARCHAR(50) | Морфема                                    |
| `morpheme_type` | VARCHAR(20) | Тип: корень, приставка, суффикс, окончание |

## 4. Практические примеры

### Пример 1. Простая сортировка по одному столбцу (ASC)

Вывести все слова словаря в алфавитном порядке:

```sql
SELECT word_id, word, part_of_speech
FROM lexicon
ORDER BY word ASC;
```

**Результат (фрагмент):**

| word_id | word      | part_of_speech  |
|---------|-----------|-----------------|
| 12      | анализ    | существительное |
| 10      | атмосфера | существительное |
| 7       | мысль     | существительное |
| ...     | ...       | ...             |

---

### Пример 2. Сортировка по убыванию

Вывести слова от самого частотного к наименее частотному:

```sql
SELECT word, frequency, part_of_speech
FROM lexicon
ORDER BY frequency DESC;
```

---

### Пример 3. Сортировка по нескольким столбцам

Вывести слова, сгруппировав по части речи (по алфавиту), внутри каждой группы — по убыванию частоты:

```sql
SELECT word, part_of_speech, frequency
FROM lexicon
ORDER BY part_of_speech ASC, frequency DESC;
```

---

### Пример 4. NULL и управление их позицией

Вывести слова с частотой, размещая `NULL` в самом конце:

```sql
SELECT word, frequency
FROM lexicon
ORDER BY frequency DESC NULLS LAST;
```

---

### Пример 5. Сортировка по вычисляемому выражению

Вывести тексты корпуса, отсортировав по «возрасту» текста (от самого старого к новому):

```sql
SELECT title, author, year_published,
       (2024 - year_published) AS age_years
FROM texts
ORDER BY age_years DESC;
```

---

### Пример 6. Сортировка по номеру столбца

```sql
SELECT word, syllable_count, frequency
FROM lexicon
ORDER BY 2 ASC, 3 DESC;
-- Сначала односложные, внутри — по убыванию частоты
```

## Справочник: порядок предложений в операторе SELECT

```sql
SELECT   [DISTINCT] столбцы        -- 1. Что выбрать
FROM     таблица                   -- 2. Откуда
WHERE    условие                   -- 3. Фильтр строк
GROUP BY столбцы                   -- 4. Группировка
HAVING   условие_группы            -- 5. Фильтр групп
ORDER BY столбцы [ASC|DESC]        -- 6. Сортировка
LIMIT    n;                        -- 7. Ограничение строк
```

> **Запомните:** `ORDER BY` всегда идёт предпоследним (перед `LIMIT`), потому что сортировка применяется к **готовому результату** — после всех фильтраций и группировок.
