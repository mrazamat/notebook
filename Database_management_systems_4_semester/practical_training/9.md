## Группировка данных в SQL: GROUP BY, HAVING и агрегатные функции

**Дисциплина:** Система управления базами данных  
**Специальность:** Компьютерная лингвистика  
**Тема:** Группировка данных, агрегатные функции, фильтрация групп

## Цели работы

- Освоить синтаксис оператора `GROUP BY` для группировки строк по одному или нескольким столбцам
- Научиться применять агрегатные функции (`COUNT`, `SUM`, `AVG`, `MIN`, `MAX`) совместно с группировкой
- Понять разницу между `WHERE` и `HAVING` при фильтрации данных
- Научиться группировать данные по нескольким столбцам и использовать псевдонимы в агрегатных запросах
- Применить навыки группировки к задачам корпусной лингвистики

## Теоретическая часть

### 1. Зачем нужна группировка?

В лингвистических базах данных часто возникают вопросы вида:
- «Сколько словоформ у каждой части речи?»
- «Какова средняя длина слова по каждому автору?»
- «Сколько раз каждая лемма встречается в корпусе?»

Для ответа на такие вопросы SQL предоставляет механизм **группировки строк** — оператор `GROUP BY`.

### 2. Синтаксис GROUP BY

```sql
SELECT column1, aggregate_function(column2)
FROM table_name
WHERE condition
GROUP BY column1
HAVING aggregate_condition
ORDER BY column1;
```

**Порядок выполнения запроса (логический):**

| Шаг | Оператор   | Что происходит                          |
|-----|------------|-----------------------------------------|
| 1   | `FROM`     | Определяется источник данных            |
| 2   | `WHERE`    | Фильтруются строки до группировки       |
| 3   | `GROUP BY` | Строки объединяются в группы            |
| 4   | `HAVING`   | Фильтруются группы после группировки    |
| 5   | `SELECT`   | Формируются итоговые столбцы            |
| 6   | `ORDER BY` | Результат сортируется                   |

### 3. Агрегатные функции

| Функция      | Описание                                      | Пример                        |
|--------------|-----------------------------------------------|-------------------------------|
| `COUNT(*)`   | Количество строк в группе                     | `COUNT(*)`                    |
| `COUNT(col)` | Количество строк, где col не NULL             | `COUNT(frequency)`            |
| `SUM(col)`   | Сумма значений                                | `SUM(frequency)`              |
| `AVG(col)`   | Среднее значение                              | `AVG(word_length)`            |
| `MIN(col)`   | Минимальное значение                          | `MIN(frequency)`              |
| `MAX(col)`   | Максимальное значение                         | `MAX(frequency)`              |
| `GROUP_CONCAT(col)` | Объединение значений в строку (MySQL) | `GROUP_CONCAT(word_form)`     |

> **Важно:** `COUNT(*)` считает все строки включая NULL, а `COUNT(col)` — только строки с непустым значением в указанном столбце.

### 4. WHERE против HAVING

```sql
-- WHERE: фильтрует строки ДО группировки
SELECT pos_tag, COUNT(*) AS total
FROM lemmas
WHERE word_length > 3          -- сначала отбираем леммы длиннее 3 символов
GROUP BY pos_tag;

-- HAVING: фильтрует группы ПОСЛЕ группировки
SELECT pos_tag, COUNT(*) AS total
FROM lemmas
GROUP BY pos_tag
HAVING COUNT(*) > 100;         -- оставляем только группы с более чем 100 леммами
```

**Ключевое правило:** в `HAVING` нельзя использовать псевдонимы из `SELECT` в MySQL 5.x, но в MySQL 8.x это допустимо. В `WHERE` нельзя использовать агрегатные функции.

### 5. Группировка по нескольким столбцам

```sql
SELECT author_name, pos_tag, COUNT(*) AS word_count
FROM tokens
JOIN lemmas ON tokens.lemma_id = lemmas.lemma_id
JOIN authors ON tokens.author_id = authors.author_id
GROUP BY author_name, pos_tag;
```

При группировке по нескольким столбцам создаётся отдельная группа для каждой уникальной комбинации значений.

### 6. Правило SELECT при GROUP BY

**В SELECT при использовании GROUP BY разрешено указывать только:**
1. Столбцы, перечисленные в `GROUP BY`
2. Агрегатные функции
3. Константы и выражения над п.1–2

```sql
-- ОШИБКА: word_form не входит в GROUP BY и не агрегирована
SELECT pos_tag, word_form, COUNT(*)   
FROM lemmas
GROUP BY pos_tag;

-- ПРАВИЛЬНО
SELECT pos_tag, COUNT(*), MAX(word_form)
FROM lemmas
GROUP BY pos_tag;
```

## Подготовка базы данных

Выполните следующие скрипты для создания и заполнения учебных таблиц.

### Создание таблиц

```sql
CREATE DATABASE IF NOT EXISTS corpus_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE corpus_db;

CREATE TABLE IF NOT EXISTS authors (
    author_id   INT PRIMARY KEY AUTO_INCREMENT,
    author_name VARCHAR(100) NOT NULL,
    birth_year  INT,
    literary_period VARCHAR(50)
);

CREATE TABLE IF NOT EXISTS texts (
    text_id     INT PRIMARY KEY AUTO_INCREMENT,
    author_id   INT NOT NULL,
    title       VARCHAR(200) NOT NULL,
    genre       VARCHAR(50),
    pub_year    INT,
    FOREIGN KEY (author_id) REFERENCES authors(author_id)
);

CREATE TABLE IF NOT EXISTS lemmas (
    lemma_id    INT PRIMARY KEY AUTO_INCREMENT,
    lemma_form  VARCHAR(100) NOT NULL,
    pos_tag     VARCHAR(20)  NOT NULL,   -- часть речи: NOUN, VERB, ADJ, ADV, PREP, CONJ ...
    word_length INT GENERATED ALWAYS AS (CHAR_LENGTH(lemma_form)) STORED
);

CREATE TABLE IF NOT EXISTS tokens (
    token_id    INT PRIMARY KEY AUTO_INCREMENT,
    text_id     INT NOT NULL,
    lemma_id    INT NOT NULL,
    word_form   VARCHAR(100) NOT NULL,
    position    INT NOT NULL,
    frequency   INT DEFAULT 1,
    FOREIGN KEY (text_id)  REFERENCES texts(text_id),
    FOREIGN KEY (lemma_id) REFERENCES lemmas(lemma_id)
);

CREATE TABLE IF NOT EXISTS morphemes (
    morpheme_id   INT PRIMARY KEY AUTO_INCREMENT,
    lemma_id      INT NOT NULL,
    morpheme_form VARCHAR(50) NOT NULL,
    morpheme_type VARCHAR(20) NOT NULL,  -- ROOT, PREFIX, SUFFIX, ENDING, POSTFIX
    FOREIGN KEY (lemma_id) REFERENCES lemmas(lemma_id)
);
```

## Примеры запросов с пояснениями

### Пример 1. Количество лемм по частям речи

```sql
SELECT pos_tag,
       COUNT(*) AS lemma_count
FROM lemmas
GROUP BY pos_tag
ORDER BY lemma_count DESC;
```

**Результат:**

| pos_tag | lemma_count |
|---------|-------------|
| NOUN    | 10          |
| VERB    | 10          |
| ADJ     | 7           |
| ADV     | 5           |
| PREP    | 5           |
| CONJ    | 3           |

### Пример 2. Суммарная частота токенов по тексту

```sql
SELECT t.title,
       SUM(tk.frequency) AS total_tokens
FROM texts t
JOIN tokens tk ON t.text_id = tk.text_id
GROUP BY t.text_id, t.title
ORDER BY total_tokens DESC;
```

### Пример 3. Средняя длина леммы по части речи

```sql
SELECT pos_tag,
       ROUND(AVG(word_length), 2) AS avg_length,
       MIN(word_length)           AS min_length,
       MAX(word_length)           AS max_length
FROM lemmas
GROUP BY pos_tag
ORDER BY avg_length DESC;
```

### Пример 4. Фильтрация групп с HAVING

```sql
-- Части речи, у которых средняя длина леммы больше 4 символов
SELECT pos_tag,
       ROUND(AVG(word_length), 2) AS avg_length
FROM lemmas
GROUP BY pos_tag
HAVING AVG(word_length) > 4
ORDER BY avg_length DESC;
```

### Пример 5. Комбинация WHERE и HAVING

```sql
-- Авторы-реалисты, у которых суммарная частота токенов превышает 1000
SELECT a.author_name,
       SUM(tk.frequency) AS total_freq
FROM authors a
JOIN texts t   ON a.author_id = t.author_id
JOIN tokens tk ON t.text_id   = tk.text_id
WHERE a.literary_period = 'Реализм'       -- WHERE фильтрует строки до группировки
GROUP BY a.author_id, a.author_name
HAVING SUM(tk.frequency) > 1000           -- HAVING фильтрует группы после
ORDER BY total_freq DESC;
```

### Пример 6. Группировка по нескольким столбцам

```sql
-- Количество токенов по каждому жанру и части речи
SELECT t.genre,
       l.pos_tag,
       COUNT(tk.token_id) AS token_count
FROM texts t
JOIN tokens tk ON t.text_id  = tk.text_id
JOIN lemmas l  ON tk.lemma_id = l.lemma_id
GROUP BY t.genre, l.pos_tag
ORDER BY t.genre, token_count DESC;
```

### Пример 7. GROUP_CONCAT — объединение значений в строку

```sql
-- Список словоформ для каждой части речи
SELECT pos_tag,
       GROUP_CONCAT(lemma_form ORDER BY lemma_form SEPARATOR ', ') AS lemmas_list
FROM lemmas
GROUP BY pos_tag;
```

**Задание D3.** Для каждого литературного направления найдите самую длинную лемму, встречающуюся в текстах авторов этого направления (используйте `MAX(word_length)` после серии JOIN).

**Задание D4 (оконные функции — дополнительно).** Используя `ROW_NUMBER()` или `RANK()`, пронумеруйте леммы внутри каждой части речи в порядке убывания их суммарной частоты по всему корпусу. Выведите топ-3 леммы для каждой части речи.

```sql
-- Подсказка к D4
SELECT pos_tag, lemma_form, total_freq,
       RANK() OVER (PARTITION BY pos_tag ORDER BY total_freq DESC) AS freq_rank
FROM (
    SELECT l.pos_tag, l.lemma_form, SUM(tk.frequency) AS total_freq
    FROM lemmas l
    JOIN tokens tk ON l.lemma_id = tk.lemma_id
    GROUP BY l.lemma_id, l.pos_tag, l.lemma_form
) AS freq_table
QUALIFY freq_rank <= 3;   -- в MySQL используйте подзапрос вместо QUALIFY
```
