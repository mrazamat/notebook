## Теоретический раздел

### 1. Что такое выражение в SQL?

**Выражение (expression)** — это комбинация значений, операторов и функций, которая вычисляется и возвращает результат. Выражения могут использоваться в `SELECT`, `WHERE`, `ORDER BY`, `GROUP BY`, `HAVING` и других частях запроса.

```
SELECT <выражение> FROM <таблица> WHERE <условие-выражение>;
```

### 2. Арифметические выражения

Арифметические операторы применяются к числовым столбцам и литералам:

| Оператор | Описание         | Пример                          |
|----------|------------------|---------------------------------|
| `+`      | Сложение         | `frequency + 10`                |
| `-`      | Вычитание        | `total_words - stop_words`      |
| `*`      | Умножение        | `frequency * 1.5`               |
| `/`      | Деление          | `frequency / total_words`       |
| `%`      | Остаток от деления | `word_id % 2`                 |
| `DIV`    | Целочисленное деление | `total_words DIV 100`      |

**Псевдонимы (алиасы)** — переименование выражения в результате:

```sql
SELECT word_form, frequency * 100.0 / total AS frequency_percent
FROM word_forms;
```

**Приоритет операций:** `*`, `/`, `%` выполняются раньше `+` и `-`. Для изменения порядка используются круглые скобки.

### 3. Строковые выражения

| Оператор / Функция           | Описание                              | Пример                                         |
|------------------------------|---------------------------------------|------------------------------------------------|
| `CONCAT(s1, s2, ...)`        | Конкатенация строк                    | `CONCAT(lemma, ' (', pos, ')')`                |
| `CONCAT_WS(sep, s1, s2, ...)` | Конкатенация с разделителем          | `CONCAT_WS(', ', author, title)`               |
| `LENGTH(s)`                  | Длина строки в байтах                 | `LENGTH(word_form)`                             |
| `CHAR_LENGTH(s)`             | Длина строки в символах               | `CHAR_LENGTH(lemma)`                            |
| `UPPER(s)` / `LOWER(s)`      | Регистр                               | `UPPER(pos)`                                    |
| `SUBSTRING(s, pos, len)`     | Подстрока                             | `SUBSTRING(word_form, 1, 3)`                   |
| `TRIM(s)`                    | Удаление пробелов                     | `TRIM(lemma)`                                   |
| `REPLACE(s, from, to)`       | Замена подстроки                      | `REPLACE(word_form, 'ё', 'е')`                 |
| `REPEAT(s, n)`               | Повторение строки                     | `REPEAT('*', frequency)`                        |
| `LPAD(s, len, pad)`          | Дополнение слева                      | `LPAD(word_id, 5, '0')`                         |

### 4. Логические выражения

Логические выражения возвращают `TRUE`, `FALSE` или `NULL`. Используются в `WHERE`, `HAVING`, `CASE WHEN`.

| Оператор     | Описание                              |
|--------------|---------------------------------------|
| `AND`        | Логическое «и»                        |
| `OR`         | Логическое «или»                      |
| `NOT`        | Логическое «не»                       |
| `IS NULL`    | Проверка на NULL                      |
| `IS NOT NULL`| Проверка на не-NULL                   |
| `IN (...)`   | Принадлежность множеству              |
| `BETWEEN`    | Диапазон (включительно)               |
| `LIKE`       | Шаблонный поиск                       |

### 5. Условные выражения: CASE WHEN

`CASE WHEN` позволяет возвращать разные значения в зависимости от условия — аналог `if/else` в программировании.

**Синтаксис (поисковая форма):**

```sql
CASE
    WHEN условие1 THEN результат1
    WHEN условие2 THEN результат2
    ...
    ELSE результат_по_умолчанию
END
```

**Синтаксис (простая форма):**

```sql
CASE выражение
    WHEN значение1 THEN результат1
    WHEN значение2 THEN результат2
    ...
    ELSE результат_по_умолчанию
END
```

**Вложенный CASE:**

```sql
CASE
    WHEN условие1 THEN
        CASE
            WHEN вложенное_условие THEN результат
            ELSE другой_результат
        END
    ELSE результат_по_умолчанию
END
```

> **Важно:** `CASE WHEN` возвращает `NULL`, если ни одно условие не совпало и `ELSE` не указан.

### 6. Псевдонимы столбцов и таблиц

```sql
-- Псевдоним столбца
SELECT frequency * 100.0 / total AS freq_percent FROM word_forms;

-- Псевдоним таблицы
SELECT wf.word_form, l.lemma
FROM word_forms AS wf
JOIN lemmas AS l ON wf.lemma_id = l.lemma_id;
```

> **Важно:** Псевдоним столбца (алиас из `SELECT`) **нельзя** использовать в `WHERE` того же запроса — только в `ORDER BY` и `HAVING`. Используйте подзапрос или повторите выражение.

## Схема базы данных

В этой лабораторной работе используется база данных лингвистического корпуса. Выполните следующие SQL-скрипты для создания и заполнения таблиц.

### Создание таблиц

```sql
CREATE TABLE authors (
    author_id   INT PRIMARY KEY AUTO_INCREMENT,
    last_name   VARCHAR(100) NOT NULL,
    first_name  VARCHAR(100) NOT NULL,
    birth_year  INT,
    death_year  INT
);

CREATE TABLE texts (
    text_id     INT PRIMARY KEY AUTO_INCREMENT,
    title       VARCHAR(255) NOT NULL,
    author_id   INT,
    genre       VARCHAR(50),
    year_written INT,
    word_count  INT,
    FOREIGN KEY (author_id) REFERENCES authors(author_id)
);

CREATE TABLE lemmas (
    lemma_id    INT PRIMARY KEY AUTO_INCREMENT,
    lemma       VARCHAR(100) NOT NULL,
    pos         VARCHAR(20),       -- part of speech
    is_stop_word TINYINT DEFAULT 0
);

CREATE TABLE word_forms (
    form_id     INT PRIMARY KEY AUTO_INCREMENT,
    word_form   VARCHAR(100) NOT NULL,
    lemma_id    INT,
    frequency   INT DEFAULT 0,
    text_id     INT,
    syllable_count INT,
    FOREIGN KEY (lemma_id) REFERENCES lemmas(lemma_id),
    FOREIGN KEY (text_id)  REFERENCES texts(text_id)
);

CREATE TABLE morphemes (
    morpheme_id   INT PRIMARY KEY AUTO_INCREMENT,
    morpheme      VARCHAR(50) NOT NULL,
    morpheme_type VARCHAR(20),     -- root, prefix, suffix, ending
    lemma_id      INT,
    FOREIGN KEY (lemma_id) REFERENCES lemmas(lemma_id)
);
```

```sql
FROM texts AS t
JOIN authors AS a ON t.author_id = a.author_id
GROUP BY a.author_id, a.last_name, a.first_name;
```
