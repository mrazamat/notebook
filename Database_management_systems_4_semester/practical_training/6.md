# Фильтрация данных в MySQL

**Предмет:** Система управления базами данных
**Специальность:** Компьютерная лингвистика

## Теоретический материал

### Что такое фильтрация данных?

Фильтрация данных — это процесс выборки строк из таблицы, удовлетворяющих заданным условиям. В MySQL фильтрация выполняется с помощью оператора `WHERE` в составе запроса `SELECT`.

Общий синтаксис:

```sql
SELECT столбец1, столбец2, ...
FROM таблица
WHERE условие;
```

### Операторы сравнения

| Оператор      | Описание         | Пример            |
|---------------|------------------|-------------------|
| `=`           | Равно            | `pos = 'NOUN'`    |
| `<>` или `!=` | Не равно         | `frequency <> 0`  |
| `>`           | Больше           | `frequency > 100` |
| `<`           | Меньше           | `length < 5`      |
| `>=`          | Больше или равно | `frequency >= 50` |
| `<=`          | Меньше или равно | `doc_count <= 10` |

### Оператор BETWEEN

Позволяет задать диапазон значений (включительно):

```sql
SELECT lemma FROM words
WHERE frequency BETWEEN 10 AND 100;
```

### Оператор IN

Проверяет, входит ли значение в список:

```sql
SELECT lemma FROM words
WHERE pos IN ('NOUN', 'VERB', 'ADJ');
```

### Оператор LIKE

Используется для поиска по шаблону в строках:

```sql
SELECT lemma FROM words
WHERE lemma LIKE 'пре%';   -- начинается на «пре»
WHERE lemma LIKE '%ость';  -- заканчивается на «ость»
WHERE lemma LIKE '%лов%';  -- содержит «лов»
WHERE lemma LIKE '_дти';   -- ровно один любой символ перед «дти»
```

Символы шаблона:
- `%` — любое количество любых символов (в том числе ноль)
- `_` — ровно один любой символ

### Оператор IS NULL / IS NOT NULL

Проверяет наличие или отсутствие значения:

```sql
SELECT lemma FROM words
WHERE gloss IS NULL;

SELECT lemma FROM words
WHERE gloss IS NOT NULL;
```

### Логические операторы AND, OR, NOT

Условия можно комбинировать:

```sql
-- AND: оба условия должны быть истинны
SELECT lemma FROM words
WHERE pos = 'VERB' AND frequency > 50;

-- OR: хотя бы одно из условий истинно
SELECT lemma FROM words
WHERE pos = 'NOUN' OR pos = 'PRON';

-- NOT: отрицание условия
SELECT lemma FROM words
WHERE NOT pos = 'PART';
```

Приоритет операторов: `NOT` → `AND` → `OR`. Используйте скобки для явного задания порядка:

```sql
SELECT lemma FROM words
WHERE (pos = 'NOUN' OR pos = 'ADJ') AND frequency > 100;
```

### Сравнение с NULL

Важно: сравнение вида `column = NULL` всегда возвращает `NULL` (не `TRUE`). Для проверки на `NULL` следует использовать только `IS NULL` или `IS NOT NULL`.

```sql
-- Неверно:
WHERE gloss = NULL

-- Верно:
WHERE gloss IS NULL
```

## Схема учебной базы данных

В этой лабораторной работе используется база данных лингвистического корпуса русского языка. Выполните следующие DDL- и DML-скрипты, чтобы создать и заполнить таблицы.

### DDL — создание таблиц

```sql
CREATE DATABASE IF NOT EXISTS corpus_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE corpus_db;

-- Корпуса текстов
CREATE TABLE IF NOT EXISTS corpora (
    corpus_id   INT AUTO_INCREMENT PRIMARY KEY,
    corpus_name VARCHAR(100) NOT NULL,
    language    VARCHAR(50)  NOT NULL,
    domain      VARCHAR(50),          -- жанр / область (fiction, news, academic…)
    doc_count   INT          NOT NULL DEFAULT 0,
    token_count BIGINT       NOT NULL DEFAULT 0
);

-- Тексты (документы) в корпусах
CREATE TABLE IF NOT EXISTS texts (
    text_id    INT AUTO_INCREMENT PRIMARY KEY,
    corpus_id  INT          NOT NULL,
    title      VARCHAR(200) NOT NULL,
    author     VARCHAR(100),
    pub_year   SMALLINT,
    word_count INT          NOT NULL DEFAULT 0,
    FOREIGN KEY (corpus_id) REFERENCES corpora(corpus_id)
);

-- Леммы (словарные формы)
CREATE TABLE IF NOT EXISTS lemmas (
    lemma_id    INT AUTO_INCREMENT PRIMARY KEY,
    lemma       VARCHAR(100) NOT NULL,
    pos         VARCHAR(20)  NOT NULL, -- часть речи: NOUN, VERB, ADJ, ADV, PRON, PREP, CONJ, PART, INTJ
    frequency   INT          NOT NULL DEFAULT 0,
    gloss       VARCHAR(200),          -- краткий перевод / толкование (может быть NULL)
    is_stopword TINYINT(1)   NOT NULL DEFAULT 0
);

-- Словоформы конкретного текста
CREATE TABLE IF NOT EXISTS word_forms (
    form_id    INT AUTO_INCREMENT PRIMARY KEY,
    text_id    INT          NOT NULL,
    lemma_id   INT          NOT NULL,
    word_form  VARCHAR(100) NOT NULL,
    position   INT          NOT NULL, -- позиция в тексте
    FOREIGN KEY (text_id)  REFERENCES texts(text_id),
    FOREIGN KEY (lemma_id) REFERENCES lemmas(lemma_id)
);
```

## Практические задания

### Задание 1. Простая фильтрация с оператором `=`

Напишите запросы:

1. Выведите все леммы, у которых часть речи (`pos`) равна `'NOUN'`.
2. Выведите названия текстов (`title`), опубликованных в 1866 году.
3. Выведите все корпуса, язык которых — `'Russian'`.
4. Выведите все леммы, которые являются стоп-словами (`is_stopword = 1`).
5. Найдите лемму со значением `lemma = 'язык'` и выведите её `pos`, `frequency` и `gloss`.

### Задание 2. Операторы сравнения `>`, `<`, `>=`, `<=`, `<>`

Напишите запросы:

1. Выведите леммы с частотой (`frequency`) больше 50 000.
2. Выведите тексты с количеством слов (`word_count`) менее 100 000.
3. Выведите корпуса, в которых число токенов (`token_count`) не менее 10 000 000.
4. Выведите леммы, у которых `frequency` меньше или равна 3 000.
5. Выведите все леммы, у которых часть речи **не равна** `'NOUN'`.

### Задание 3. Оператор `BETWEEN`

Напишите запросы:

1. Выведите леммы с частотой от 5 000 до 30 000 включительно.
2. Выведите тексты, опубликованные в период с 1850 по 1870 год.
3. Выведите корпуса, в которых число документов (`doc_count`) от 60 до 200.
4. Выведите словоформы из таблицы `word_forms`, чья позиция (`position`) находится в диапазоне от 5 до 15.
5. Выведите леммы с частотой **вне** диапазона от 1 000 до 10 000 (используйте `NOT BETWEEN`).

### Задание 4. Оператор `IN`

Напишите запросы:

1. Выведите леммы, у которых `pos` — одна из следующих частей речи: `'NOUN'`, `'VERB'`, `'ADJ'`.
2. Выведите тексты, чей `corpus_id` равен 2 или 4.
3. Выведите леммы со значениями `'язык'`, `'лемма'`, `'корпус'`, `'морфема'`.
4. Выведите корпуса с доменом (`domain`) `'fiction'` или `'poetry'`.
5. Выведите леммы, у которых `pos` **не входит** в список `('CONJ', 'PREP', 'PART', 'INTJ')` (используйте `NOT IN`).

### Задание 5. Оператор `LIKE`

Напишите запросы:

1. Выведите леммы, начинающиеся на букву «с».
2. Выведите леммы, оканчивающиеся на «ость».
3. Выведите леммы, содержащие подстроку «форм».
4. Выведите названия текстов (`title`), содержащих слово «корпус».
5. Выведите леммы, у которых вторая буква — «а» (используйте `_`).
6. Выведите все корпуса, название которых начинается со слова «Корпус».

### Задание 6. Оператор `IS NULL` / `IS NOT NULL`

Напишите запросы:

1. Выведите леммы, у которых поле `gloss` не заполнено (`IS NULL`).
2. Выведите леммы, у которых `gloss` заполнен.
3. Выведите тексты, у которых не указан автор (`author IS NULL`).
4. Выведите тексты, у которых автор указан.
5. Выведите названия корпусов, у которых поле `domain` заполнено.

### Задание 7. Логические операторы `AND`, `OR`, `NOT`

Напишите запросы:

1. Выведите леммы с `pos = 'NOUN'` **и** `frequency > 20 000`.
2. Выведите леммы с `pos = 'VERB'` **или** `pos = 'ADV'`.
3. Выведите тексты корпуса №2 с `word_count > 150 000`.
4. Выведите леммы, которые **не** являются стоп-словами (`NOT is_stopword`), **и** у которых `frequency` больше 10 000.
5. Выведите леммы, у которых `pos = 'NOUN'`, `gloss IS NOT NULL` и `frequency` между 2 000 и 40 000.
6. Выведите леммы, у которых (`pos = 'NOUN'` **или** `pos = 'ADJ'`) **и** `is_stopword = 0`.

### Задание 8. Комбинированные запросы

Напишите запросы, сочетающие несколько видов фильтрации:

1. Найдите все лингвистические термины (слова из таблицы `lemmas`, содержащие «морф», «фон», «лекс», «синт» или «грамм» в любой позиции), у которых `frequency > 1000`.
2. Выведите леммы с `pos IN ('NOUN', 'VERB')`, `frequency BETWEEN 3000 AND 60000` и `gloss IS NOT NULL`.
3. Выведите тексты, опубликованные **до** 1900 года, у которых `word_count >= 100 000` **или** автор — Толстой.
4. Выведите леммы, которые **не** являются стоп-словами, **не** относятся к частям речи `'INTJ'` и `'PART'`, и частота которых не менее 5 000.
5. Выведите все словоформы (`word_form`, `position`) из текста с `text_id = 1`, чья позиция нечётная (подсказка: используйте `position % 2 = 1`).
