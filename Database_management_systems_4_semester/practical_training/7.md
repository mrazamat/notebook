## Теоретический материал

### Что такое JOIN?

JOIN — оператор SQL, позволяющий объединять строки из двух или более таблиц на основе связанных столбцов. В реляционных базах данных информация намеренно разделяется по нескольким таблицам, чтобы избежать избыточности. JOIN восстанавливает связи между этими данными во время запроса.

Общий синтаксис:

```sql
SELECT <столбцы>
FROM <таблица_A>
<ТИП_JOIN> <таблица_B>
ON <таблица_A>.<ключ> = <таблица_B>.<ключ>;
```

### Типы объединений

| Тип JOIN | Описание | Что включается в результат |
|---|---|---|
| `INNER JOIN` | Внутреннее объединение | Только строки с совпадениями в обеих таблицах |
| `LEFT JOIN` | Левое внешнее | Все строки левой таблицы + совпадения из правой |
| `RIGHT JOIN` | Правое внешнее | Все строки правой таблицы + совпадения из левой |
| `FULL OUTER JOIN` | Полное внешнее | Все строки из обеих таблиц |
| `CROSS JOIN` | Декартово произведение | Все возможные комбинации строк |
| `SELF JOIN` | Самосоединение | Таблица объединяется сама с собой |

### Визуальная схема типов JOIN

```
Таблица A      Таблица B

  [A ∩ B]  ← INNER JOIN: только пересечение

  [A] + [A ∩ B]  ← LEFT JOIN: вся левая + пересечение

  [A ∩ B] + [B]  ← RIGHT JOIN: пересечение + вся правая

  [A] + [A ∩ B] + [B]  ← FULL OUTER JOIN: всё
```

### INNER JOIN

Возвращает только те строки, для которых найдено совпадение в обеих таблицах.

```sql
SELECT a.column1, b.column2
FROM table_a AS a
INNER JOIN table_b AS b
ON a.id = b.a_id;
```

### LEFT JOIN

Возвращает все строки левой таблицы. Если совпадения в правой таблице нет — значения из правой таблицы будут NULL.

```sql
SELECT a.column1, b.column2
FROM table_a AS a
LEFT JOIN table_b AS b
ON a.id = b.a_id;
```

### RIGHT JOIN

Аналогично LEFT JOIN, но приоритет у правой таблицы.

```sql
SELECT a.column1, b.column2
FROM table_a AS a
RIGHT JOIN table_b AS b
ON a.id = b.a_id;
```

### FULL OUTER JOIN

Возвращает все строки из обеих таблиц. Там, где нет совпадения, подставляется NULL. **Важно:** MySQL не поддерживает FULL OUTER JOIN напрямую — его эмулируют через UNION:

```sql
SELECT a.column1, b.column2
FROM table_a AS a
LEFT JOIN table_b AS b ON a.id = b.a_id
UNION
SELECT a.column1, b.column2
FROM table_a AS a
RIGHT JOIN table_b AS b ON a.id = b.a_id;
```

### CROSS JOIN

Возвращает декартово произведение — каждую строку первой таблицы объединяет с каждой строкой второй. Используется редко, но полезен для генерации комбинаций.

```sql
SELECT a.column1, b.column2
FROM table_a AS a
CROSS JOIN table_b AS b;
```

Если в `table_a` 4 строки, а в `table_b` 3 строки — результат будет содержать 4 × 3 = 12 строк.

### SELF JOIN

Таблица объединяется сама с собой. Для этого обязательно используются псевдонимы (алиасы):

```sql
SELECT a.column1 AS name1, b.column1 AS name2
FROM table_a AS a
INNER JOIN table_a AS b
ON a.related_id = b.id;
```

### Использование псевдонимов (алиасов)

При работе с JOIN рекомендуется всегда использовать псевдонимы таблиц и уточнять принадлежность столбцов:

```sql
-- Без псевдонима — громоздко и неоднозначно
SELECT corpus_texts.title, authors.full_name
FROM corpus_texts
INNER JOIN authors ON corpus_texts.author_id = authors.id;

-- С псевдонимом — компактно и читаемо
SELECT t.title, a.full_name
FROM corpus_texts AS t
INNER JOIN authors AS a ON t.author_id = a.id;
```

### Объединение более двух таблиц

JOIN можно соединять цепочкой:

```sql
SELECT t.title, a.full_name, l.lemma_text
FROM corpus_texts AS t
INNER JOIN authors AS a ON t.author_id = a.id
INNER JOIN word_forms AS wf ON wf.text_id = t.id
INNER JOIN lemmas AS l ON wf.lemma_id = l.id;
```

## Подготовка учебной базы данных

### Создание таблиц

Выполните следующие SQL-команды для создания учебной схемы лингвистического корпуса:

```sql
-- Авторы произведений
CREATE TABLE authors (
    id        INT PRIMARY KEY AUTO_INCREMENT,
    full_name VARCHAR(100) NOT NULL,
    birth_year INT,
    century    VARCHAR(10)
);

-- Тексты корпуса
CREATE TABLE corpus_texts (
    id        INT PRIMARY KEY AUTO_INCREMENT,
    title     VARCHAR(200) NOT NULL,
    author_id INT,
    genre     VARCHAR(50),
    word_count INT,
    FOREIGN KEY (author_id) REFERENCES authors(id)
);

-- Части речи
CREATE TABLE pos_tags (
    id       INT PRIMARY KEY AUTO_INCREMENT,
    tag_code VARCHAR(10) NOT NULL UNIQUE,
    tag_name VARCHAR(50) NOT NULL
);

-- Леммы (начальные формы слов)
CREATE TABLE lemmas (
    id         INT PRIMARY KEY AUTO_INCREMENT,
    lemma_text VARCHAR(100) NOT NULL,
    pos_id     INT,
    frequency  INT DEFAULT 0,
    FOREIGN KEY (pos_id) REFERENCES pos_tags(id)
);

-- Словоформы (конкретные вхождения слов в тексты)
CREATE TABLE word_forms (
    id        INT PRIMARY KEY AUTO_INCREMENT,
    form_text VARCHAR(100) NOT NULL,
    lemma_id  INT,
    text_id   INT,
    position  INT,
    FOREIGN KEY (lemma_id) REFERENCES lemmas(id),
    FOREIGN KEY (text_id)  REFERENCES corpus_texts(id)
);

-- Морфемы
CREATE TABLE morphemes (
    id            INT PRIMARY KEY AUTO_INCREMENT,
    morpheme_text VARCHAR(50) NOT NULL,
    morpheme_type VARCHAR(20) NOT NULL  -- root, prefix, suffix, ending
);

-- Связь лемм и морфем (многие ко многим)
CREATE TABLE lemma_morphemes (
    lemma_id    INT NOT NULL,
    morpheme_id INT NOT NULL,
    position    INT,
    PRIMARY KEY (lemma_id, morpheme_id),
    FOREIGN KEY (lemma_id)    REFERENCES lemmas(id),
    FOREIGN KEY (morpheme_id) REFERENCES morphemes(id)
);
```

## Примеры запросов с пояснениями

### Пример 1. INNER JOIN — тексты с авторами

Получить список текстов с именами их авторов:

```sql
SELECT
    t.title        AS text_title,
    a.full_name    AS author_name,
    t.genre,
    t.word_count
FROM corpus_texts AS t
INNER JOIN authors AS a ON t.author_id = a.id
ORDER BY a.full_name;
```

Результат включает только тексты, у которых указан автор. Текст «Шинель» (author_id = NULL) не попадёт в выборку.

### Пример 2. LEFT JOIN — все тексты, включая без автора

```sql
SELECT
    t.title     AS text_title,
    a.full_name AS author_name,
    t.genre
FROM corpus_texts AS t
LEFT JOIN authors AS a ON t.author_id = a.id
ORDER BY t.title;
```

Текст «Шинель» отобразится со значением NULL в столбце `author_name`.

### Пример 3. Трёхтабличный JOIN — словоформы с леммами и частями речи

```sql
SELECT
    wf.form_text    AS word_form,
    l.lemma_text    AS lemma,
    p.tag_name      AS part_of_speech,
    t.title         AS source_text
FROM word_forms AS wf
INNER JOIN lemmas       AS l ON wf.lemma_id = l.id
INNER JOIN pos_tags     AS p ON l.pos_id    = p.id
INNER JOIN corpus_texts AS t ON wf.text_id  = t.id
ORDER BY p.tag_name, l.lemma_text;
```

### Пример 4. LEFT JOIN для поиска лемм без морфемного разбора

```sql
SELECT
    l.lemma_text,
    p.tag_name AS pos,
    lm.morpheme_id
FROM lemmas AS l
LEFT JOIN pos_tags      AS p  ON l.pos_id    = p.id
LEFT JOIN lemma_morphemes AS lm ON lm.lemma_id = l.id
WHERE lm.morpheme_id IS NULL
ORDER BY l.lemma_text;
```

### Пример 5. CROSS JOIN — все комбинации авторов и жанров

```sql
SELECT
    a.full_name AS author,
    g.genre
FROM authors AS a
CROSS JOIN (
    SELECT DISTINCT genre FROM corpus_texts
) AS g
ORDER BY a.full_name, g.genre;
```

### Пример 6. SELF JOIN — авторы одного века

Найти все пары авторов, живших в одном веке:

```sql
SELECT
    a1.full_name AS author_1,
    a2.full_name AS author_2,
    a1.century
FROM authors AS a1
INNER JOIN authors AS a2
    ON  a1.century = a2.century
    AND a1.id < a2.id  -- исключить дублирование пар и самосоединение
ORDER BY a1.century, a1.full_name;
```

```sql
    t.word_count,
    RANK() OVER (PARTITION BY t.author_id ORDER BY t.word_count DESC) AS rank_by_size
FROM corpus_texts AS t
INNER JOIN authors AS a ON t.author_id = a.id;
```
